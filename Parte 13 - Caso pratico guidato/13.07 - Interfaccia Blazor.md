# 13.07 — Interfaccia Blazor

Nei capitoli precedenti abbiamo costruito il backend del progetto finale: dominio, use case, EF Core, repository, API REST, OpenAPI e test.

Ora realizziamo il client web con **Blazor**.

Il punto centrale è questo:

```text id="y9v4zz"
Blazor sarà il client principale del gestionale editoriale:
una UI web per dashboard, CRUD, form, tabelle, report
e chiamate HTTP verso App.Api.
```

Blazor è adatto a questo progetto perché permette di costruire UI web con componenti Razor e C#, integrandosi bene con ASP.NET Core, API, autenticazione e Dependency Injection. Il modello Blazor Web App è il punto di partenza moderno per creare UI web con componenti Razor e supporta rendering lato server, lato client, streaming rendering e interattività configurabile per componente. ([Microsoft Learn][1])

# CatalogoApiClient

`Services/CatalogoApiClient.cs`:

```csharp id="x1el26"
using System.Net.Http.Json;
using App.Application.Catalogo.Libri;

namespace App.Web.Services;

public class CatalogoApiClient
{
    private readonly HttpClient httpClient;

    public CatalogoApiClient(HttpClient httpClient)
    {
        this.httpClient = httpClient;
    }

    public async Task<IReadOnlyList<LibroListItemDto>> GetLibriAsync(
        CancellationToken cancellationToken = default)
    {
        IReadOnlyList<LibroListItemDto>? libri =
            await httpClient.GetFromJsonAsync<IReadOnlyList<LibroListItemDto>>(
                "/api/catalogo/libri",
                cancellationToken);

        return libri ?? [];
    }

    public async Task<LibroDetailDto?> GetLibroAsync(
        int id,
        CancellationToken cancellationToken = default)
    {
        return await httpClient.GetFromJsonAsync<LibroDetailDto>(
            $"/api/catalogo/libri/{id}",
            cancellationToken);
    }

    public async Task<int> CreateLibroAsync(
        CreateLibroRequest request,
        CancellationToken cancellationToken = default)
    {
        HttpResponseMessage response =
            await httpClient.PostAsJsonAsync(
                "/api/catalogo/libri",
                request,
                cancellationToken);

        await EnsureSuccessOrThrowAsync(response);

        CreatedIdResponse? created =
            await response.Content.ReadFromJsonAsync<CreatedIdResponse>(
                cancellationToken);

        return created?.Id
            ?? throw new InvalidOperationException(
                "La risposta di creazione non contiene l'Id.");
    }

    public async Task UpdateLibroAsync(
        int id,
        UpdateLibroRequest request,
        CancellationToken cancellationToken = default)
    {
        HttpResponseMessage response =
            await httpClient.PutAsJsonAsync(
                $"/api/catalogo/libri/{id}",
                request,
                cancellationToken);

        await EnsureSuccessOrThrowAsync(response);
    }

    public async Task DisattivaLibroAsync(
        int id,
        CancellationToken cancellationToken = default)
    {
        HttpResponseMessage response =
            await httpClient.DeleteAsync(
                $"/api/catalogo/libri/{id}",
                cancellationToken);

        await EnsureSuccessOrThrowAsync(response);
    }

    private static async Task EnsureSuccessOrThrowAsync(
        HttpResponseMessage response)
    {
        if (response.IsSuccessStatusCode)
        {
            return;
        }

        string content = await response.Content.ReadAsStringAsync();

        throw new ApiException(
            response.StatusCode,
            string.IsNullOrWhiteSpace(content)
                ? "Errore durante la chiamata API."
                : content);
    }

    private sealed record CreatedIdResponse(int Id);
}
```

`ApiException.cs`:

```csharp id="p7spvf"
using System.Net;

namespace App.Web.Services;

public class ApiException : Exception
{
    public ApiException(HttpStatusCode statusCode, string message)
        : base(message)
    {
        StatusCode = statusCode;
    }

    public HttpStatusCode StatusCode { get; }
}
```

# Pagina creazione/modifica libro

`Pages/LibroEdit.razor`:

```razor id="lks9rb"
@page "/libri/nuovo"

        <div class="form-group">
            <label>ISBN</label>
            <InputText class="form-control"
                       @bind-Value="model.Isbn" />
            <ValidationMessage For="@(() => model.Isbn)" />
        </div>

        <div class="form-group">
            <label>Prezzo</label>
            <InputNumber class="form-control"
                         @bind-Value="model.Prezzo" />
            <ValidationMessage For="@(() => model.Prezzo)" />
        </div>

        <div class="form-group">
            <label>Data pubblicazione</label>
            <InputDate class="form-control"
                       @bind-Value="model.DataPubblicazione" />
        </div>

        @if (Id is not null)
        {
            <div class="form-check">
                <InputCheckbox class="form-check-input"
                               @bind-Value="model.Attivo" />
                <label class="form-check-label">
                    Attivo
                </label>
            </div>
        }

        <div class="form-actions">
            <button type="submit"
                    class="btn btn-primary"
                    disabled="@isSaving">
                @(isSaving ? "Salvataggio..." : "Salva")
            </button>

            <a class="btn btn-secondary" href="/libri">
                Annulla
            </a>
        </div>
    </EditForm>
}

@code {
    [Parameter]
    public int? Id { get; set; }

    private LibroFormModel model = new();
    private bool isLoading = true;
    private bool isSaving;
    private string? errorMessage;

    private string PageTitle =>
        Id is null ? "Nuovo libro" : "Modifica libro";

    protected override async Task OnInitializedAsync()
    {
        try
        {
            if (Id is not null)
            {
                LibroDetailDto? libro =
                    await CatalogoApiClient.GetLibroAsync(Id.Value);

                if (libro is null)
                {
                    errorMessage = "Libro non trovato.";
                    return;
                }

                model = new LibroFormModel
                {
                    Titolo = libro.Titolo,
                    Isbn = libro.Isbn,
                    Prezzo = libro.Prezzo,
                    DataPubblicazione = libro.DataPubblicazione,
                    Attivo = libro.Attivo
                };
            }
        }
        catch (ApiException ex)
        {
            errorMessage = ex.Message;
        }
        catch (Exception ex)
        {
            errorMessage = ex.Message;
        }
        finally
        {
            isLoading = false;
        }
    }

    private async Task HandleValidSubmitAsync()
    {
        isSaving = true;
        errorMessage = null;

        try
        {
            if (Id is null)
            {
                CreateLibroRequest request = new(
                    model.Titolo,
                    model.Isbn,
                    model.Prezzo,
                    model.DataPubblicazione,
                    model.CollanaId,
                    AutoriIds: []);

                await CatalogoApiClient.CreateLibroAsync(request);
            }
            else
            {
                UpdateLibroRequest request = new(
                    model.Titolo,
                    model.Isbn,
                    model.Prezzo,
                    model.DataPubblicazione,
                    model.CollanaId,
                    AutoriIds: [],
                    model.Attivo);

                await CatalogoApiClient.UpdateLibroAsync(
                    Id.Value,
                    request);
            }

            NavigationManager.NavigateTo("/libri");
        }
        catch (ApiException ex)
        {
            errorMessage = ex.Message;
        }
        catch (Exception ex)
        {
            errorMessage = ex.Message;
        }
        finally
        {
            isSaving = false;
        }
    }
}
```

# Validazione

La validazione avviene su più livelli:

```text id="a0y2pz"
UI Blazor
  aiuta l’utente prima dell’invio.

API
  protegge il backend da input non validi.

Application
  verifica regole applicative.

Domain
  protegge invarianti fondamentali.
```

La validazione UI non basta.

Regola:

```text id="ee2rb0"
Il client può sbagliare, essere vecchio o essere malevolo.
L’API deve validare sempre.
```

Blazor supporta `EditForm`, `EditContext`, `ValidationMessageStore` e validazione con DataAnnotations o logica custom; Microsoft mostra anche validazioni personalizzate con `EditContext` e `ValidationMessageStore`. ([Microsoft Learn][6])

```csharp id="xh5tvn"
            $"/api/documenti/{id}",
            cancellationToken);
    }

    public async Task<int> CreateDocumentoAsync(
        CreateDocumentoRequest request,

    public async Task ConfermaDocumentoAsync(
        int id,

    {

    public async Task AnnullaDocumentoAsync(
        int id,
        CancellationToken cancellationToken = default)
    {
        HttpResponseMessage response =
            await httpClient.PostAsync(

        await EnsureSuccessOrThrowAsync(response);
    }

    private static async Task EnsureSuccessOrThrowAsync(
        HttpResponseMessage response)
    {
        if (response.IsSuccessStatusCode)
        {
            return;
        }

        string content = await response.Content.ReadAsStringAsync();

        throw new ApiException(
            response.StatusCode,
            string.IsNullOrWhiteSpace(content)
                ? "Errore durante la chiamata API."
                : content);
    }

    private sealed record CreatedIdResponse(int Id);
}
```

Blazor usa i meccanismi di autenticazione di ASP.NET Core; il modo esatto dipende dal modello di hosting, lato server o lato client. Microsoft documenta anche `AuthenticationStateProvider`, che consente di fornire e osservare lo stato di autenticazione dell’utente nei componenti. ([Microsoft Learn][7])

# Token verso API

Se `App.Web` deve chiamare `App.Api` con JWT, il client HTTP deve aggiungere:


Nelle soluzioni Blazor con OIDC/Entra, le versioni recenti della documentazione Microsoft mostrano la chiamata sicura a una API separata tramite token handler e client HTTP nominato; le note ASP.NET Core .NET 10 indicano che gli esempi OIDC ed Entra includono una API Web separata per dimostrare questo scenario. ([Microsoft Learn][8])

Nel progetto finale possiamo presentare l’autenticazione come evoluzione:

```text id="fzacn6"

  Blazor senza autenticazione

fase 2:
  autenticazione in App.Web

fase 3:
  token verso App.Api

fase 4:
  policy su API e UI
```

Questo evita di sovraccaricare il primo sviluppo.

# Errori comuni

## Chiamare il database da Blazor

Da evitare nel nostro progetto.

Blazor deve chiamare l’API.

---

## Mettere troppa logica nelle pagine

Se una pagina supera rapidamente molte centinaia di righe, estrai componenti e servizi.

---

## Usare direttamente gli entity model

La UI deve usare DTO o modelli UI, non entity EF Core.

---

## Validare solo nella UI

La validazione Blazor migliora l’esperienza utente, ma l’API deve validare sempre.

---

## Non gestire caricamento ed errori

Ogni pagina che chiama API dovrebbe gestire:

```text id="g4i9gn"
isLoading
errorMessage
stato vuoto
```

---

## Dimenticare il doppio click sui submit

Durante il salvataggio:

```razor id="i1v5yi"
disabled="@isSaving"
```

---

## Nascondere pulsanti e pensare che basti per la sicurezza

La sicurezza deve stare anche nell’API.
