# 13.06 — API REST

Nel capitolo precedente abbiamo collegato il progetto a EF Core e SQLite: `DbContext`, mapping, migration, repository concreti, seed dati e test di integrazione.

Ora esponiamo il sistema verso l’esterno tramite una **API REST** con ASP.NET Core e Minimal API.

Il punto centrale è questo:

```text
Una buona API non è solo una lista di endpoint.
È un contratto stabile tra client e backend:
URL, metodi HTTP, DTO, validazioni, errori, sicurezza e documentazione.
```

Useremo Minimal API perché sono leggere, leggibili e adatte a un progetto didattico. Microsoft descrive le Minimal API come un modello per creare HTTP API con dipendenze minime, utile per microservizi e applicazioni che vogliono includere solo i file e le funzionalità necessari. ([Microsoft Learn][1])

# Metodi HTTP

Useremo:

```text
GET
  lettura

POST
  creazione o azione di dominio

PUT
  aggiornamento completo o sostanziale

DELETE
  cancellazione o disattivazione

PATCH
  aggiornamento parziale, se introdotto più avanti
```

Nel nostro progetto `DELETE /libri/{id}` non farà necessariamente una cancellazione fisica.

Potrà fare una disattivazione.

Questo va documentato bene.

```text
DELETE /api/catalogo/libri/10
  disattiva il libro se non deve essere eliminato fisicamente
```

In un gestionale reale, eliminare dati storici è spesso pericoloso.

# DTO

Le API non devono esporre direttamente le entity EF Core.

Errore:

```csharp
return Results.Ok(await dbContext.Libri.ToListAsync());
```

Problemi:

```text
esponi struttura interna
rischi cicli di serializzazione
le modifiche al dominio rompono il contratto API
esponi campi che non volevi esporre
mescoli persistenza e contratto HTTP
```

Meglio:

```csharp
public record LibroListItemDto(
    int Id,
    string Titolo,
    string? Isbn,
    decimal Prezzo,
    bool Attivo);
```

Regola:

```text
Entity:
  modello interno.

DTO:
  contratto verso client.
```

# Endpoint CRUD: Catalogo

`App.Api/Endpoints/CatalogoEndpoints.cs`:

```csharp
using App.Api.Extensions;
using App.Application.Catalogo.Libri;
using App.Application.Common;

namespace App.Api.Endpoints;

public static class CatalogoEndpoints
{
    public static IEndpointRouteBuilder MapCatalogoEndpoints(
        this IEndpointRouteBuilder app)
    {
        RouteGroupBuilder group = app
            .MapGroup("/api/catalogo")
            .WithTags("Catalogo");

        group.MapGet("/libri", GetLibriAsync)
            .WithName("GetLibri")
            .WithSummary("Restituisce l'elenco dei libri");

        group.MapGet("/libri/{id:int}", GetLibroAsync)
            .WithName("GetLibroById")
            .WithSummary("Restituisce il dettaglio di un libro");

        group.MapPost("/libri", CreateLibroAsync)
            .WithName("CreateLibro")
            .WithSummary("Crea un nuovo libro");

        group.MapPut("/libri/{id:int}", UpdateLibroAsync)
            .WithName("UpdateLibro")
            .WithSummary("Aggiorna un libro esistente");

        group.MapDelete("/libri/{id:int}", DisattivaLibroAsync)
            .WithName("DisattivaLibro")
            .WithSummary("Disattiva un libro");

        return app;
    }

    private static async Task<IResult> GetLibriAsync(
        ILibroService libroService,
        CancellationToken cancellationToken)
    {
        IReadOnlyList<LibroListItemDto> libri =
            await libroService.GetLibriAsync(cancellationToken);

        return Results.Ok(libri);
    }

    private static async Task<IResult> GetLibroAsync(
        int id,
        ILibroService libroService,
        CancellationToken cancellationToken)
    {
        Result<LibroDetailDto> result =
            await libroService.GetLibroAsync(id, cancellationToken);

        return result.ToHttpResult();
    }

    private static async Task<IResult> CreateLibroAsync(
        CreateLibroRequest request,
        ILibroService libroService,
        CancellationToken cancellationToken)
    {
        Result<int> result =
            await libroService.CreateLibroAsync(
                request,
                cancellationToken);

        if (!result.Success)
        {
            return result.ToHttpResult();
        }

        return Results.Created(
            $"/api/catalogo/libri/{result.Value}",
            new { id = result.Value });
    }

    private static async Task<IResult> UpdateLibroAsync(
        int id,
        UpdateLibroRequest request,
        ILibroService libroService,
        CancellationToken cancellationToken)
    {
        Result result =
            await libroService.UpdateLibroAsync(
                id,
                request,
                cancellationToken);

        return result.ToHttpResult();
    }

    private static async Task<IResult> DisattivaLibroAsync(
        int id,
        ILibroService libroService,
        CancellationToken cancellationToken)
    {
        Result result =
            await libroService.DisattivaLibroAsync(
                id,
                cancellationToken);

        return result.ToHttpResult();
    }
}
```

```text

trasforma il risultato in HTTP
```

# Endpoint Documenti

`App.Api/Endpoints/DocumentiEndpoints.cs`:

```csharp
using App.Api.Extensions;
using App.Application.Common;
using App.Application.Documenti;

namespace App.Api.Endpoints;

public static class DocumentiEndpoints
{
    public static IEndpointRouteBuilder MapDocumentiEndpoints(
        this IEndpointRouteBuilder app)
    {
        RouteGroupBuilder group = app
            .MapGroup("/api/documenti")
            .WithTags("Documenti");

        group.MapGet("/{id:int}", GetDocumentoByIdAsync)
            .WithName("GetDocumentoById")
            .WithSummary("Restituisce il dettaglio di un documento");

        group.MapPost("/", CreateDocumentoAsync)
            .WithName("CreateDocumento")
            .WithSummary("Crea un nuovo documento");

        group.MapPost("/{id:int}/conferma", ConfermaDocumentoAsync)
            .WithName("ConfermaDocumento")
            .WithSummary("Conferma un documento in bozza");

        group.MapPost("/{id:int}/annulla", AnnullaDocumentoAsync)
            .WithName("AnnullaDocumento")
            .WithSummary("Annulla un documento");

        return app;
    }

    private static async Task<IResult> GetDocumentoByIdAsync(
        int id,
        GetDocumentoByIdHandler handler,
        CancellationToken cancellationToken)
    {
        Result<DocumentoDetailDto> result =
            await handler.HandleAsync(
                new GetDocumentoByIdQuery(id),
                cancellationToken);

        return result.ToHttpResult();
    }

    private static async Task<IResult> CreateDocumentoAsync(
        CreateDocumentoRequest request,
        CreateDocumentoHandler handler,
        CancellationToken cancellationToken)
    {
        CreateDocumentoCommand command = new(
            request.Data,
            request.ClienteId,
            request.TipoDocumento,
            request.Righe
                .Select(r => new CreateDocumentoRigaCommand(
                    r.LibroId,
                    r.Quantita,
                    r.PrezzoUnitario,
                    r.ScontoPercentuale))
                .ToList());

        Result<int> result =
            await handler.HandleAsync(command, cancellationToken);

        if (!result.Success)
        {
            return result.ToHttpResult();
        }

        return Results.Created(
            $"/api/documenti/{result.Value}",
            new { id = result.Value });
    }

    private static async Task<IResult> ConfermaDocumentoAsync(
        int id,
        ConfermaDocumentoHandler handler,
        CancellationToken cancellationToken)
    {
        Result result =
            await handler.HandleAsync(
                new ConfermaDocumentoCommand(id),
                cancellationToken);

        return result.ToHttpResult();
    }

    private static async Task<IResult> AnnullaDocumentoAsync(
        int id,
        AnnullaDocumentoHandler handler,
        CancellationToken cancellationToken)
    {
        Result result =
            await handler.HandleAsync(
                new AnnullaDocumentoCommand(id),
                cancellationToken);

        return result.ToHttpResult();
    }
}
```

```text

```

# AnnullaDocumentoCommand e handler

Se non lo abbiamo ancora creato:

```csharp
namespace App.Application.Documenti;

public record AnnullaDocumentoCommand(int DocumentoId);
```

```csharp
using App.Application.Common;
using App.Domain.Common;
using App.Domain.Documenti;
using Microsoft.Extensions.Logging;

namespace App.Application.Documenti;

public class AnnullaDocumentoHandler
{
    private readonly IDocumentoRepository documentoRepository;
    private readonly ILogger<AnnullaDocumentoHandler> logger;

    public AnnullaDocumentoHandler(
        IDocumentoRepository documentoRepository,
        ILogger<AnnullaDocumentoHandler> logger)
    {
        this.documentoRepository = documentoRepository;
        this.logger = logger;
    }

    public async Task<Result> HandleAsync(
        AnnullaDocumentoCommand command,
        CancellationToken cancellationToken)
    {
        if (command.DocumentoId <= 0)
        {
            return Result.Fail("Documento non valido.");
        }

        Documento? documento =
            await documentoRepository.GetByIdAsync(
                command.DocumentoId,
                cancellationToken);

        if (documento is null)
        {
            return Result.Fail(
                "Documento non trovato.",
                ErrorType.NotFound);
        }

        try
        {
            documento.Annulla();

            await documentoRepository.SaveChangesAsync(
                cancellationToken);

            logger.LogInformation(
                "Documento {DocumentoId} annullato",
                documento.Id);

            return Result.Ok();
        }
        catch (DomainException ex)
        {
            return Result.Fail(
                ex.Message,
                ErrorType.Business);
        }
    }
}
```

Registrazione:

```csharp
services.AddScoped<AnnullaDocumentoHandler>();
```

# Validazione

La validazione può stare a più livelli:

```text
DTO / input HTTP
  dati obbligatori, formati, range semplici

Application
  cliente esiste, libro esiste, ISBN unico

Domain
  invarianti profonde: documento confermabile, sconto valido, quantità valida
```

Per il progetto base possiamo usare validazione manuale nei validator applicativi.

Esempio:

```csharp
public static class CreateLibroValidator
{
    public static Result Validate(CreateLibroRequest request)
    {
        if (string.IsNullOrWhiteSpace(request.Titolo))
        {
            return Result.Fail("Il titolo è obbligatorio.");
        }

        if (request.Prezzo < 0)
        {
            return Result.Fail("Il prezzo non può essere negativo.");
        }

        return Result.Ok();
    }
}
```

In ASP.NET Core .NET 10 è stata introdotta la validazione integrata per Minimal API, con endpoint filter applicati ai tipi rilevati negli handler; può essere disabilitata per endpoint specifici con `DisableValidation`. ([Microsoft Learn][4])

Per un libro didattico, conviene dire:

```text
Prima mostriamo validazione esplicita e comprensibile.
Poi possiamo citare validazione automatica, endpoint filters o librerie dedicate.
```

# Endpoint filters

Gli endpoint filter permettono di eseguire logica prima o dopo l’endpoint handler, ispezionare parametri e intercettare la risposta. Sono utili per validazione, logging, controlli trasversali e trasformazioni comuni. ([Microsoft Learn][5])

Esempio concettuale:

```csharp
public class LoggingEndpointFilter : IEndpointFilter
{
    private readonly ILogger<LoggingEndpointFilter> logger;

    public LoggingEndpointFilter(
        ILogger<LoggingEndpointFilter> logger)
    {
        this.logger = logger;
    }

    public async ValueTask<object?> InvokeAsync(
        EndpointFilterInvocationContext context,
        EndpointFilterDelegate next)
    {
        logger.LogInformation(
            "Esecuzione endpoint {Endpoint}",
            context.HttpContext.GetEndpoint()?.DisplayName);

        object? result = await next(context);

        logger.LogInformation("Endpoint completato");

        return result;
    }
}
```

Applicazione a un gruppo:

```csharp
group.AddEndpointFilter<LoggingEndpointFilter>();
```

Per il progetto finale possiamo usarli con parsimonia.

Regola:

```text
Non mettere business logic negli endpoint filter.
Usali per logica trasversale.
```

# ResultExtensions

`App.Api/Extensions/ResultExtensions.cs`:

```csharp
using App.Application.Common;

namespace App.Api.Extensions;

public static class ResultExtensions
{
    public static IResult ToHttpResult(this Result result)
    {
        if (result.Success)
        {
            return Results.NoContent();
        }

        return result.ErrorType switch
        {
            ErrorType.NotFound =>
                Results.Problem(
                    title: "Risorsa non trovata",
                    detail: result.Error,
                    statusCode: StatusCodes.Status404NotFound),

            ErrorType.Conflict =>
                Results.Problem(
                    title: "Conflitto",
                    detail: result.Error,
                    statusCode: StatusCodes.Status409Conflict),

            ErrorType.Validation =>
                Results.ValidationProblem(
                    new Dictionary<string, string[]>
                    {
                        ["Error"] = [result.Error ?? "Errore di validazione."]
                    }),

            ErrorType.Business =>
                Results.Problem(
                    title: "Errore di business",
                    detail: result.Error,
                    statusCode: StatusCodes.Status400BadRequest),

            _ =>
                Results.Problem(
                    title: "Errore",
                    detail: result.Error,
                    statusCode: StatusCodes.Status400BadRequest)
        };
    }

    public static IResult ToHttpResult<T>(this Result<T> result)
    {
        if (result.Success)
        {
            return Results.Ok(result.Value);
        }

        return ((Result)result).ToHttpResult();
    }
}
```

Questo rende gli endpoint più puliti.

# Configurazione JWT Bearer

Pacchetto, se necessario:

```bash
dotnet add src/App.Api/App.Api.csproj package Microsoft.AspNetCore.Authentication.JwtBearer
```

`Program.cs`:

```csharp
using Microsoft.AspNetCore.Authentication.JwtBearer;

builder.Services
    .AddAuthentication(JwtBearerDefaults.AuthenticationScheme)
    .AddJwtBearer(options =>
    {
        options.Authority = builder.Configuration["Authentication:Authority"];
        options.Audience = builder.Configuration["Authentication:Audience"];
    });

builder.Services.AddAuthorization(options =>
{
    options.AddPolicy("Catalogo.Write", policy =>
    {
        policy.RequireAuthenticatedUser();
        policy.RequireClaim("scope", "catalogo.write");
    });

    options.AddPolicy("Documenti.Write", policy =>
    {
        policy.RequireAuthenticatedUser();
        policy.RequireClaim("scope", "documenti.write");
    });
});
```

Pipeline:

```csharp
app.UseAuthentication();
app.UseAuthorization();
```

Ordine:

```csharp
app.UseCors("Web");

app.UseAuthentication();
app.UseAuthorization();

app.MapCatalogoEndpoints();
```

# Autenticazione nel progetto finale: scelta consigliata

Per non appesantire troppo il caso pratico, suggerisco questa progressione:

```text
Versione 1:
  API senza autenticazione,
  per concentrarsi su dominio, EF Core, Blazor, test.

Versione 2:
  aggiunta JWT Bearer,
  policy su scrittura catalogo e documenti.

Versione 3:
  integrazione con identity provider esterno.
```

Nel capitolo 13.06 possiamo quindi includere la sezione autenticazione come preparazione, ma non bloccare il progetto finale su un identity provider reale.

# Factory test

`App.Tests/Integration/CustomWebApplicationFactory.cs`:

```csharp
using App.Domain.Catalogo;
using App.Domain.Clienti;
using App.Infrastructure.Data;
using Microsoft.AspNetCore.Hosting;
using Microsoft.AspNetCore.Mvc.Testing;
using Microsoft.Data.Sqlite;
using Microsoft.EntityFrameworkCore;
using Microsoft.Extensions.DependencyInjection;

namespace App.Tests.Integration;

public class CustomWebApplicationFactory
    : WebApplicationFactory<Program>
{
    private SqliteConnection? connection;

    protected override void ConfigureWebHost(IWebHostBuilder builder)
    {
        builder.ConfigureServices(services =>
        {
            ServiceDescriptor? descriptor = services
                .SingleOrDefault(d =>
                    d.ServiceType == typeof(DbContextOptions<AppDbContext>));

            if (descriptor is not null)
            {
                services.Remove(descriptor);
            }

            connection = new SqliteConnection("DataSource=:memory:");
            connection.Open();

            services.AddDbContext<AppDbContext>(options =>
            {
                options.UseSqlite(connection);
            });

            ServiceProvider provider = services.BuildServiceProvider();

            using IServiceScope scope = provider.CreateScope();

            AppDbContext dbContext =
                scope.ServiceProvider.GetRequiredService<AppDbContext>();

            dbContext.Database.EnsureCreated();

            Seed(dbContext);
        });
    }

    private static void Seed(AppDbContext dbContext)
    {
        if (dbContext.Clienti.Any())
        {
            return;
        }

        Cliente cliente = new(
            ragioneSociale: "Cliente Test",
            email: "cliente@example.com",
            telefono: null,
            partitaIva: null,
            codiceFiscale: null);

        Libro libro = new(
            titolo: "Libro Test",
            isbn: "TEST-ISBN",
            prezzo: 20m,
            dataPubblicazione: null,
            collanaId: null);

        dbContext.Clienti.Add(cliente);
        dbContext.Libri.Add(libro);

        dbContext.SaveChanges();
    }

    protected override void Dispose(bool disposing)
    {
        base.Dispose(disposing);

        if (disposing)
        {
            connection?.Dispose();
        }
    }
}
```

# Endpoint CRUD: Clienti

Esempio sintetico:

```csharp
using App.Api.Extensions;
using App.Application.Clienti;
using App.Application.Common;

namespace App.Api.Endpoints;

public static class ClientiEndpoints
{
    public static IEndpointRouteBuilder MapClientiEndpoints(
        this IEndpointRouteBuilder app)
    {
        RouteGroupBuilder group = app
            .MapGroup("/api/clienti")
            .WithTags("Clienti");

        group.MapGet("/", GetClientiAsync)
            .WithName("GetClienti");

        group.MapGet("/{id:int}", GetClienteAsync)
            .WithName("GetClienteById");

        group.MapPost("/", CreateClienteAsync)
            .WithName("CreateCliente");

        group.MapPut("/{id:int}", UpdateClienteAsync)
            .WithName("UpdateCliente");

        return app;
    }

    private static async Task<IResult> GetClientiAsync(
        IClienteService clienteService,
        CancellationToken cancellationToken)
    {
        IReadOnlyList<ClienteListItemDto> clienti =
            await clienteService.GetClientiAsync(cancellationToken);

        return Results.Ok(clienti);
    }

    private static async Task<IResult> GetClienteAsync(
        int id,
        IClienteService clienteService,
        CancellationToken cancellationToken)
    {
        Result<ClienteDetailDto> result =
            await clienteService.GetClienteAsync(id, cancellationToken);

        return result.ToHttpResult();
    }

    private static async Task<IResult> CreateClienteAsync(
        CreateClienteRequest request,
        IClienteService clienteService,
        CancellationToken cancellationToken)
    {
        Result<int> result =
            await clienteService.CreateClienteAsync(
                request,
                cancellationToken);

        if (!result.Success)
        {
            return result.ToHttpResult();
        }

        return Results.Created(
            $"/api/clienti/{result.Value}",
            new { id = result.Value });
    }

    private static async Task<IResult> UpdateClienteAsync(
        int id,
        UpdateClienteRequest request,
        IClienteService clienteService,
        CancellationToken cancellationToken)
    {
        Result result =
            await clienteService.UpdateClienteAsync(
                id,
                request,
                cancellationToken);

        return result.ToHttpResult();
    }
}
```

## Endpoint troppo grandi

```text
MapPost contiene 80 righe di logica.
```

Meglio estrarre handler o service.

Rischioso e fragile.

Usa DTO.

# Riepilogo

