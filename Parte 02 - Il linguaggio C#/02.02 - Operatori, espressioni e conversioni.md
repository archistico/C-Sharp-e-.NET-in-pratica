# 02.02 — Operatori, espressioni e conversioni

Nel capitolo precedente abbiamo introdotto variabili, costanti e tipi fondamentali. Ora dobbiamo imparare a **combinare** quei valori.

Un programma non si limita a dichiarare dati. Li confronta, li trasforma, li somma, li converte, li assegna, li verifica e li usa per prendere decisioni.

Esempio:

```csharp id="ddu5ic"
decimal prezzo = 100m;
decimal sconto = 20m;

decimal totale = prezzo - sconto;

bool spedizioneGratuita = totale >= 50m;

Console.WriteLine($"Totale: {totale}");
Console.WriteLine($"Spedizione gratuita: {spedizioneGratuita}");
```

In poche righe abbiamo usato:

* operatori aritmetici;
* assegnazioni;
* confronti;
* valori booleani;
* interpolazione di stringhe.

Questo capitolo serve a capire proprio queste operazioni.

---

# Espressioni e operatori

Una **espressione** è una porzione di codice che produce un valore.

Esempio:

```csharp id="kb0454"
10 + 5
```

Questa espressione produce il valore `15`.

Anche una variabile è un’espressione:

```csharp id="v88iw9"
prezzo
```

Anche una chiamata a metodo può essere un’espressione:

```csharp id="mobzci"
DateTime.Now
```

Un **operatore** è un simbolo o una parola che applica un’operazione a uno o più valori.

Esempi:

```text id="o9ckjl"
+
-
*
/
%
==
!=
>
<
&&
||
??
?.
```

Gli elementi su cui un operatore lavora si chiamano **operandi**.

Nell’espressione:

```csharp id="quoeox"
10 + 5
```

`+` è l’operatore.
`10` e `5` sono gli operandi.

---

## Precedenza degli operatori

Gli operatori non vengono tutti valutati nello stesso ordine.

Esempio:

```csharp id="y6v4s7"
int risultato = 10 + 5 * 2;
```

Il risultato è:

```text id="i03jpa"
20
```

Non:

```text id="d0eai6"
30
```

Perché la moltiplicazione ha precedenza sull’addizione.

L’espressione viene interpretata così:

```csharp id="ifgjfq"
int risultato = 10 + (5 * 2);
```

Possiamo usare le parentesi per rendere esplicito l’ordine:

```csharp id="mvkda8"
int risultato = (10 + 5) * 2;
```

In questo caso il risultato è:

```text id="ba81gd"
30
```

La documentazione Microsoft ricorda che le parentesi permettono di modificare l’ordine imposto dalla precedenza e dall’associatività degli operatori. ([Microsoft Learn][1])

# Operatori aritmetici

Gli operatori aritmetici servono per eseguire calcoli numerici.

I principali sono:

```text id="fls8wg"
+   addizione
-   sottrazione
*   moltiplicazione
/   divisione
%   resto della divisione
```

Esempio:

```csharp id="ojswb7"
int a = 10;
int b = 3;

Console.WriteLine(a + b);
Console.WriteLine(a - b);
Console.WriteLine(a * b);
Console.WriteLine(a / b);
Console.WriteLine(a % b);
```

Output:

```text id="dlk7vg"
13
7
30
3
1
```

```csharp id="tx2e1a"
int somma = 10 + 5;
```

## Divisione

La divisione richiede un po’ di attenzione.

```csharp id="csuuui"
int risultato = 10 / 3;

Console.WriteLine(risultato);
```

Output:

```text id="h0ztqw"
3
```

Il risultato non è `3.333...` perché stiamo dividendo due interi. In C#, la divisione tra interi produce un intero: la parte decimale viene eliminata.

Se vogliamo un risultato decimale, dobbiamo usare un tipo decimale o in virgola mobile:

```csharp id="n7e4yt"
double risultato = 10.0 / 3.0;

Console.WriteLine(risultato);
```

Oppure:

```csharp id="jli5qu"
decimal risultato = 10m / 3m;

Console.WriteLine(risultato);
```

---

## Resto della divisione

L’operatore `%` restituisce il resto della divisione.

```csharp id="c0mtyi"
int resto = 10 % 3;

Console.WriteLine(resto);
```

Output:

```text id="xfkyxg"
1
```

Perché:

```text id="l66ecr"
10 / 3 = 3 con resto 1
```

L’operatore `%` è utile, per esempio, per verificare se un numero è pari:

```csharp id="7rdnp9"
int numero = 8;

bool pari = numero % 2 == 0;

Console.WriteLine(pari);
```

Output:

```text id="p6wjhm"
True
```

---

## Incremento e decremento

C# offre anche operatori per aumentare o diminuire di uno una variabile numerica.

```csharp id="88j1pw"
int contatore = 0;

contatore++;

Console.WriteLine(contatore);
```

Output:

```text id="yw4da7"
1
```

`contatore++` equivale a:

```csharp id="c5g0mk"
contatore = contatore + 1;
```

Per diminuire:

```csharp id="r6jv2a"
contatore--;
```

equivale a:

```csharp id="f48dkw"
contatore = contatore - 1;
```

---

## Prefisso e postfisso

Esistono due forme:

```csharp id="1gm437"
++x
x++
```

La differenza si nota quando l’operatore è usato dentro un’espressione.

```csharp id="vefblp"
int x = 5;
int y = x++;

Console.WriteLine(x);
Console.WriteLine(y);
```

Output:

```text id="v254aj"
6
5
```

Con `x++`, prima viene usato il valore attuale, poi viene incrementato.

Con `++x`:

```csharp id="sr1yc2"
int x = 5;
int y = ++x;

Console.WriteLine(x);
Console.WriteLine(y);
```

Output:

```text id="ej8fbg"
6
6
```

Con `++x`, prima viene incrementato, poi viene usato.

Nei casi reali è meglio evitare codice troppo furbo. Spesso è più leggibile scrivere l’incremento su una riga separata.

# Operatori di confronto

Gli operatori di confronto confrontano due valori e producono un risultato booleano: `true` oppure `false`.

I principali sono:

```text id="aaw45y"
==   uguale
!=   diverso
>    maggiore
<    minore
>=   maggiore o uguale
<=   minore o uguale
```

Esempio:

```csharp id="5qjzrv"
int eta = 20;

Console.WriteLine(eta == 18);
Console.WriteLine(eta != 18);
Console.WriteLine(eta > 18);
Console.WriteLine(eta < 18);
Console.WriteLine(eta >= 18);
Console.WriteLine(eta <= 18);
```

Output:

```text id="g2nvz8"
False
True
True
False
True
False
```

---

## Uguale e assegnazione

Attenzione a non confondere:

```text id="3glxn8"
=    assegnazione
==   confronto di uguaglianza
```

Esempio:

```csharp id="p52wi0"
int eta = 18;
```

Qui assegniamo `18` alla variabile `eta`.

Esempio:

```csharp id="uzfo86"
bool maggiorenne = eta == 18;
```

Qui confrontiamo `eta` con `18`.

---

## Confrontare stringhe

Possiamo confrontare stringhe con `==`:

```csharp id="3iey67"
string nome = "Emilie";

bool risultato = nome == "Emilie";

Console.WriteLine(risultato);
```

Output:

```text id="9p1gwx"
True
```

Tuttavia, i confronti tra stringhe possono essere più delicati quando entrano in gioco maiuscole, minuscole, culture e ordinamenti.

Esempio:

```csharp id="3dfy3s"
string a = "emilie";
string b = "Emilie";

Console.WriteLine(a == b);
```

Output:

```text id="rawk92"
False
```

Per confronti più controllati si può usare:

```csharp id="919443"
bool uguali = string.Equals(a, b, StringComparison.OrdinalIgnoreCase);
```

Questo confronta ignorando maiuscole e minuscole.

# Operatori logici

Gli operatori logici servono a combinare valori booleani.

I principali sono:

```text id="24yn5x"
&&   AND logico
||   OR logico
!    NOT logico
```

---

## AND logico `&&`

`&&` restituisce `true` solo se entrambe le condizioni sono vere.

```csharp id="4dofc9"
int eta = 25;
bool documentoValido = true;

bool accessoConsentito = eta >= 18 && documentoValido;

Console.WriteLine(accessoConsentito);
```

Output:

```text id="w0pkvo"
True
```

Se una delle due condizioni è falsa, il risultato è falso.

```csharp id="kcyubd"
int eta = 16;
bool documentoValido = true;

bool accessoConsentito = eta >= 18 && documentoValido;

Console.WriteLine(accessoConsentito);
```

Output:

```text id="16o95a"
False
```

---

## OR logico `||`

`||` restituisce `true` se almeno una condizione è vera.

```csharp id="r72hse"
bool clienteVip = false;
bool haInvito = true;

bool ingressoPrioritario = clienteVip || haInvito;

Console.WriteLine(ingressoPrioritario);
```

Output:

```text id="pfx3bn"
True
```

---

## NOT logico `!`

`!` inverte un valore booleano.

```csharp id="rnr893"
bool bloccato = false;

if (!bloccato)
{
    Console.WriteLine("Utente attivo");
}
```

`!bloccato` significa:

> non bloccato.

---

## Short-circuit evaluation

Gli operatori `&&` e `||` valutano le condizioni in modo intelligente.

Con `&&`, se la prima condizione è falsa, la seconda non viene valutata.

```csharp id="lb3h02"
Cliente? cliente = null;

if (cliente != null && cliente.Attivo)
{
    Console.WriteLine("Cliente attivo");
}
```

Se `cliente` è `null`, la seconda parte `cliente.Attivo` non viene eseguita. Questo evita un errore.

Con `||`, se la prima condizione è vera, la seconda non viene valutata.

La documentazione Microsoft include `&&` e `||` tra gli operatori che valutano operandi in modo condizionale, cioè non sempre valutano tutti gli operandi. ([Microsoft Learn][1])

# Operatori di assegnazione

L’operatore di assegnazione principale è:

```text id="0p5cwr"
=
```

```csharp id="ekktug"
int numero = 10;
```

> assegna il valore `10` alla variabile `numero`.

## Assegnazioni composte

C# offre operatori di assegnazione composti:

```text id="xlxd9h"
+=
-=
*=
/=
%=
```

Esempio:

```csharp id="7uumrj"
int totale = 10;

totale += 5;

Console.WriteLine(totale);
```

Output:

```text id="9hq2y2"
15
```

`totale += 5` equivale a:

```csharp id="ko3tyk"

```

Altri esempi:


---

## Assegnazioni concatenate

Gli operatori di assegnazione sono associativi a destra. Questo significa che:

```csharp id="nx8mdu"
int a;
int b;
int c;

a = b = c = 10;
```

viene interpretato come:

```csharp id="h2kcjh"
a = (b = (c = 10));
```

Alla fine tutte e tre le variabili valgono `10`.

La documentazione C# indica che gli operatori di assegnazione sono right-associative, cioè vengono raggruppati da destra verso sinistra. ([Microsoft Learn][2])

# Operatore ternario `?:`

L’operatore ternario permette di scegliere tra due valori in base a una condizione.

Sintassi:

```csharp id="7wq1ss"
condizione ? valoreSeVero : valoreSeFalso
```

Esempio:

```csharp id="7qbkn5"
int eta = 20;

string risultato = eta >= 18 ? "Maggiorenne" : "Minorenne";

Console.WriteLine(risultato);
```

Output:

```text id="rs8wl7"
Maggiorenne
```

Equivale a:

```csharp id="q5mws9"
string risultato;

if (eta >= 18)
{
    risultato = "Maggiorenne";
}
else
{
    risultato = "Minorenne";
}
```

L’operatore ternario è utile per assegnazioni semplici.

---

## Quando usarlo

Uso buono:

```csharp id="q4gcrq"
string stato = attivo ? "Attivo" : "Non attivo";
```

Uso meno leggibile:

```csharp id="j2vxq4"
string risultato = eta >= 18 ? clienteVip ? "VIP maggiorenne" : "Maggiorenne" : "Minorenne";
```

In casi complessi è meglio usare `if`.

Regola pratica:

```text id="a8yejh"
Condizione semplice
  operatore ternario.

Logica articolata
  if/else.
```

# Null-coalescing operator `??`

L’operatore `??` permette di fornire un valore alternativo quando un’espressione è `null`.

Esempio:

```csharp id="yxb2n5"
string? nome = null;

string nomeDaStampare = nome ?? "Nome non disponibile";

Console.WriteLine(nomeDaStampare);
```

Output:

```text id="d4jdtm"
Nome non disponibile
```

Significa:

> se `nome` non è null, usa `nome`; altrimenti usa `"Nome non disponibile"`.

---

## Esempio con input da console

```csharp id="24d8wd"
Console.Write("Nome: ");

string? nome = Console.ReadLine();

string nomeFinale = nome ?? "";

Console.WriteLine($"Nome inserito: {nomeFinale}");
```

Attenzione: se l’utente preme Invio senza scrivere nulla, `Console.ReadLine()` può restituire una stringa vuota, non necessariamente `null`.

Quindi spesso è meglio usare:

```csharp id="fn0nvh"
string nomeFinale = string.IsNullOrWhiteSpace(nome)
    ? "Nome non disponibile"
    : nome;
```

---

## Null-coalescing assignment `??=`

Esiste anche l’operatore:

```text id="cq79nl"
??=
```

Serve ad assegnare un valore solo se la variabile è `null`.

Esempio:

```csharp id="x1woib"
string? nome = null;

nome ??= "Nome predefinito";

Console.WriteLine(nome);
```

Output:

```text id="xs0km8"
Nome predefinito
```

Se invece `nome` ha già un valore:

```csharp id="v959e8"
string? nome = "Emilie";

nome ??= "Nome predefinito";

Console.WriteLine(nome);
```

Output:

```text id="bgl75y"
Emilie
```

La documentazione Microsoft include `??` e `??=` tra gli operatori null-coalescing, valutati in modo condizionale. ([Microsoft Learn][1])

---

# Null-conditional operator `?.`

L’operatore `?.` permette di accedere a un membro solo se l’oggetto non è `null`.

Esempio:

```csharp id="9zkhth"
Cliente? cliente = null;

string? nome = cliente?.Nome;
```

Se `cliente` è `null`, l’espressione restituisce `null` invece di generare un errore.

Senza `?.`, questo codice genererebbe una `NullReferenceException`:

```csharp id="xfegpe"
string nome = cliente.Nome; // errore se cliente è null
```

La documentazione Microsoft spiega che `?.` e `?[]` applicano l’accesso a membro o elemento solo se l’operando non è null; altrimenti restituiscono `null`. ([Microsoft Learn][3])

---

## Catene null-conditional

Possiamo concatenare più accessi:

```csharp id="3omoyb"
string? città = cliente?.Indirizzo?.Citta;
```

Significa:

> se `cliente` è null, restituisci null;
> se `cliente.Indirizzo` è null, restituisci null;
> altrimenti restituisci `cliente.Indirizzo.Citta`.

Gli operatori null-conditional sono short-circuiting: se un passaggio della catena restituisce `null`, il resto della catena non viene valutato. ([Microsoft Learn][3])

---

## Uso con `??`

Molto spesso `?.` e `??` si usano insieme.

```csharp id="6nv0ui"
string città = cliente?.Indirizzo?.Citta ?? "Città non disponibile";
```

Questa riga significa:

> prova a leggere la città; se qualcosa nella catena è null, usa il testo predefinito.

---

# Null-conditional assignment

Nelle versioni recenti del linguaggio C# è stato introdotto anche il supporto all’assegnazione tramite accesso null-conditional, spesso indicato come **null-conditional assignment**.

L’idea è permettere istruzioni del tipo:

```csharp id="rd46cu"
cliente?.Nome = "Emilie";
```

Il significato è:

> se `cliente` non è null, assegna `"Emilie"` a `Nome`;
> se `cliente` è null, non fare nulla.

Questa funzionalità è associata a C# 14. Conviene quindi trattarla come caratteristica moderna e verificare che il progetto usi una versione del linguaggio che la supporta.

In un libro didattico io la presenterei come nota avanzata, non come strumento indispensabile per i primi capitoli.

---

## Prima della null-conditional assignment

Senza questa funzionalità, si scrive:

```csharp id="8f97e1"
if (cliente is not null)
{
    cliente.Nome = "Emilie";
}
```

Questo codice è più lungo, ma molto chiaro.

---

## Con null-conditional assignment

Con la nuova sintassi:

```csharp id="xxr4id"
cliente?.Nome = "Emilie";
```

Il codice diventa più compatto.

Tuttavia, attenzione: se il fatto che `cliente` sia `null` rappresenta un errore logico, non conviene nasconderlo.

Esempio rischioso:

```csharp id="nmlid5"
ordine?.Totale = CalcolaTotale();
```

Se `ordine` non dovrebbe mai essere `null`, questa riga può nascondere un bug. In quel caso è meglio far emergere il problema.

Regola pratica:

```text id="4v7izf"
Usa ?. quando null è una possibilità prevista.

Evita ?. quando null indica un errore di programmazione.
```

# Casting

Il **casting** è una conversione esplicita da un tipo a un altro.

Esempio:

```csharp id="k0lfi6"
double valore = 10.8;

int intero = (int)valore;

Console.WriteLine(intero);
```

Output:

```text id="2kkc1j"
10
```

La parte decimale viene eliminata.

---

## Cast numerici

Alcune conversioni numeriche sono implicite:

```csharp id="f641d3"
int piccolo = 100;
long grande = piccolo;
```

Altre richiedono cast:

```csharp id="86l2j8"
long grande = 100;
int piccolo = (int)grande;
```

Perché un `long` può contenere valori troppo grandi per un `int`.

---

## Cast tra tipi riferimento

Il casting è usato anche con classi e gerarchie di ereditarietà.

Esempio:

```csharp id="bqw96v"
object valore = "Ciao";

string testo = (string)valore;

Console.WriteLine(testo);
```

Questo funziona perché `valore` contiene effettivamente una stringa.

Ma se sbagliamo:

```csharp id="6asgcv"
object valore = 123;

string testo = (string)valore; // errore a runtime
```

otteniamo un errore a runtime.

Per evitare questo tipo di problema useremo `is` e `as`.


## Convert e null

`Convert` ha alcuni comportamenti specifici con `null`.

Esempio:

```csharp id="47qi0v"
string? testo = null;

int numero = Convert.ToInt32(testo);

Console.WriteLine(numero);
```

Output:

```text id="jh8gkk"
0
```

Questo può sorprendere.

Con `int.Parse`, invece, `null` genera errore.

Per questo motivo è importante conoscere bene lo strumento che si usa.

---

# `Parse`

Molti tipi offrono un metodo `Parse`.

Esempio:

```csharp id="gfy40d"
string testo = "123";

int numero = int.Parse(testo);

Console.WriteLine(numero);
```

Output:

```text id="6byf2w"
123
```

Altri esempi:

```csharp id="61opuh"
decimal prezzo = decimal.Parse("19.90");
DateTime data = DateTime.Parse("2026-04-24");
bool attivo = bool.Parse("true");
```

---

## Il problema di Parse

`Parse` funziona solo se il testo è valido.

```csharp id="pz0s4h"
int numero = int.Parse("ciao");
```

Questo codice genera un’eccezione.

Per input controllato dal programmatore può andare bene.
Per input dell’utente è meglio usare `TryParse`.

---

# `TryParse`

`TryParse` prova a convertire un valore testuale senza generare eccezione in caso di fallimento.

Esempio:

```csharp id="zdztui"
string testo = "123";

bool riuscito = int.TryParse(testo, out int numero);

Console.WriteLine(riuscito);
Console.WriteLine(numero);
```

Output:

```text id="nrtqs3"
True
123
```

Se il testo non è valido:

```csharp id="xwhhmd"
string testo = "ciao";

bool riuscito = int.TryParse(testo, out int numero);

Console.WriteLine(riuscito);
Console.WriteLine(numero);
```

Output:

```text id="72kgwq"
False
0
```

---

## TryParse con input utente

```csharp id="56imzp"
Console.Write("Inserisci la tua età: ");

string? input = Console.ReadLine();

if (int.TryParse(input, out int eta))
{
    Console.WriteLine($"Età inserita: {eta}");
}
else
{
    Console.WriteLine("Età non valida.");
}
```

Questo è il modo corretto per convertire input non affidabile.

---

## Parse o TryParse?

Regola pratica:

```text id="q83sod"
Parse
  quando il valore deve essere valido e un errore è davvero eccezionale.

TryParse
  quando il valore può essere non valido, soprattutto se arriva dall’utente.
```

Esempio:

```csharp id="06k09q"
int porta = int.Parse("8080");
```

Qui può andare bene, perché la stringa è scritta dal programmatore.

Esempio:

```csharp id="go9tih"
int.TryParse(Console.ReadLine(), out int eta);
```

Qui `TryParse` è preferibile.

# Boxing e unboxing

C# distingue tra:

* value types;
* reference types.

Tipi come `int`, `decimal`, `bool`, `DateTime` sono value types.

Il tipo `object` è il tipo base da cui derivano tutti i tipi C#.

Quando un value type viene trattato come `object`, avviene il **boxing**.

---

## Boxing

Esempio:

```csharp id="xwd33l"
int numero = 42;

object oggetto = numero;
```

Qui il valore `int` viene “inscatolato” dentro un oggetto.

Questo processo si chiama boxing.

---

## Unboxing

Per recuperare il valore originale serve un cast.

```csharp id="3cpjfj"
object oggetto = 42;

int numero = (int)oggetto;

Console.WriteLine(numero);
```

Questo è unboxing.

---

## Errore di unboxing

L’unboxing deve essere fatto verso il tipo corretto.

```csharp id="o3zjyg"
object oggetto = 42;

long numero = (long)oggetto; // errore a runtime
```

Anche se `int` può essere convertito in `long`, l’oggetto contiene un `int` boxed. Prima deve essere estratto come `int`.

Corretto:

```csharp id="zvfp9v"
object oggetto = 42;

int valoreIntero = (int)oggetto;
long valoreLong = valoreIntero;
```

---

## Perché è importante

Boxing e unboxing possono avere costi di performance e possono introdurre errori a runtime.

Oggi si usano meno direttamente grazie ai generics.

Per esempio, meglio:

```csharp id="1srsu6"
List<int> numeri = new List<int>();
```

che:

```csharp id="hpcxzo"
ArrayList numeri = new ArrayList();
```

Le collezioni generiche evitano boxing inutile e sono più sicure.

---

# Operatore `is`

L’operatore `is` verifica se un valore è compatibile con un certo tipo.

Esempio:

```csharp id="0ztdwa"
object valore = "Ciao";

if (valore is string)
{
    Console.WriteLine("È una stringa");
}
```

Molto spesso si usa con pattern matching:

```csharp id="eed2px"
object valore = "Ciao";

if (valore is string testo)
{
    Console.WriteLine(testo.ToUpper());
}
```

Qui succedono due cose:

1. verifichiamo che `valore` sia una stringa;
2. se lo è, creiamo la variabile `testo` già tipizzata come `string`.

---

## `is not null`

Un uso molto comune è:

```csharp id="lyj9f9"
if (cliente is not null)
{
    Console.WriteLine(cliente.Nome);
}
```

È una forma leggibile per controllare che un riferimento non sia `null`.

---

# Operatore `as`

L’operatore `as` prova a convertire un valore in un tipo riferimento.

Se la conversione riesce, restituisce il valore convertito.
Se fallisce, restituisce `null`.

Esempio:

```csharp id="ymie2i"
object valore = "Ciao";

string? testo = valore as string;

if (testo is not null)
{
    Console.WriteLine(testo.ToUpper());
}
```

Se invece:

```csharp id="wkcjc9"
object valore = 123;

string? testo = valore as string;
```

`testo` sarà `null`.

---

## `is` o `as`?

Oggi, spesso si preferisce `is` con pattern matching:

```csharp id="136o59"
if (valore is string testo)
{
    Console.WriteLine(testo.ToUpper());
}
```

Rispetto a:

```csharp id="cwt0yv"
string? testo = valore as string;

if (testo is not null)
{
    Console.WriteLine(testo.ToUpper());
}
```

Entrambe le forme sono valide.

Regola pratica:

```text id="b9ow30"
Vuoi controllare e usare subito il valore convertito?
  is con pattern matching.

Vuoi tentare una conversione e gestire null dopo?
  as.
```

---

# `nameof`

L’operatore `nameof` restituisce il nome di una variabile, tipo o membro sotto forma di stringa.

Esempio:

```csharp id="e6v69a"
string nome = nameof(Cliente);

Console.WriteLine(nome);
```

Output:

```text id="jbez9j"
Cliente
```

Esempio con una variabile:

```csharp id="aqopgr"
int eta = 42;

Console.WriteLine(nameof(eta));
```

Output:

```text id="dn80e3"
eta
```

---

## Perché usare nameof

`nameof` è molto utile perché evita stringhe scritte a mano.

Esempio:

```csharp id="5it9lw"
void ImpostaNome(string nome)
{
    if (string.IsNullOrWhiteSpace(nome))
    {
        throw new ArgumentException("Il nome è obbligatorio", nameof(nome));
    }
}
```

Se in futuro rinominiamo il parametro `nome`, gli strumenti di refactoring aggiorneranno anche `nameof(nome)`.

Se invece avessimo scritto:

```csharp id="unmhwn"
throw new ArgumentException("Il nome è obbligatorio", "nome");
```

la stringa `"nome"` potrebbe restare sbagliata dopo un refactoring.

---

# `typeof`

L’operatore `typeof` restituisce un oggetto `Type` che rappresenta un tipo.

Esempio:

```csharp id="yfs60s"
Type tipo = typeof(string);

Console.WriteLine(tipo.FullName);
```

Output:

```text id="qa0ywm"
System.String
```

`typeof` è utile in scenari come:

* reflection;
* logging;
* configurazione;
* dependency injection;
* serializzazione;
* framework e librerie.

Esempio:

```csharp id="u96m0q"
Console.WriteLine(typeof(DateTime).FullName);
Console.WriteLine(typeof(List<int>).Name);
```

---

## `nameof` e `typeof`

Non vanno confusi.

```csharp id="gqekah"
Console.WriteLine(nameof(string));
Console.WriteLine(typeof(string));
```

`nameof(string)` restituisce una stringa con il nome del tipo:

```text id="y9we5d"
String
```

`typeof(string)` restituisce un oggetto `Type` che descrive il tipo:

```text id="9njfa4"
System.String
```

Regola pratica:

```text id="8s584r"
nameof
  mi serve il nome come testo.

typeof
  mi serve la descrizione del tipo a runtime.
```


## Esempio

```csharp id="236xoh"
Console.WriteLine($"int: {sizeof(int)} byte");
```

# `checked` e `unchecked`

I tipi numerici hanno limiti.

Per esempio, `int` ha un valore massimo.

Possiamo verificarlo:

```csharp id="dp31aj"
Console.WriteLine(int.MaxValue);
```

Output:

```text id="7vaivz"
2147483647
```

Cosa succede se superiamo questo valore?

---

## Overflow

Esempio:

```csharp id="g7vg1p"
int massimo = int.MaxValue;

int risultato = massimo + 1;

Console.WriteLine(risultato);
```

In certi contesti, il valore può “girare” e diventare negativo.

Questo fenomeno si chiama **overflow**.

---

## checked

Con `checked` chiediamo a C# di generare un’eccezione in caso di overflow aritmetico.

```csharp id="efwn6t"
checked
{
    int massimo = int.MaxValue;
    int risultato = massimo + 1;

    Console.WriteLine(risultato);
}
```

Questo codice genera una `OverflowException`.

---

## unchecked

Con `unchecked` permettiamo esplicitamente l’overflow.

```csharp id="mgmgy7"
unchecked
{
    int massimo = int.MaxValue;
    int risultato = massimo + 1;

    Console.WriteLine(risultato);
}
```

Output tipico:

```text id="w6xxuv"
-2147483648
```

---

## Quando usare checked

`checked` è utile quando un overflow sarebbe un errore grave.

Esempi:

* calcoli finanziari;
* quantità;
* contatori critici;
* importi;
* valori di business;
* conversioni numeriche rischiose.

Esempio:

```csharp id="ybrcpj"
checked
{
    int prezzoUnitario = 1_000_000;
    int quantita = 3_000;

    int totale = prezzoUnitario * quantita;

    Console.WriteLine(totale);
}
```

Se il valore supera la capacità di `int`, vogliamo saperlo.

---

## Quando usare unchecked

`unchecked` è utile in scenari dove l’overflow è voluto o accettato.

Esempi:

* algoritmi hash;
* codice a basso livello;
* compatibilità con comportamenti specifici;
* ottimizzazioni particolari.

Nel normale codice applicativo, `unchecked` è raro.

---

# Errori comuni

## Confondere `=` e `==`

Sbagliato quando si vuole confrontare:

```csharp id="2bepl0"
if (eta = 18)
{
}
```

Corretto:

```csharp id="0dj7wf"
if (eta == 18)
{
}
```

`=` assegna.
`==` confronta.

---

## Usare `Parse` su input utente

Rischioso:

```csharp id="1il00k"
int eta = int.Parse(Console.ReadLine()!);
```

Meglio:

```csharp id="l43zhh"
if (int.TryParse(Console.ReadLine(), out int eta))
{
    Console.WriteLine(eta);
}
```

---

## Usare `?.` per nascondere bug

Rischioso:

```csharp id="3hfcxw"
ordine?.Conferma();
```

Se `ordine` non dovrebbe mai essere `null`, questa riga nasconde un problema.

Meglio:

```csharp id="q6nvqu"
ordine.Conferma();
```

oppure controllare esplicitamente:

```csharp id="e9w71e"
if (ordine is null)
{
    throw new InvalidOperationException("Ordine non inizializzato.");
}

ordine.Conferma();
```

---

## Usare il cast pensando che arrotondi

```csharp id="lbnsz9"
double valore = 10.9;
int intero = (int)valore;
```

`intero` vale `10`, non `11`.

Per arrotondare:

```csharp id="9j4f4s"
int intero = (int)Math.Round(valore);
```

---

## Non considerare overflow

```csharp id="lo0ksj"
int totale = prezzo * quantita;
```

Se i numeri sono grandi, il risultato potrebbe superare i limiti di `int`.

In contesti delicati:

```csharp id="rdt5y7"
checked
{
    int totale = prezzo * quantita;
}
```

---

## Usare `as` con value type non nullable

Questo non è valido:

```csharp id="v356bc"
object valore = 42;

int numero = valore as int; // errore
```

`as` funziona con tipi riferimento o nullable value types.

Possibile:

```csharp id="aawro7"
object valore = 42;

int? numero = valore as int?;
```

Ma spesso è più chiaro:

```csharp id="nkw6db"
if (valore is int numero)
{
    Console.WriteLine(numero);
}
```
