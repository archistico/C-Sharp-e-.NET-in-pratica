# 02.01 — Sintassi, variabili e tipi fondamentali

Nella prima parte abbiamo costruito il contesto: che cos’è C#, che cos’è .NET, quali strumenti servono e come si crea un primo programma.

Ora entriamo nel linguaggio.

Questa parte del libro è fondamentale perché tutto il resto dipende da qui. Prima di parlare di oggetti, classi, database, API, interfacce grafiche o architetture complesse, dobbiamo imparare a leggere e scrivere correttamente le istruzioni di base.

Un programma C# è fatto di elementi semplici:

* parole chiave;
* nomi;
* variabili;
* costanti;
* tipi;
* valori;
* espressioni;
* istruzioni;
* blocchi di codice.

In questo capitolo ci concentriamo sulla grammatica iniziale del linguaggio: come è fatto un file C#, come si scrivono i commenti, come si dichiarano variabili e costanti, quali sono i tipi fondamentali e come funzionano le conversioni più semplici.

---

## Struttura di un file C#

Un file C# ha normalmente estensione:

```text
.cs
```

Esempio:

```text
Program.cs
Cliente.cs
CalcolatorePrezzi.cs
```

Un file C# può contenere diversi elementi:

* istruzioni top-level;
* dichiarazioni `using`;
* namespace;
* classi;
* record;
* struct;
* enum;
* interfacce;
* metodi;
* commenti.

Nei primi esempi useremo spesso i **top-level statements**, cioè istruzioni scritte direttamente nel file `Program.cs`.

Esempio:

```csharp
Console.WriteLine("Ciao, C#!");
```

Questo è un programma valido.

In uno stile più classico, lo stesso programma può essere scritto così:

```csharp
using System;

namespace EsempioBase;

internal class Program
{
    static void Main(string[] args)
    {
        Console.WriteLine("Ciao, C#!");
    }
}
```

Per ora non serve comprendere ogni dettaglio. Ci basta osservare alcune cose:

```text
using System;
  importa funzionalità disponibili nella libreria .NET.

namespace EsempioBase;
  organizza il codice dentro uno spazio dei nomi.

class Program
  definisce una classe.

Main
  è il punto di ingresso del programma.

Console.WriteLine
  scrive una riga nella console.
```

Nei prossimi capitoli approfondiremo classi, metodi e namespace. In questo capitolo ci concentriamo sulle istruzioni più semplici.

---

## Istruzioni e punto e virgola

In C#, molte istruzioni terminano con il punto e virgola:

```csharp
Console.WriteLine("Prima riga");
Console.WriteLine("Seconda riga");
```

Il punto e virgola indica la fine dell’istruzione.

Se lo dimentichiamo:

```csharp
Console.WriteLine("Ciao")
```

il compilatore segnalerà un errore.

La versione corretta è:

```csharp
Console.WriteLine("Ciao");
```

Questa regola non vale per tutto. Per esempio, un blocco `if` o un blocco di classe usa le parentesi graffe:

```csharp
if (true)
{
    Console.WriteLine("Condizione vera");
}
```

Ma le singole istruzioni interne continuano normalmente a terminare con `;`.

---

## Blocchi di codice

Un blocco di codice è delimitato da parentesi graffe:

```csharp
{
    Console.WriteLine("Sono dentro un blocco");
}
```

I blocchi servono a raggruppare istruzioni.

Esempio:

```csharp
if (eta >= 18)
{
    Console.WriteLine("Maggiorenne");
    Console.WriteLine("Accesso consentito");
}
```

Le parentesi graffe dicono che entrambe le istruzioni fanno parte del ramo `if`.

In C# l’indentazione non determina il significato del codice, ma è essenziale per la leggibilità.

Questo codice funziona:

```csharp
if (eta >= 18)
{
Console.WriteLine("Maggiorenne");
Console.WriteLine("Accesso consentito");
}
```

Ma è scritto male.

Molto meglio:

```csharp
if (eta >= 18)
{
    Console.WriteLine("Maggiorenne");
    Console.WriteLine("Accesso consentito");
}
```

Un buon programmatore non scrive codice solo per il compilatore. Scrive codice anche per le persone che dovranno leggerlo.

# Commenti

I commenti sono parti del codice ignorate dal compilatore.

Servono a spiegare, annotare, chiarire o disattivare temporaneamente alcune righe durante lo sviluppo.

C# supporta tre forme principali di commento:

* commenti su una riga;
* commenti su più righe;
* commenti XML per documentazione.

---

## Commenti su una riga

Un commento su una riga inizia con `//`.

```csharp
// Questo programma stampa un messaggio
Console.WriteLine("Ciao");
```

Tutto ciò che si trova dopo `//` sulla stessa riga viene ignorato dal compilatore.

Possiamo anche scrivere un commento alla fine di una riga:

```csharp
int eta = 42; // età dell'utente
```

Questa forma è utile per brevi chiarimenti, ma non bisogna abusarne.

---

## Commenti su più righe

Un commento su più righe inizia con `/*` e termina con `*/`.

```csharp
/*
 Questo è un commento
 distribuito su più righe.
*/
Console.WriteLine("Ciao");
```

È utile per spiegazioni più lunghe.

Può anche essere usato temporaneamente per escludere un blocco di codice:

```csharp
/*
Console.WriteLine("Riga disattivata");
Console.WriteLine("Altra riga disattivata");
*/
```

Tuttavia, nei progetti reali è meglio evitare grandi blocchi di codice commentato. Se un pezzo di codice non serve più, di solito è meglio rimuoverlo: Git terrà comunque traccia della storia.

---

## Commenti XML

I commenti XML iniziano con `///`.

Sono usati per documentare classi, metodi, proprietà e altri membri.

Esempio:

```csharp
/// <summary>
/// Calcola il totale applicando lo sconto indicato.
/// </summary>
decimal CalcolaTotale(decimal prezzo, decimal sconto)
{
    return prezzo - sconto;
}
```

Questi commenti possono essere usati dagli strumenti di sviluppo per mostrare descrizioni durante l’uso del codice.

Li vedremo meglio quando parleremo di metodi, classi e librerie.

---

## Commenti utili e commenti inutili

Un commento utile spiega il **perché**.

```csharp
// Usiamo decimal per evitare errori di arrotondamento sugli importi monetari.
decimal prezzo = 19.90m;
```

Un commento poco utile ripete il **cosa**.

```csharp
// Assegna 19.90 alla variabile prezzo
decimal prezzo = 19.90m;
```

Il codice lo dice già.

Regola pratica:

```text
Commenta le decisioni.
Non commentare l'ovvio.
```

# Keyword

Le **keyword** sono parole riservate del linguaggio C#.

Hanno un significato speciale e non possono essere usate liberamente come nomi di variabili, classi o metodi.

Esempi di keyword:

```csharp
int
string
bool
class
namespace
using
public
private
if
else
for
while
return
static
void
new
true
false
null
```

Per esempio, questo codice non è valido:

```csharp
int class = 10;
```

`class` è una parola chiave, quindi non può essere usata come nome di variabile.

Versione corretta:

```csharp
int numeroClassi = 10;
```

---

## Keyword contestuali

C# ha anche parole che sono speciali solo in certi contesti.

Per esempio:

```csharp
var nome = "Emilie";
```

`var` viene usato per l’inferenza del tipo, ma il suo comportamento dipende dal contesto.

In generale, quando si scrive codice C#, è meglio evitare nomi che possano creare ambiguità con parole del linguaggio.

---

# Identificatori

Un **identificatore** è un nome scelto dal programmatore.

Sono identificatori:

* nomi di variabili;
* nomi di classi;
* nomi di metodi;
* nomi di namespace;
* nomi di proprietà;
* nomi di parametri.

Esempio:

```csharp
int eta = 42;
string nomeCliente = "Mario Rossi";
decimal totaleDocumento = 125.50m;
```

Qui gli identificatori sono:

```text
eta
nomeCliente
totaleDocumento
```

---

## Regole sintattiche

Un identificatore C#:

* può contenere lettere;
* può contenere numeri;
* può contenere `_`;
* non può iniziare con un numero;
* non può essere una keyword riservata;
* è case-sensitive.

Esempio valido:

```csharp
int numeroClienti = 100;
```

Esempio non valido:

```csharp
int 2clienti = 100;
```

Esempio valido ma sconsigliato:

```csharp
int _x = 100;
```

Può essere corretto in alcuni contesti, ma per una variabile normale è poco descrittivo.

---

## C# è case-sensitive

C# distingue maiuscole e minuscole.

Queste sono tre variabili diverse:

```csharp
int numero = 1;
int Numero = 2;
int NUMERO = 3;
```

Il codice è valido, ma pessimo.

Meglio evitare nomi che differiscono solo per maiuscole/minuscole.

---

## Convenzioni di denominazione

In C# esistono convenzioni molto diffuse.

Per variabili locali si usa normalmente il **camelCase**:

```csharp
string nomeCliente = "Mario";
int numeroOrdini = 5;
decimal totaleDocumento = 120.50m;
```

Per classi, metodi e proprietà si usa normalmente il **PascalCase**:

```csharp
class Cliente
{
    public string NomeCompleto { get; set; } = "";

    public void StampaDettaglio()
    {
        Console.WriteLine(NomeCompleto);
    }
}
```

Confronto:

```text
camelCase
  nomeCliente
  totaleDocumento
  dataNascita

PascalCase
  Cliente
  NomeCompleto
  CalcolaTotale
```

Nomi chiari rendono il codice molto più leggibile.

Meglio:

```csharp
decimal totaleDocumento = 150.00m;
```

Peggio:

```csharp
decimal td = 150.00m;
```

Le abbreviazioni fanno risparmiare pochi caratteri ma spesso fanno perdere molta chiarezza.

---

# Variabili

Una **variabile** è un nome associato a un valore.

Il valore può cambiare durante l’esecuzione del programma.

Esempio:

```csharp
int eta = 42;
```

Questa istruzione dichiara una variabile chiamata `eta`, di tipo `int`, con valore iniziale `42`.

Possiamo poi modificarla:

```csharp
eta = 43;
```

Esempio completo:

```csharp
int contatore = 0;

Console.WriteLine(contatore);

contatore = 1;

Console.WriteLine(contatore);
```

Output:

```text
0
1
```

---

## Dichiarazione e assegnazione

Questa istruzione fa due cose insieme:

```csharp
int eta = 42;
```

1. dichiara la variabile;
2. assegna un valore iniziale.

Possiamo anche separare i due passaggi:

```csharp
int eta;
eta = 42;
```

Tuttavia, in molti casi è preferibile inizializzare subito la variabile.

---

## Tipo, nome e valore

Una dichiarazione di variabile contiene tre elementi principali:

```csharp
int eta = 42;
```

```text
int
  tipo della variabile.

eta
  nome della variabile.

42
  valore assegnato.
```

Il tipo stabilisce quali valori la variabile può contenere e quali operazioni possiamo fare.

Una variabile `int` può contenere numeri interi:

```csharp
int numero = 10;
```

Non può contenere testo:

```csharp
int numero = "ciao"; // errore
```

Questo è un aspetto fondamentale di C#: è un linguaggio **fortemente tipizzato**.

---

## C# è fortemente tipizzato

In C#, ogni variabile ha un tipo.

Il compilatore controlla che i valori assegnati siano compatibili con quel tipo.

Esempio valido:

```csharp
string nome = "Emilie";
```

Esempio non valido:

```csharp
string nome = 123;
```

Il valore `123` è un numero intero, non una stringa.

Questa rigidità può sembrare fastidiosa all’inizio, ma è uno dei motivi per cui C# aiuta a trovare molti errori prima dell’esecuzione del programma.

---

## Modificare il valore di una variabile

Una variabile può cambiare valore:

```csharp
int punteggio = 10;

punteggio = 20;

Console.WriteLine(punteggio);
```

Output:

```text
20
```

Il secondo valore sostituisce il primo.

---

## Assegnazione non significa uguaglianza matematica

In C#, l’operatore `=` significa assegnazione.

```csharp
int x = 10;
```

Significa:

> assegna il valore 10 alla variabile x.

Non significa “x è uguale a 10” nel senso matematico.

Questo permette istruzioni come:

```csharp
int x = 10;
x = x + 1;
```

Dal punto di vista matematico `x = x + 1` sarebbe assurdo.
Nel codice significa:

> prendi il valore attuale di x, aggiungi 1, salva il risultato di nuovo in x.

Dopo questa istruzione, `x` vale `11`.

---

# Costanti

Una **costante** è simile a una variabile, ma il suo valore non può essere modificato dopo la dichiarazione.

Si dichiara con la keyword `const`.

Esempio:

```csharp
const decimal IvaOrdinaria = 22m;
```

Se proviamo a modificarla:

```csharp
IvaOrdinaria = 10m; // errore
```

il compilatore segnalerà un errore.

---

## Quando usare const

`const` è adatto per valori noti a tempo di compilazione e che non devono cambiare.

Esempi:

```csharp
const int GiorniSettimana = 7;
const int MesiAnno = 12;
const decimal PercentualeIva = 22m;
```

Una costante deve essere inizializzata subito:

```csharp
const int NumeroMassimo = 100;
```

Non possiamo scrivere:

```csharp
const int NumeroMassimo;
NumeroMassimo = 100;
```

---

## const e leggibilità

Le costanti rendono il codice più leggibile.

Peggio:

```csharp
decimal totale = imponibile + (imponibile * 22 / 100);
```

Meglio:

```csharp
const decimal AliquotaIva = 22m;

decimal totale = imponibile + (imponibile * AliquotaIva / 100);
```

Il secondo codice spiega meglio il significato del numero `22`.

I numeri scritti direttamente nel codice senza spiegazione vengono spesso chiamati **magic numbers**.

---

# `var`

In C# possiamo dichiarare una variabile usando `var`.

Esempio:

```csharp
var nome = "Emilie";
var eta = 42;
var prezzo = 19.90m;
```

`var` non significa “tipo dinamico”.

Significa:

> il compilatore deduce il tipo dalla parte destra dell’assegnazione.

Quindi:

```csharp
var nome = "Emilie";
```

è equivalente a:

```csharp
string nome = "Emilie";
```

E:

```csharp
var eta = 42;
```

è equivalente a:

```csharp
int eta = 42;
```

---

## var non cambia tipo durante l’esecuzione

Questa variabile:

```csharp
var numero = 10;
```

è di tipo `int`.

Non possiamo poi assegnarle una stringa:

```csharp
numero = "ciao"; // errore
```

Il tipo è stato dedotto all’inizio e rimane quello.

---

## var richiede un valore iniziale

Questo codice è valido:

```csharp
var nome = "Emilie";
```

Questo no:

```csharp
var nome;
```

Il compilatore non può dedurre il tipo se non c’è un valore iniziale.

---

## Quando usare var

`var` è utile quando il tipo è evidente:

```csharp
var nome = "Emilie";
var totale = 100.50m;
var clienti = new List<string>();
```

Può essere meno chiaro quando il tipo non è evidente:

```csharp
var risultato = servizio.Calcola();
```

Qui il lettore non sa subito che tipo abbia `risultato`.

Regola pratica:

```text
Usa var quando il tipo è evidente dal contesto.
Usa il tipo esplicito quando migliora la leggibilità.
```

Esempio leggibile:

```csharp
decimal totale = CalcolaTotale();
```

Qui `decimal` aiuta a capire che stiamo lavorando con un importo.

---

# Tipi numerici

C# offre diversi tipi numerici.

I più comuni sono:

```text
int
long
decimal
double
float
```

La scelta del tipo è importante, perché non tutti i numeri sono uguali.

Alcuni tipi rappresentano numeri interi.
Altri rappresentano numeri con decimali.
Alcuni sono più precisi.
Altri sono più veloci o occupano meno memoria.

---

## `int`

`int` rappresenta un numero intero.

Esempio:

```csharp
int eta = 42;
int numeroClienti = 1000;
int saldo = -50;
```

È uno dei tipi più usati.

Si usa per valori interi di dimensione normale:

* età;
* conteggi;
* quantità;
* indici;
* numero di tentativi;
* numero di righe.

Esempio:

```csharp
int quantita = 3;
int prezzoUnitario = 10;
int totale = quantita * prezzoUnitario;

Console.WriteLine(totale);
```

Output:

```text
30
```

`int` non può contenere numeri decimali.

Questo codice non è valido:

```csharp
int prezzo = 10.5; // errore
```

---

## `long`

`long` rappresenta un numero intero più grande di `int`.

Esempio:

```csharp
long popolazioneMondiale = 8000000000L;
```

La `L` finale indica che il numero deve essere trattato come `long`.

`long` si usa quando un `int` potrebbe non bastare:

* identificativi molto grandi;
* contatori molto grandi;
* dimensioni di file;
* valori numerici interi elevati.

Esempio:

```csharp
long dimensioneFile = 5_000_000_000L;

Console.WriteLine(dimensioneFile);
```

In C# possiamo usare `_` nei numeri per migliorarne la leggibilità:

```csharp
int milione = 1_000_000;
```

Il valore è sempre `1000000`, ma è più leggibile.

---

## `decimal`

`decimal` rappresenta numeri decimali con alta precisione, ed è particolarmente adatto per importi monetari.

Esempio:

```csharp
decimal prezzo = 19.90m;
decimal iva = 4.38m;
decimal totale = prezzo + iva;
```

La `m` finale indica che il valore è un `decimal`.

Senza `m`, un numero decimale viene normalmente interpretato come `double`.

Questo codice non è valido:

```csharp
decimal prezzo = 19.90; // errore
```

Versione corretta:

```csharp
decimal prezzo = 19.90m;
```

`decimal` è la scelta consigliata per:

* prezzi;
* importi;
* contabilità;
* percentuali finanziarie;
* calcoli monetari.

Esempio:

```csharp
decimal imponibile = 100m;
decimal aliquotaIva = 22m;

decimal iva = imponibile * aliquotaIva / 100;
decimal totale = imponibile + iva;

Console.WriteLine(totale);
```

Output:

```text
122
```

---

## `double`

`double` rappresenta numeri decimali in virgola mobile.

Esempio:

```csharp
double temperatura = 36.6;
double distanza = 123.45;
double media = 10.0 / 3.0;
```

`double` è molto usato per:

* calcoli scientifici;
* misure;
* coordinate;
* statistiche;
* grafica;
* simulazioni;
* valori approssimati.

Non è ideale per importi monetari, perché usa una rappresentazione binaria che può produrre piccole imprecisioni.

Esempio:

```csharp
double risultato = 0.1 + 0.2;

Console.WriteLine(risultato);
```

Il risultato potrebbe non essere esattamente quello che ci aspettiamo dal punto di vista decimale.

Per la contabilità è meglio usare `decimal`.

---

## `float`

`float` è simile a `double`, ma usa meno memoria e ha meno precisione.

Esempio:

```csharp
float temperatura = 36.6f;
```

La `f` finale indica che il valore è un `float`.

Senza `f`, il valore `36.6` viene considerato `double`.

`float` è usato in scenari specifici, per esempio:

* grafica;
* videogiochi;
* elaborazioni dove si accetta minore precisione;
* grandi quantità di dati numerici.

Nel codice applicativo comune, se serve un numero decimale approssimato, si usa più spesso `double`.

---

## Confronto tra tipi numerici

```text
int
  numero intero normale.

long
  numero intero molto grande.

decimal
  numero decimale preciso, ideale per denaro.

double
  numero decimale approssimato, ideale per calcoli scientifici e misure.

float
  numero decimale approssimato con minore precisione, utile in scenari specifici.
```

Regola pratica:

```text
Conteggi e quantità normali
  int

Interi molto grandi
  long

Soldi e importi
  decimal

Misure e calcoli scientifici
  double

Grafica o scenari con memoria/prestazioni specifiche
  float
```

---

# `bool`

Il tipo `bool` rappresenta un valore booleano.

Può contenere solo due valori:

```csharp
true
false
```

Esempio:

```csharp
bool maggiorenne = true;
bool accessoConsentito = false;
```

I booleani sono usati per rappresentare condizioni.

Esempio:

```csharp
int eta = 20;

bool maggiorenne = eta >= 18;

Console.WriteLine(maggiorenne);
```

Output:

```text
True
```

In C#, le espressioni di confronto producono valori booleani.

```csharp
bool risultato = 10 > 5;
```

`risultato` vale `true`.

---

## Booleani e condizioni

I valori booleani sono usati con `if`.

```csharp
bool utenteAttivo = true;

if (utenteAttivo)
{
    Console.WriteLine("Utente attivo");
}
```

Non serve scrivere:

```csharp
if (utenteAttivo == true)
{
    Console.WriteLine("Utente attivo");
}
```

È valido, ma più verboso.

Meglio:

```csharp
if (utenteAttivo)
{
    Console.WriteLine("Utente attivo");
}
```

Per il caso opposto:

```csharp
if (!utenteAttivo)
{
    Console.WriteLine("Utente non attivo");
}
```

L’operatore `!` significa negazione.

---

# `char`

Il tipo `char` rappresenta un singolo carattere.

Esempio:

```csharp
char iniziale = 'E';
char separatore = '-';
char cifra = '7';
```

I valori `char` si scrivono tra apici singoli:

```csharp
'A'
```

Le stringhe invece si scrivono tra doppi apici:

```csharp
"A"
```

Quindi:

```csharp
char lettera = 'A';
string testo = "A";
```

Sembrano simili, ma non sono la stessa cosa.

```text
char
  un singolo carattere.

string
  una sequenza di caratteri.
```

Questo codice non è valido:

```csharp
char nome = 'AB'; // errore
```

Un `char` contiene un solo carattere.

---

# `string`

Il tipo `string` rappresenta testo.

Esempio:

```csharp
string nome = "Emilie";
string città = "Aosta";
string messaggio = "Benvenuta";
```

Le stringhe si scrivono tra doppi apici:

```csharp
"Ciao"
```

Una stringa può essere vuota:

```csharp
string testo = "";
```

Può contenere spazi:

```csharp
string spazio = " ";
```

Può contenere anche caratteri speciali:

```csharp
string percorso = "C:\\Temp\\file.txt";
```

In questo caso `\\` serve per rappresentare il carattere `\`.

---

## Concatenazione di stringhe

Possiamo unire stringhe con `+`.

```csharp
string nome = "Emilie";
string messaggio = "Ciao, " + nome;

Console.WriteLine(messaggio);
```

Output:

```text
Ciao, Emilie
```

La concatenazione è semplice, ma può diventare poco leggibile quando le parti sono molte.

---

## Interpolazione di stringhe

In C# è molto comune usare l’interpolazione.

```csharp
string nome = "Emilie";
int eta = 42;

string messaggio = $"Ciao, {nome}. Hai {eta} anni.";

Console.WriteLine(messaggio);
```

Output:

```text
Ciao, Emilie. Hai 42 anni.
```

La stringa interpolata inizia con `$`.

Dentro le parentesi graffe possiamo inserire variabili o espressioni:

```csharp
int prezzo = 10;
int quantita = 3;

Console.WriteLine($"Totale: {prezzo * quantita}");
```

Output:

```text
Totale: 30
```

---

## Stringhe multilinea

C# supporta diversi modi per scrivere stringhe su più righe.

Una forma moderna sono le raw string literals, utili per testi lunghi, JSON, SQL o HTML.

Esempio:

```csharp
string testo = """
Questo è un testo
su più righe.
""";
```

Le raw string literals sono molto comode quando il testo contiene virgolette o caratteri speciali.

Esempio:

```csharp
string json = """
{
  "nome": "Emilie",
  "citta": "Aosta"
}
""";
```

Approfondiremo questi dettagli più avanti. Per ora basta sapere che `string` è il tipo principale per lavorare con testo.

---

## string è un tipo riferimento

`string` si comporta in modo molto comodo, ma tecnicamente è un tipo riferimento.

Questo significa che può anche avere valore `null`.

Con i Nullable Reference Types abilitati, possiamo distinguere:

```csharp
string nome = "Emilie";
string? soprannome = null;
```

La differenza tra `string` e `string?` diventerà molto importante nel capitolo sui nullable reference types.

Per ora ricordiamo:

```text
string
  dovrebbe contenere sempre un testo.

string?
  può contenere testo oppure null.
```

# Date e orari

Le applicazioni reali lavorano spesso con date e orari:

* data di nascita;
* data documento;
* data scadenza;
* ora di ingresso;
* durata di un’attività;
* intervallo tra due eventi.

In .NET i tipi più comuni sono:

```text
DateTime
DateOnly
TimeOnly
TimeSpan
```

---

## `DateTime`

`DateTime` rappresenta una data con un orario.

Esempio:

```csharp
DateTime adesso = DateTime.Now;

Console.WriteLine(adesso);
```

`DateTime.Now` restituisce data e ora correnti del sistema.

Possiamo creare una data specifica:

```csharp
DateTime dataNascita = new DateTime(1980, 5, 20);
```

I parametri sono:

```text
anno
mese
giorno
```

Possiamo anche specificare ora, minuti e secondi:

```csharp
DateTime appuntamento = new DateTime(2026, 4, 24, 15, 30, 0);
```

---

## Proprietà utili di DateTime

```csharp
DateTime oggi = DateTime.Today;

Console.WriteLine(oggi.Year);
Console.WriteLine(oggi.Month);
Console.WriteLine(oggi.Day);
```

Possiamo usare:

```text
Year
Month
Day
Hour
Minute
Second
DayOfWeek
DayOfYear
```

Esempio:

```csharp
DateTime oggi = DateTime.Today;

Console.WriteLine($"Anno: {oggi.Year}");
Console.WriteLine($"Mese: {oggi.Month}");
Console.WriteLine($"Giorno: {oggi.Day}");
```

---

## `DateOnly`

`DateOnly` rappresenta solo una data, senza orario.

Esempio:

```csharp
DateOnly dataNascita = new DateOnly(1980, 5, 20);
```

È utile quando l’ora non ha significato.

Esempi:

* data di nascita;
* data di emissione documento;
* data di scadenza;
* giorno di prenotazione;
* giorno contabile.

Usare `DateTime` per una data senza orario può creare ambiguità, perché il valore contiene comunque una componente oraria, spesso impostata a mezzanotte.

Se il concetto è solo una data, `DateOnly` è più espressivo.

---

## `TimeOnly`

`TimeOnly` rappresenta solo un orario, senza data.

Esempio:

```csharp
TimeOnly oraApertura = new TimeOnly(9, 30);
TimeOnly oraChiusura = new TimeOnly(18, 0);
```

È utile per:

* orario di apertura;
* orario di chiusura;
* orario di un turno;
* ora giornaliera ricorrente.

Esempio:

```csharp
TimeOnly oraIngresso = new TimeOnly(14, 45);

Console.WriteLine(oraIngresso);
```

---

## `TimeSpan`

`TimeSpan` rappresenta una durata o un intervallo di tempo.

Esempio:

```csharp
TimeSpan durata = TimeSpan.FromMinutes(90);
```

Possiamo creare durate in vari modi:

```csharp
TimeSpan unOra = TimeSpan.FromHours(1);
TimeSpan trentaMinuti = TimeSpan.FromMinutes(30);
TimeSpan dieciSecondi = TimeSpan.FromSeconds(10);
```

Esempio con differenza tra due date:

```csharp
DateTime inizio = new DateTime(2026, 4, 24, 10, 0, 0);
DateTime fine = new DateTime(2026, 4, 24, 12, 30, 0);

TimeSpan durata = fine - inizio;

Console.WriteLine(durata.TotalMinutes);
```

Output:

```text
150
```

`TimeSpan` non rappresenta “le 10:30”.
Rappresenta “una durata di 10 ore e 30 minuti” oppure una differenza tra due momenti.

---

## Confronto tra DateTime, DateOnly, TimeOnly e TimeSpan

```text
DateTime
  data + ora.

DateOnly
  solo data.

TimeOnly
  solo ora del giorno.

TimeSpan
  durata/intervallo.
```

Esempi:

```csharp
DateTime ingresso = new DateTime(2026, 4, 24, 21, 30, 0);

DateOnly giornoEvento = new DateOnly(2026, 4, 24);

TimeOnly oraApertura = new TimeOnly(9, 0);

TimeSpan durataMassima = TimeSpan.FromHours(4);
```

Regola pratica:

```text
Mi serve un momento preciso?
  DateTime

Mi serve solo una data?
  DateOnly

Mi serve solo un orario?
  TimeOnly

Mi serve una durata?
  TimeSpan
```

# Nullable value types

Abbiamo visto tipi come:

```text
int
decimal
bool
DateTime
DateOnly
TimeOnly
```

Questi sono **value types**.

Normalmente un value type non può contenere `null`.

Questo codice non è valido:

```csharp
int eta = null; // errore
```

Ma in molte applicazioni reali un valore può essere assente.

Esempi:

* data di nascita non comunicata;
* data di scadenza non presente;
* importo non ancora calcolato;
* numero documento mancante;
* consenso non ancora espresso.

Per rappresentare questa possibilità, C# offre i **nullable value types**.

---

## Sintassi con `?`

Per rendere nullable un value type, aggiungiamo `?`.

```csharp
int? eta = null;
decimal? importo = null;
DateTime? dataScadenza = null;
bool? consensoMarketing = null;
```

`int?` significa:

> può contenere un `int` oppure `null`.

Esempio:

```csharp
int? eta = null;

Console.WriteLine(eta);
```

Poi possiamo assegnare un valore:

```csharp
eta = 42;
```

---

## HasValue e Value

Un nullable value type ha proprietà come:

```text
HasValue
Value
```

Esempio:

```csharp
int? eta = 42;

if (eta.HasValue)
{
    Console.WriteLine($"Età: {eta.Value}");
}
else
{
    Console.WriteLine("Età non disponibile");
}
```

`HasValue` dice se il valore è presente.

`Value` restituisce il valore vero e proprio, ma va usato solo quando siamo sicuri che il valore esista.

Se usiamo `.Value` quando il valore è `null`, otteniamo un errore a runtime.

---

## Pattern più moderno

Spesso possiamo scrivere in modo più semplice:

```csharp
int? eta = 42;

if (eta is not null)
{
    Console.WriteLine($"Età: {eta}");
}
```

Oppure:

```csharp
int? eta = null;

int etaEffettiva = eta ?? 0;

Console.WriteLine(etaEffettiva);
```

L’operatore `??` dice:

> se `eta` ha un valore, usa quello; altrimenti usa `0`.

---

## Quando usare nullable value types

Usiamo nullable value types quando l’assenza del valore è significativa.

Esempio buono:

```csharp
DateTime? dataCancellazione = null;
```

Qui `null` può significare:

> il record non è stato cancellato.

Esempio:

```csharp
decimal? scontoApplicato = null;
```

Qui `null` può significare:

> lo sconto non è stato ancora calcolato.

Attenzione però: non bisogna usare `null` quando esiste un valore reale più chiaro.

Se una quantità mancante deve essere trattata come zero, forse è meglio usare direttamente:

```csharp
int quantita = 0;
```

Se invece “mancante” e “zero” hanno significati diversi, allora ha senso:

```csharp
int? quantita = null;
```

---

# Inferenza del tipo

L’inferenza del tipo è la capacità del compilatore di dedurre automaticamente il tipo di una variabile.

In C# questo avviene con `var`.

Esempio:

```csharp
var numero = 10;
var nome = "Emilie";
var prezzo = 19.90m;
var attivo = true;
```

Il compilatore deduce:

```text
numero -> int
nome   -> string
prezzo -> decimal
attivo -> bool
```

Il codice è fortemente tipizzato anche se usiamo `var`.

---

## Inferenza non significa dynamic

Questo punto è importante.

```csharp
var valore = 10;
```

`valore` è un `int`.

Non può cambiare tipo:

```csharp
valore = "ciao"; // errore
```

C# deduce il tipo a compilazione, non durante l’esecuzione.

---

## Inferenza e leggibilità

`var` può migliorare o peggiorare la leggibilità.

Esempio chiaro:

```csharp
var cliente = new Cliente();
```

Il tipo è evidente: `Cliente`.

Esempio meno chiaro:

```csharp
var risultato = servizio.Esegui();
```

Qui non sappiamo che cosa restituisce `Esegui`.

In quel caso può essere meglio scrivere:

```csharp
RisultatoOperazione risultato = servizio.Esegui();
```

Regola pratica:

```text
Il tipo è evidente?
  var va bene.

Il tipo aiuta a capire il codice?
  meglio scriverlo esplicitamente.
```

# Conversioni implicite ed esplicite

In C# capita spesso di dover convertire un valore da un tipo a un altro.

Per esempio:

* da `int` a `long`;
* da `int` a `decimal`;
* da `double` a `int`;
* da `string` a `int`;
* da `DateTime` a `string`.

Alcune conversioni sono automatiche.
Altre richiedono una dichiarazione esplicita.

---

## Conversioni implicite

Una conversione implicita avviene automaticamente quando non c’è rischio significativo di perdita di informazione.

Esempio:

```csharp
int numeroPiccolo = 100;
long numeroGrande = numeroPiccolo;
```

Un `int` può entrare senza problemi in un `long`.

Altro esempio:

```csharp
int quantita = 5;
decimal quantitaDecimale = quantita;
```

Il valore intero può essere convertito in `decimal`.

---

## Conversioni esplicite

Una conversione esplicita richiede un cast.

Esempio:

```csharp
double valore = 10.8;
int intero = (int)valore;

Console.WriteLine(intero);
```

Output:

```text
10
```

La parte decimale viene persa.

Questo è il motivo per cui il compilatore non fa questa conversione automaticamente. Potremmo perdere informazioni.

Altro esempio:

```csharp
long numeroGrande = 1000;
int numeroPiccolo = (int)numeroGrande;
```

Questo è consentito, ma può essere rischioso se il valore di `numeroGrande` supera la capacità di `int`.

---

## Conversione non significa arrotondamento

Questo codice:

```csharp
double valore = 10.9;
int intero = (int)valore;
```

non produce `11`.

Produce:

```text
10
```

Il cast tronca la parte decimale.

Se vogliamo arrotondare, dobbiamo usare metodi specifici:

```csharp
double valore = 10.9;
int arrotondato = (int)Math.Round(valore);

Console.WriteLine(arrotondato);
```

Output:

```text
11
```

---

## Conversioni da stringa

Quando leggiamo input dalla console, riceviamo testo.

```csharp
string? input = Console.ReadLine();
```

Se vogliamo ottenere un numero, dobbiamo convertirlo.

Metodo rischioso:

```csharp
int eta = int.Parse(input!);
```

Se `input` non contiene un numero valido, il programma genera un’eccezione.

Metodo più sicuro:

```csharp
if (int.TryParse(input, out int eta))
{
    Console.WriteLine($"Età: {eta}");
}
else
{
    Console.WriteLine("Valore non valido");
}
```

`TryParse` restituisce `true` se la conversione riesce, `false` altrimenti.

---

## Convert

La classe `Convert` offre metodi per convertire valori.

Esempio:

```csharp
string testo = "123";
int numero = Convert.ToInt32(testo);
```

Oppure:

```csharp
int numero = 42;
string testo = Convert.ToString(numero);
```

`Convert` è utile, ma anche qui bisogna gestire possibili errori quando il valore non è convertibile.

Per input utente, spesso `TryParse` è preferibile.

---

## Conversione a stringa

Quasi tutti i valori possono essere convertiti in stringa con `ToString`.

```csharp
int numero = 42;
string testo = numero.ToString();
```

Esempio con `DateTime`:

```csharp
DateTime oggi = DateTime.Today;
string testo = oggi.ToString();

Console.WriteLine(testo);
```

La formattazione può dipendere dalla lingua e dalla cultura del sistema.

Per formattare in modo esplicito:

```csharp
DateTime oggi = DateTime.Today;

Console.WriteLine(oggi.ToString("yyyy-MM-dd"));
```

Output possibile:

```text
2026-04-24
```

# Errori comuni

## Usare `double` per importi monetari

Sconsigliato:

```csharp
double prezzo = 19.90;
```

Meglio:

```csharp
decimal prezzo = 19.90m;
```

Per importi, prezzi, sconti e contabilità, `decimal` è quasi sempre la scelta corretta.

---

## Dimenticare il suffisso `m` per decimal

Questo codice non è corretto:

```csharp
decimal prezzo = 19.90;
```

Versione corretta:

```csharp
decimal prezzo = 19.90m;
```

---

## Confondere `char` e `string`

Questo è un carattere:

```csharp
char iniziale = 'E';
```

Questa è una stringa:

```csharp
string iniziale = "E";
```

Gli apici singoli sono per `char`.
I doppi apici sono per `string`.

---

## Usare `Parse` su input non controllato

Rischioso:

```csharp
int eta = int.Parse(Console.ReadLine()!);
```

Meglio:

```csharp
string? input = Console.ReadLine();

if (int.TryParse(input, out int eta))
{
    Console.WriteLine(eta);
}
else
{
    Console.WriteLine("Numero non valido");
}
```

---

## Usare `var` quando nasconde informazioni importanti

Poco chiaro:

```csharp
var risultato = servizio.Calcola();
```

Più chiaro:

```csharp
decimal totale = servizio.CalcolaTotale();
```

`var` è utile, ma non deve rendere il codice più misterioso.

---

## Usare `null` quando zero sarebbe più corretto

Ambiguo:

```csharp
int? quantita = null;
```

Se il significato corretto è “nessuna quantità”, forse basta:

```csharp
int quantita = 0;
```

Ma se “non indicato” e “zero” sono due situazioni diverse, allora `int?` è corretto.
