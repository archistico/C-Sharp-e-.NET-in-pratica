# 02.03 — Controllo del flusso

Nei capitoli precedenti abbiamo imparato a dichiarare variabili, usare tipi fondamentali, comporre espressioni, confrontare valori e convertire dati.

Ora dobbiamo imparare a controllare **il percorso di esecuzione** del programma.

Un programma reale non esegue sempre tutte le istruzioni dall’alto verso il basso. Deve prendere decisioni:

```text
Se il cliente è maggiorenne, permetti l’accesso.
Altrimenti, nega l’accesso.
```

Deve ripetere operazioni:

```text
Per ogni riga del file, elabora il contenuto.
```

Deve interrompersi quando una situazione non è valida:

```text
Se il codice cliente è mancante, termina subito.
```

Deve riconoscere casi diversi:

```text
Se il documento è una carta d’identità, applica una regola.
Se è un passaporto, applicane un’altra.
Se è una patente, applicane un’altra ancora.
```

In questo capitolo vediamo gli strumenti principali di C# per controllare il flusso:

* `if`;
* `else`;
* `switch`;
* `switch expression`;
* `while`;
* `do while`;
* `for`;
* `foreach`;
* `break`;
* `continue`;
* `return`;
* pattern matching;
* guard clauses;
* codice difensivo.

---

# Il flusso di esecuzione

In assenza di condizioni o cicli, un programma esegue le istruzioni in ordine.

```csharp
Console.WriteLine("Prima istruzione");
Console.WriteLine("Seconda istruzione");
Console.WriteLine("Terza istruzione");
```

Output:

```text
Prima istruzione
Seconda istruzione
Terza istruzione
```

Il flusso è lineare.

Ma spesso vogliamo eseguire alcune istruzioni solo in certi casi.

Esempio:

```csharp
int eta = 20;

if (eta >= 18)
{
    Console.WriteLine("Maggiorenne");
}
```

Qui la scritta viene stampata solo se `eta >= 18`.

Il codice inizia a diventare decisionale.

---

# `if`

L’istruzione `if` permette di eseguire un blocco di codice solo se una condizione è vera.

Sintassi:

```csharp
if (condizione)
{
    // codice eseguito se la condizione è vera
}
```

Esempio:

```csharp
int eta = 20;

if (eta >= 18)
{
    Console.WriteLine("Accesso consentito");
}
```

La condizione deve produrre un valore `bool`.

Questa espressione:

```csharp
eta >= 18
```

produce `true` oppure `false`.

Se è `true`, il blocco viene eseguito.
Se è `false`, il blocco viene saltato.

---

## Esempio con input

```csharp
Console.Write("Inserisci la tua età: ");

string? input = Console.ReadLine();

if (int.TryParse(input, out int eta))
{
    Console.WriteLine($"Età inserita: {eta}");
}
```

Il messaggio viene stampato solo se la conversione riesce.

---

## Blocchi con parentesi graffe

In C#, se il blocco contiene una sola istruzione, le parentesi graffe possono essere omesse.

```csharp
if (eta >= 18)
    Console.WriteLine("Accesso consentito");
```

Tuttavia, nei progetti reali è meglio usare quasi sempre le parentesi graffe.

Meglio:

```csharp
if (eta >= 18)
{
    Console.WriteLine("Accesso consentito");
}
```

Questo evita errori quando si aggiungono istruzioni in seguito.

Esempio rischioso:

```csharp
if (eta >= 18)
    Console.WriteLine("Accesso consentito");
    Console.WriteLine("Benvenuto");
```

La seconda `WriteLine` viene eseguita sempre, non solo quando la condizione è vera.

Con le parentesi graffe il codice è più chiaro:

```csharp
if (eta >= 18)
{
    Console.WriteLine("Accesso consentito");
    Console.WriteLine("Benvenuto");
}
```

---

# `else`

`else` permette di specificare cosa fare quando la condizione dell’`if` è falsa.

```csharp
int eta = 16;

if (eta >= 18)
{
    Console.WriteLine("Accesso consentito");
}
else
{
    Console.WriteLine("Accesso negato");
}
```

Output:

```text
Accesso negato
```

La struttura è:

```csharp
if (condizione)
{
    // condizione vera
}
else
{
    // condizione falsa
}
```

---

## Esempio completo

```csharp
Console.Write("Inserisci la tua età: ");

string? input = Console.ReadLine();

if (int.TryParse(input, out int eta))
{
    if (eta >= 18)
    {
        Console.WriteLine("Sei maggiorenne.");
    }
    else
    {
        Console.WriteLine("Sei minorenne.");
    }
}
else
{
    Console.WriteLine("Il valore inserito non è valido.");
}
```

Qui abbiamo due decisioni:

1. l’input è un numero valido?
2. l’età è almeno 18?

---

# `else if`

Quando abbiamo più casi alternativi, possiamo usare `else if`.

```csharp
int punteggio = 85;

if (punteggio >= 90)
{
    Console.WriteLine("Ottimo");
}
else if (punteggio >= 70)
{
    Console.WriteLine("Buono");
}
else if (punteggio >= 60)
{
    Console.WriteLine("Sufficiente");
}
else
{
    Console.WriteLine("Insufficiente");
}
```

Output:

```text
Buono
```

Il programma valuta le condizioni dall’alto verso il basso.

Appena trova una condizione vera, esegue il blocco corrispondente e salta gli altri.

---

## Attenzione all’ordine

L’ordine delle condizioni è importante.

Questo codice è corretto:

```csharp
if (punteggio >= 90)
{
    Console.WriteLine("Ottimo");
}
else if (punteggio >= 70)
{
    Console.WriteLine("Buono");
}
```

Questo invece è sbagliato dal punto di vista logico:

```csharp
if (punteggio >= 70)
{
    Console.WriteLine("Buono");
}
else if (punteggio >= 90)
{
    Console.WriteLine("Ottimo");
}
```

Se `punteggio` vale `95`, la prima condizione `punteggio >= 70` è già vera. Il ramo `Ottimo` non verrà mai raggiunto.

Regola pratica:

```text
Quando le condizioni sono sovrapposte,
metti prima quelle più specifiche.
```

---

# Condizioni annidate

Possiamo inserire un `if` dentro un altro `if`.

```csharp
bool documentoValido = true;
int eta = 20;

if (eta >= 18)
{
    if (documentoValido)
    {
        Console.WriteLine("Accesso consentito");
    }
    else
    {
        Console.WriteLine("Documento non valido");
    }
}
else
{
    Console.WriteLine("Cliente minorenne");
}
```

Il codice funziona, ma può diventare difficile da leggere se l’annidamento cresce troppo.

Un’alternativa più chiara è combinare le condizioni.

```csharp
if (eta >= 18 && documentoValido)
{
    Console.WriteLine("Accesso consentito");
}
else
{
    Console.WriteLine("Accesso non consentito");
}
```

Oppure usare guard clauses, che vedremo più avanti.

# `switch`

`switch` serve a scegliere tra più casi in base al valore di un’espressione.

Esempio:

```csharp
string tipoDocumento = "CI";

switch (tipoDocumento)
{
    case "CI":
        Console.WriteLine("Carta d'identità");
        break;

    case "PA":
        Console.WriteLine("Passaporto");
        break;

    case "PT":
        Console.WriteLine("Patente");
        break;

    default:
        Console.WriteLine("Tipo documento non riconosciuto");
        break;
}
```

Output:

```text
Carta d'identità
```

---

## Struttura di uno switch

Uno `switch` classico ha questa forma:

```csharp
switch (valore)
{
    case valore1:
        // codice
        break;

    case valore2:
        // codice
        break;

    default:
        // codice se nessun caso corrisponde
        break;
}
```

`default` viene eseguito se nessun `case` corrisponde.

---

## Perché serve `break`

Nel C# classico, ogni `case` deve terminare in modo esplicito, spesso con `break`.

```csharp
case "CI":
    Console.WriteLine("Carta d'identità");
    break;
```

`break` dice:

> esci dallo switch.

Senza `break`, il compilatore segnala errore in molti casi, perché C# evita il fall-through implicito tipico di altri linguaggi come C o JavaScript.

---

## Più case con lo stesso comportamento

Possiamo raggruppare più casi.

```csharp
string tipoDocumento = "ID";

switch (tipoDocumento)
{
    case "CI":
    case "ID":
        Console.WriteLine("Carta d'identità");
        break;

    case "PA":
    case "PAS":
        Console.WriteLine("Passaporto");
        break;

    default:
        Console.WriteLine("Documento non riconosciuto");
        break;
}
```

Qui `"CI"` e `"ID"` producono lo stesso risultato.

---

## Quando usare switch

`switch` è adatto quando:

* abbiamo un valore da classificare;
* i casi sono distinti;
* ci sono molte alternative;
* vogliamo evitare una lunga catena di `else if`.

Esempio adatto:

```csharp
switch (codiceStato)
{
    case "A":
        Console.WriteLine("Attivo");
        break;

    case "S":
        Console.WriteLine("Sospeso");
        break;

    case "C":
        Console.WriteLine("Cancellato");
        break;

    default:
        Console.WriteLine("Sconosciuto");
        break;
}
```

---

# `switch expression`

La **switch expression** è una forma moderna e compatta di `switch`.

È molto utile quando vogliamo calcolare un valore in base a un altro valore.

Esempio:

```csharp
string tipoDocumento = "CI";

string descrizione = tipoDocumento switch
{
    "CI" => "Carta d'identità",
    "PA" => "Passaporto",
    "PT" => "Patente",
    _ => "Tipo documento non riconosciuto"
};

Console.WriteLine(descrizione);
```

Output:

```text
Carta d'identità
```

---

## Differenza tra switch statement e switch expression

Lo `switch` classico è un’istruzione:

```csharp
switch (tipoDocumento)
{
    case "CI":
        Console.WriteLine("Carta d'identità");
        break;
}
```

La `switch expression` produce un valore:

```csharp
string descrizione = tipoDocumento switch
{
    "CI" => "Carta d'identità",
    _ => "Sconosciuto"
};
```

Regola pratica:

```text
Devi eseguire blocchi di codice?
  switch statement.

Devi calcolare un valore?
  switch expression.
```

---

## `_` come caso predefinito

Nelle switch expression il simbolo `_` indica il caso predefinito.

```csharp
string descrizione = codice switch
{
    "A" => "Attivo",
    "S" => "Sospeso",
    _ => "Sconosciuto"
};
```

È simile a `default` nello switch classico.

---

## Esempio con numeri

```csharp
int voto = 8;

string giudizio = voto switch
{
    10 => "Eccellente",
    9 => "Ottimo",
    8 => "Buono",
    7 => "Discreto",
    6 => "Sufficiente",
    _ => "Da valutare"
};

Console.WriteLine(giudizio);
```

Output:

```text
Buono
```

# Cicli

I cicli servono a ripetere istruzioni.

Senza cicli, se volessimo stampare i numeri da 1 a 5, dovremmo scrivere:

```csharp
Console.WriteLine(1);
Console.WriteLine(2);
Console.WriteLine(3);
Console.WriteLine(4);
Console.WriteLine(5);
```

Con un ciclo:

```csharp
for (int i = 1; i <= 5; i++)
{
    Console.WriteLine(i);
}
```

Molto meglio.

I principali cicli in C# sono:

* `while`;
* `do while`;
* `for`;
* `foreach`.

---

# `while`

Il ciclo `while` ripete un blocco finché una condizione è vera.

Sintassi:

```csharp
while (condizione)
{
    // codice da ripetere
}
```

Esempio:

```csharp
int contatore = 1;

while (contatore <= 5)
{
    Console.WriteLine(contatore);
    contatore++;
}
```

Output:

```text
1
2
3
4
5
```

---

## Come funziona while

Il ciclo valuta la condizione prima di ogni iterazione.

```text
1. Controlla la condizione.
2. Se è vera, esegue il blocco.
3. Torna al punto 1.
4. Se è falsa, esce dal ciclo.
```

Se la condizione è falsa fin dall’inizio, il blocco non viene mai eseguito.

```csharp
int contatore = 10;

while (contatore <= 5)
{
    Console.WriteLine(contatore);
    contatore++;
}
```

Questo codice non stampa nulla.

---

## Attenzione ai cicli infiniti

Un ciclo può diventare infinito se la condizione non diventa mai falsa.

```csharp
int contatore = 1;

while (contatore <= 5)
{
    Console.WriteLine(contatore);
}
```

Qui `contatore` non cambia mai. La condizione resta sempre vera.

Versione corretta:

```csharp
int contatore = 1;

while (contatore <= 5)
{
    Console.WriteLine(contatore);
    contatore++;
}
```

---

## Esempio pratico con input

```csharp
string? comando = "";

while (comando != "esci")
{
    Console.Write("Comando: ");
    comando = Console.ReadLine();

    Console.WriteLine($"Hai scritto: {comando}");
}
```

Il ciclo continua finché l’utente non scrive `esci`.

---

# `do while`

Il ciclo `do while` è simile a `while`, ma valuta la condizione alla fine.

Questo significa che il blocco viene eseguito almeno una volta.

Sintassi:

```csharp
do
{
    // codice
}
while (condizione);
```

Esempio:

```csharp
string? comando;

do
{
    Console.Write("Scrivi 'esci' per terminare: ");
    comando = Console.ReadLine();
}
while (comando != "esci");
```

Anche se la condizione fosse falsa alla prima verifica, il blocco verrebbe comunque eseguito una volta.

---

## Differenza tra while e do while

```csharp
int numero = 10;

while (numero < 5)
{
    Console.WriteLine("while");
}
```

Non stampa nulla.

```csharp
int numero = 10;

do
{
    Console.WriteLine("do while");
}
while (numero < 5);
```

Stampa:

```text
do while
```

Regola pratica:

```text
while
  usa il ciclo quando potresti volerlo eseguire zero volte.

do while
  usa il ciclo quando vuoi eseguirlo almeno una volta.
```

---

# `for`

Il ciclo `for` è adatto quando sappiamo quante volte vogliamo ripetere un blocco.

Sintassi:

```csharp
for (inizializzazione; condizione; aggiornamento)
{
    // codice
}
```

Esempio:

```csharp
for (int i = 1; i <= 5; i++)
{
    Console.WriteLine(i);
}
```

Output:

```text
1
2
3
4
5
```

---

## Le tre parti del for

```csharp
for (int i = 1; i <= 5; i++)
```

Contiene:

```text
int i = 1
  inizializzazione.

i <= 5
  condizione.

i++
  aggiornamento.
```

Il funzionamento è:

```text
1. Crea i e assegna 1.
2. Verifica i <= 5.
3. Esegue il blocco.
4. Esegue i++.
5. Torna al punto 2.
```

---

## Ciclo all’indietro

```csharp
for (int i = 5; i >= 1; i--)
{
    Console.WriteLine(i);
}
```

Output:

```text
5
4
3
2
1
```

---

## Scorrere un array con for

```csharp
string[] nomi = { "Anna", "Luca", "Emilie" };

for (int i = 0; i < nomi.Length; i++)
{
    Console.WriteLine(nomi[i]);
}
```

Output:

```text
Anna
Luca
Emilie
```

Gli indici degli array partono da `0`.

Quindi, per un array di 3 elementi:

```text
Indice 0 -> primo elemento
Indice 1 -> secondo elemento
Indice 2 -> terzo elemento
```

L’ultimo indice è:

```csharp
nomi.Length - 1
```

Per questo la condizione corretta è:

```csharp
i < nomi.Length
```

non:

```csharp
i <= nomi.Length
```

---

# `foreach`

Il ciclo `foreach` serve a scorrere tutti gli elementi di una sequenza.

Esempio:

```csharp
string[] nomi = { "Anna", "Luca", "Emilie" };

foreach (string nome in nomi)
{
    Console.WriteLine(nome);
}
```

Output:

```text
Anna
Luca
Emilie
```

`foreach` è spesso più leggibile di `for` quando non ci serve l’indice.

---

## foreach con liste

```csharp
List<string> città = new List<string>
{
    "Aosta",
    "Torino",
    "Milano"
};

foreach (string nomeCitta in città)
{
    Console.WriteLine(nomeCitta);
}
```

---

## for o foreach?

Regola pratica:

```text
Ti serve l'indice?
  usa for.

Vuoi solo leggere tutti gli elementi?
  usa foreach.
```

Esempio con indice:

```csharp
for (int i = 0; i < nomi.Length; i++)
{
    Console.WriteLine($"{i}: {nomi[i]}");
}
```

Esempio senza indice:

```csharp
foreach (string nome in nomi)
{
    Console.WriteLine(nome);
}
```

---

# `break`

`break` interrompe un ciclo o uno `switch`.

Esempio con ciclo:

```csharp
for (int i = 1; i <= 10; i++)
{
    if (i == 5)
    {
        break;
    }

    Console.WriteLine(i);
}
```

Output:

```text
1
2
3
4
```

Quando `i` vale `5`, il ciclo termina.

---

## Esempio pratico

```csharp
string[] codici = { "A1", "B2", "C3", "D4" };

string codiceDaCercare = "C3";
bool trovato = false;

foreach (string codice in codici)
{
    if (codice == codiceDaCercare)
    {
        trovato = true;
        break;
    }
}

Console.WriteLine(trovato);
```

Appena il codice viene trovato, non ha senso continuare a cercare.

---

# `continue`

`continue` salta il resto dell’iterazione corrente e passa alla successiva.

Esempio:

```csharp
for (int i = 1; i <= 5; i++)
{
    if (i == 3)
    {
        continue;
    }

    Console.WriteLine(i);
}
```

Output:

```text
1
2
4
5
```

Quando `i` vale `3`, il programma salta la `WriteLine` e passa al valore successivo.

---

## Esempio pratico

Stampiamo solo i numeri pari:

```csharp
for (int i = 1; i <= 10; i++)
{
    if (i % 2 != 0)
    {
        continue;
    }

    Console.WriteLine(i);
}
```

Output:

```text
2
4
6
8
10
```

Il ciclo salta i numeri dispari.

# `return`

`return` termina l’esecuzione del metodo corrente.

Nei top-level statements, `return` termina il programma principale.

Esempio:

```csharp
Console.Write("Inserisci un numero: ");

string? input = Console.ReadLine();

if (!int.TryParse(input, out int numero))
{
    Console.WriteLine("Numero non valido.");
    return;
}

Console.WriteLine($"Numero inserito: {numero}");
```

Se la conversione fallisce, il programma stampa il messaggio ed esce subito.

La riga finale non viene eseguita.

---

## return con valore

Quando un metodo restituisce un valore, `return` specifica il valore restituito.

```csharp
int Somma(int a, int b)
{
    return a + b;
}
```

Questo argomento sarà approfondito nel capitolo sui metodi.

Per ora ricordiamo:

```text
return
  esce dal metodo corrente.

return valore
  esce dal metodo corrente restituendo un valore.
```

# Pattern matching

Il **pattern matching** è un insieme di funzionalità che permette di controllare se un valore ha una certa forma.

Non ci limitiamo più a chiedere:

```text
Questo valore è uguale a X?
```

Possiamo chiedere:

```text
Questo valore è di un certo tipo?
Ha certe proprietà?
È dentro un certo intervallo?
Rispetta una certa combinazione logica?
È una lista con una certa struttura?
```

Il pattern matching rende più espressive le condizioni e gli `switch`.

---

# Type pattern

Il **type pattern** verifica se un valore è di un certo tipo e, se lo è, lo assegna a una variabile già tipizzata.

Esempio:

```csharp
object valore = "Ciao";

if (valore is string testo)
{
    Console.WriteLine(testo.ToUpper());
}
```

Qui:

```csharp
valore is string testo
```

significa:

> se `valore` è una stringa, crea una variabile `testo` di tipo `string`.

Altro esempio:

```csharp
object valore = 123;

if (valore is int numero)
{
    Console.WriteLine(numero * 2);
}
```

---

## Type pattern con switch

```csharp
object valore = 12.5m;

string descrizione = valore switch
{
    int => "Numero intero",
    decimal => "Numero decimale",
    string => "Testo",
    _ => "Altro"
};

Console.WriteLine(descrizione);
```

Output:

```text
Numero decimale
```

Se vogliamo usare il valore:

```csharp
object valore = 12.5m;

string descrizione = valore switch
{
    int n => $"Intero: {n}",
    decimal d => $"Decimale: {d}",
    string s => $"Testo: {s}",
    _ => "Altro"
};
```

---

# Property pattern

Il **property pattern** permette di verificare le proprietà di un oggetto.

Supponiamo di avere una classe:

```csharp
public class Cliente
{
    public string Nome { get; set; } = "";
    public int Eta { get; set; }
    public bool Attivo { get; set; }
}
```

Possiamo scrivere:

```csharp
Cliente cliente = new Cliente
{
    Nome = "Emilie",
    Eta = 42,
    Attivo = true
};

if (cliente is { Attivo: true })
{
    Console.WriteLine("Cliente attivo");
}
```

Possiamo controllare più proprietà:

```csharp
if (cliente is { Attivo: true, Eta: >= 18 })
{
    Console.WriteLine("Cliente attivo e maggiorenne");
}
```

---

## Property pattern con switch expression

```csharp
string categoria = cliente switch
{
    { Attivo: false } => "Cliente non attivo",
    { Eta: < 18 } => "Cliente minorenne",
    { Eta: >= 18 } => "Cliente maggiorenne",
    _ => "Cliente non classificato"
};

Console.WriteLine(categoria);
```

Il codice è molto leggibile perché descrive direttamente le forme dei casi.

---

# Relational pattern

Il **relational pattern** permette di usare confronti come pattern.

Esempio:

```csharp
int eta = 20;

string categoria = eta switch
{
    < 18 => "Minorenne",
    >= 18 and < 65 => "Adulto",
    >= 65 => "Senior"
};

Console.WriteLine(categoria);
```

Output:

```text
Adulto
```

I relational pattern sono utili con numeri, date e valori confrontabili.

---

## Esempio con importi

```csharp
decimal totale = 750m;

string fascia = totale switch
{
    < 100m => "Ordine piccolo",
    >= 100m and < 500m => "Ordine medio",
    >= 500m => "Ordine grande"
};

Console.WriteLine(fascia);
```

Output:

```text
Ordine grande
```

# Logical pattern

I **logical pattern** permettono di combinare pattern con:

```text
and
or
not
```

Esempio:

```csharp
int eta = 30;

if (eta is >= 18 and <= 65)
{
    Console.WriteLine("Età lavorativa");
}
```

Esempio con `or`:

```csharp
string codice = "A";

if (codice is "A" or "B" or "C")
{
    Console.WriteLine("Codice valido");
}
```

Esempio con `not`:

```csharp
string? nome = "Emilie";

if (nome is not null)
{
    Console.WriteLine(nome);
}
```

---

## Logical pattern in switch expression

```csharp
int voto = 8;

string giudizio = voto switch
{
    >= 9 and <= 10 => "Ottimo",
    >= 7 and <= 8 => "Buono",
    6 => "Sufficiente",
    >= 0 and < 6 => "Insufficiente",
    _ => "Voto non valido"
};

Console.WriteLine(giudizio);
```

---

# List pattern

I **list pattern** permettono di riconoscere la forma di una sequenza.

Esempio:

```csharp
int[] numeri = { 1, 2, 3 };

string descrizione = numeri switch
{
    [] => "Lista vuota",
    [1, 2, 3] => "Sequenza 1, 2, 3",
    [var primo, ..] => $"Inizia con {primo}"
};

Console.WriteLine(descrizione);
```

Output:

```text
Sequenza 1, 2, 3
```

---

## Alcune forme comuni

Lista vuota:

```csharp
[] => "Vuota"
```

Lista con un solo elemento:

```csharp
[var unico] => $"Un solo elemento: {unico}"
```

Lista che inizia con un valore:

```csharp
[0, ..] => "Inizia con zero"
```

Lista che finisce con un valore:

```csharp
[.., 9] => "Finisce con nove"
```

Lista con primo e ultimo elemento:

```csharp
[var primo, .., var ultimo] => $"Da {primo} a {ultimo}"
```

---

## Esempio pratico con argomenti da riga di comando

```csharp
string messaggio = args switch
{
    [] => "Nessun argomento",
    ["help"] => "Mostra aiuto",
    ["saluta", var nome] => $"Ciao, {nome}!",
    _ => "Comando non riconosciuto"
};

Console.WriteLine(messaggio);
```

Esecuzioni possibili:

```bash
dotnet run
```

Output:

```text
Nessun argomento
```

```bash
dotnet run -- help
```

Output:

```text
Mostra aiuto
```

```bash
dotnet run -- saluta Emilie
```

Output:

```text
Ciao, Emilie!
```

Questo è un esempio molto chiaro di quanto il pattern matching possa rendere leggibile il codice.

# Guard clauses

Le **guard clauses** sono controlli iniziali che escono subito dal metodo o dal blocco quando una condizione non è valida.

Sono molto utili per evitare codice troppo annidato.

Esempio senza guard clause:

```csharp
Console.Write("Inserisci età: ");
string? input = Console.ReadLine();

if (int.TryParse(input, out int eta))
{
    if (eta >= 18)
    {
        Console.WriteLine("Accesso consentito");
    }
    else
    {
        Console.WriteLine("Accesso negato");
    }
}
else
{
    Console.WriteLine("Età non valida");
}
```

Il codice funziona, ma è annidato.

Con guard clause:

```csharp
Console.Write("Inserisci età: ");
string? input = Console.ReadLine();

if (!int.TryParse(input, out int eta))
{
    Console.WriteLine("Età non valida");
    return;
}

if (eta < 18)
{
    Console.WriteLine("Accesso negato");
    return;
}

Console.WriteLine("Accesso consentito");
```

Il flusso è più lineare.

---

## Perché le guard clauses aiutano

Le guard clauses permettono di gestire subito i casi anomali.

```text
Se il dato non è valido, esci.
Se manca qualcosa, esci.
Se l’utente non ha permesso, esci.
Se lo stato non consente l’operazione, esci.

Altrimenti, prosegui con il caso principale.
```

Questo rende il codice più leggibile perché il percorso “normale” non è nascosto dentro molti livelli di `if`.

---

## Esempio con metodo

```csharp
decimal CalcolaSconto(decimal importo, decimal percentuale)
{
    if (importo < 0)
    {
        throw new ArgumentException("L'importo non può essere negativo.", nameof(importo));
    }

    if (percentuale < 0 || percentuale > 100)
    {
        throw new ArgumentException("La percentuale deve essere tra 0 e 100.", nameof(percentuale));
    }

    return importo * percentuale / 100m;
}
```

Qui i casi non validi vengono respinti subito.

La formula principale resta alla fine, chiara e isolata.

# Codice difensivo

Il **codice difensivo** è codice scritto tenendo conto del fatto che qualcosa può andare storto.

Non significa scrivere codice pessimista.
Significa scrivere codice robusto.

Un programma non dovrebbe fidarsi ciecamente di:

* input dell’utente;
* dati provenienti da file;
* dati provenienti da database;
* dati ricevuti da API;
* configurazioni;
* valori null;
* stati applicativi non previsti.

---

## Esempio non difensivo

```csharp
Console.Write("Età: ");
int eta = int.Parse(Console.ReadLine()!);

Console.WriteLine($"Età: {eta}");
```

Se l’utente scrive `ciao`, il programma va in errore.

---

## Esempio difensivo

```csharp
Console.Write("Età: ");
string? input = Console.ReadLine();

if (!int.TryParse(input, out int eta))
{
    Console.WriteLine("Età non valida.");
    return;
}

if (eta < 0)
{
    Console.WriteLine("L'età non può essere negativa.");
    return;
}

Console.WriteLine($"Età: {eta}");
```

Questo codice gestisce:

* input non numerico;
* valore negativo;
* caso valido.

---

## Validare i confini

Il codice difensivo controlla i confini dei dati.

Esempio:

```csharp
if (quantita <= 0)
{
    Console.WriteLine("La quantità deve essere maggiore di zero.");
    return;
}
```

Esempio:

```csharp
if (dataScadenza < DateOnly.FromDateTime(DateTime.Today))
{
    Console.WriteLine("La data di scadenza è già passata.");
    return;
}
```

Esempio:

```csharp
if (string.IsNullOrWhiteSpace(codiceCliente))
{
    Console.WriteLine("Codice cliente obbligatorio.");
    return;
}
```

---

## Difensivo non significa silenzioso

Un errore non va sempre nascosto.

Questo codice può essere pericoloso:

```csharp
try
{
    // operazione
}
catch
{
    // ignora tutto
}
```

Ignorare gli errori può rendere il programma apparentemente stabile, ma in realtà più difficile da correggere.

Il codice difensivo deve:

* prevenire errori prevedibili;
* segnalare chiaramente stati non validi;
* fallire presto quando necessario;
* non nascondere bug logici.

---

# Esempio completo del capitolo

Costruiamo un piccolo programma che legge un comando da console.

Il programma supporta:

```text
saluta <nome>
somma <numero1> <numero2>
help
esci
```

Codice:

```csharp
Console.WriteLine("Mini console comandi");
Console.WriteLine("Scrivi help per l'elenco dei comandi.");

while (true)
{
    Console.Write("> ");
    string? riga = Console.ReadLine();

    if (string.IsNullOrWhiteSpace(riga))
    {
        continue;
    }

    string[] parti = riga.Split(' ', StringSplitOptions.RemoveEmptyEntries);

    string risposta = parti switch
    {
        ["help"] =>
            """
            Comandi disponibili:
              help
              saluta <nome>
              somma <numero1> <numero2>
              esci
            """,

        ["saluta", var nome] =>
            $"Ciao, {nome}!",

        ["somma", var primo, var secondo]
            when int.TryParse(primo, out int a) && int.TryParse(secondo, out int b) =>
            $"Risultato: {a + b}",

        ["somma", ..] =>
            "Uso corretto: somma <numero1> <numero2>",

        ["esci"] =>
            "USCITA",

        _ =>
            "Comando non riconosciuto."
    };

    if (risposta == "USCITA")
    {
        break;
    }

    Console.WriteLine(risposta);
}
```

Questo esempio usa:

* `while`;
* `continue`;
* `break`;
* `switch expression`;
* list pattern;
* guard implicita su input vuoto;
* `TryParse`;
* pattern con `when`;
* stringa multilinea.

È ancora un programma semplice, ma mostra una caratteristica importante: con gli strumenti di controllo del flusso possiamo costruire programmi interattivi e decisionali.

# Errori comuni

## Annidare troppi if

Codice difficile da leggere:

```csharp
if (cliente != null)
{
    if (cliente.Attivo)
    {
        if (cliente.Eta >= 18)
        {
            Console.WriteLine("Accesso consentito");
        }
    }
}
```

Meglio usare condizioni combinate o guard clauses:

```csharp
if (cliente is null)
{
    return;
}

if (!cliente.Attivo)
{
    return;
}

if (cliente.Eta < 18)
{
    return;
}

Console.WriteLine("Accesso consentito");
```

---

## Dimenticare di aggiornare la variabile in un while

Errore:

```csharp
int i = 0;

while (i < 10)
{
    Console.WriteLine(i);
}
```

`i` non cambia mai. Il ciclo è infinito.

Corretto:

```csharp
int i = 0;

while (i < 10)
{
    Console.WriteLine(i);
    i++;
}
```

---

## Usare for quando foreach sarebbe più chiaro

Meno leggibile:

```csharp
for (int i = 0; i < nomi.Length; i++)
{
    Console.WriteLine(nomi[i]);
}
```

Più leggibile se non serve l’indice:

```csharp
foreach (string nome in nomi)
{
    Console.WriteLine(nome);
}
```

---

## Usare break e continue in modo eccessivo

`break` e `continue` sono utili, ma se usati troppo possono rendere il flusso difficile da seguire.

Esempio sospetto:

```csharp
foreach (var elemento in elementi)
{
    if (condizione1)
    {
        continue;
    }

    if (condizione2)
    {
        break;
    }

    if (condizione3)
    {
        continue;
    }

    // altra logica
}
```

In questi casi può essere utile estrarre metodi o semplificare la logica.

---

## Usare switch expression per logiche troppo complesse

Le switch expression sono ottime per calcolare valori.

Meno adatte per blocchi lunghi con effetti collaterali.

Buono:

```csharp
string descrizione = stato switch
{
    "A" => "Attivo",
    "S" => "Sospeso",
    _ => "Sconosciuto"
};
```

Meno buono:

```csharp
string risultato = stato switch
{
    "A" => EseguiMolteOperazioniComplesse(),
    "S" => EseguiAltreOperazioniComplesse(),
    _ => GestisciErroreComplicato()
};
```

Quando la logica cresce, spesso è meglio usare metodi separati o uno `switch` classico.

---

## Usare pattern matching dove basta una condizione semplice

Il pattern matching è potente, ma non bisogna usarlo per rendere complesso ciò che è semplice.

Sufficiente:

```csharp
if (eta >= 18)
{
    Console.WriteLine("Maggiorenne");
}
```

Inutile complicare:

```csharp
if (eta is >= 18)
{
    Console.WriteLine("Maggiorenne");
}
```

La seconda forma è valida, ma non sempre necessaria.
