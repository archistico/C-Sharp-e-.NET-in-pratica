# 02.07 — Struct, record, enum e tipi immutabili

Nel capitolo precedente abbiamo introdotto classi, oggetti, proprietà, metodi, costruttori e incapsulamento. Abbiamo visto che una classe non dovrebbe essere soltanto un contenitore di dati, ma dovrebbe proteggere le proprie regole interne.

Ora allarghiamo il discorso.

In C# non esistono solo le classi. Il linguaggio offre altri strumenti fondamentali per modellare bene i dati:

* `struct`;
* `enum`;
* `record class`;
* `record struct`;
* tipi immutabili;
* proprietà `init`;
* espressioni `with`.

Questi strumenti sono molto importanti quando costruiamo:

* DTO;
* modelli per API;
* oggetti di dominio;
* value object;
* risultati di operazioni;
* dati letti da database;
* messaggi scambiati tra sistemi;
* configurazioni;
* oggetti piccoli e significativi.

Il punto centrale del capitolo è questo:

> Non tutti i dati devono essere modellati con una classe mutabile.

A volte serve una classe.
A volte serve una `struct`.
A volte serve un `record`.
A volte basta un `enum`.

La qualità di un’applicazione dipende anche da queste scelte.

---

# Value type e reference type

Prima di parlare di `struct` e `record`, dobbiamo chiarire una distinzione fondamentale in C#:

```text id="96k0hh"
value type
reference type
```

In C# alcuni tipi si comportano come **valori**, altri come **riferimenti**.

---

## Reference type

Le classi sono reference type.

Esempio:

```csharp id="zmaurl"
public class Cliente
{
    public string Nome { get; set; } = "";
}
```

Uso:

```csharp id="3v60po"
Cliente cliente1 = new Cliente { Nome = "Emilie" };

Cliente cliente2 = cliente1;

cliente2.Nome = "Mario";

Console.WriteLine(cliente1.Nome);
Console.WriteLine(cliente2.Nome);
```

Output:

```text id="lq8a89"
Mario
Mario
```

Perché?

Perché `cliente1` e `cliente2` non contengono due oggetti separati. Contengono due riferimenti allo stesso oggetto.

Possiamo immaginarlo così:

```text id="dpf7uy"
cliente1 ─┐
          ├──> oggetto Cliente
cliente2 ─┘
```

Quando modifichiamo l’oggetto tramite `cliente2`, vediamo la modifica anche tramite `cliente1`.

---

## Value type

Tipi come `int`, `decimal`, `bool`, `DateTime` sono value type.

Esempio:

```csharp id="diyb6p"
int numero1 = 10;

int numero2 = numero1;

numero2 = 20;

Console.WriteLine(numero1);
Console.WriteLine(numero2);
```

Output:

```text id="gqnnwf"
10
20
```

Qui `numero2` riceve una copia del valore di `numero1`.

Le due variabili sono indipendenti.

---

## Differenza pratica

```text id="qsrbdk"
Reference type
  la variabile contiene un riferimento a un oggetto.

Value type
  la variabile contiene direttamente il valore.
```

Esempio con classe:

```csharp id="qa5zci"
Cliente a = new Cliente { Nome = "Emilie" };
Cliente b = a;

b.Nome = "Mario";

Console.WriteLine(a.Nome); // Mario
```

Esempio con `int`:

```csharp id="8kfouy"
int a = 10;
int b = a;

b = 20;

Console.WriteLine(a); // 10
```

Questa differenza influenza:

* assegnazioni;
* passaggio ai metodi;
* uguaglianza;
* mutabilità;
* performance;
* modellazione.

---

# `struct`

Una `struct` è un tipo valore definito dal programmatore.

Esempio:

```csharp id="3a6vkv"
public struct Punto
{
    public int X { get; set; }
    public int Y { get; set; }
}
```

Uso:

```csharp id="d9v3uq"
Punto punto1 = new Punto { X = 10, Y = 20 };

Punto punto2 = punto1;

punto2.X = 99;

Console.WriteLine(punto1.X);
Console.WriteLine(punto2.X);
```

Output:

```text id="2llkn3"
10
99
```

`punto2` è una copia di `punto1`.

Questo comportamento è diverso da quello di una classe.

---

## Struct con costruttore

Possiamo definire una `struct` con costruttore.

```csharp id="o5y3tc"
public struct Punto
{
    public int X { get; }
    public int Y { get; }

    public Punto(int x, int y)
    {
        X = x;
        Y = y;
    }
}
```

Uso:

```csharp id="sdyjjz"
Punto punto = new Punto(10, 20);

Console.WriteLine(punto.X);
Console.WriteLine(punto.Y);
```

---

## Struct immutabile

Le `struct` dovrebbero spesso essere immutabili.

Una `struct` mutabile può creare comportamenti sorprendenti, soprattutto quando viene copiata.

Meglio:

```csharp id="13uavn"
public readonly struct Punto
{
    public int X { get; }
    public int Y { get; }

    public Punto(int x, int y)
    {
        X = x;
        Y = y;
    }
}
```

`readonly struct` comunica chiaramente che il valore non deve cambiare dopo la creazione.

---

## Quando usare una struct

Una `struct` è adatta quando il tipo rappresenta un piccolo valore.

Esempi tipici:

* punto;
* coordinata;
* intervallo breve;
* quantità;
* valore monetario;
* percentuale;
* codice composto semplice;
* misura.

Esempio:

```csharp id="q09swt"
public readonly struct Percentuale
{
    public decimal Valore { get; }

    public Percentuale(decimal valore)
    {
        if (valore < 0 || valore > 100)
        {
            throw new ArgumentOutOfRangeException(nameof(valore), "La percentuale deve essere tra 0 e 100.");
        }

        Valore = valore;
    }
}
```

Uso:

```csharp id="lt2qil"
Percentuale sconto = new Percentuale(20m);

Console.WriteLine(sconto.Valore);
```

Qui `Percentuale` è più espressivo di un semplice `decimal`.

---

## Quando evitare una struct

Evita `struct` quando:

* l’oggetto è grande;
* deve essere modificabile;
* ha identità propria;
* viene passato spesso in giro e copiato molto;
* contiene molte proprietà;
* rappresenta un’entità del dominio;
* deve partecipare a gerarchie di ereditarietà.

Esempio da evitare:

```csharp id="lb8pk6"
public struct Cliente
{
    public int Id { get; set; }
    public string Nome { get; set; }
    public string Cognome { get; set; }
    public DateOnly DataNascita { get; set; }
}
```

Un cliente non è un piccolo valore. È un’entità. Meglio una classe.

---

# `enum`

Un `enum` rappresenta un insieme chiuso di valori nominati.

Esempio:

```csharp id="6frk7l"
public enum StatoCliente
{
    Attivo,
    Sospeso,
    Cancellato
}
```

Uso:

```csharp id="nbh7nl"
StatoCliente stato = StatoCliente.Attivo;

if (stato == StatoCliente.Attivo)
{
    Console.WriteLine("Cliente attivo");
}
```

Un `enum` rende il codice più leggibile rispetto a stringhe o numeri.

---

## Enum invece di stringhe

Sconsigliato:

```csharp id="a7j4k1"
string stato = "A";

if (stato == "A")
{
    Console.WriteLine("Attivo");
}
```

Meglio:

```csharp id="t0sq8r"
StatoCliente stato = StatoCliente.Attivo;

if (stato == StatoCliente.Attivo)
{
    Console.WriteLine("Attivo");
}
```

Il secondo codice è più chiaro e meno fragile.

---

## Enum con valori espliciti

Possiamo assegnare valori numerici espliciti.

```csharp id="2q2h44"
public enum StatoDocumento
{
    Bozza = 0,
    Confermato = 1,
    Annullato = 2
}
```

Questo può essere utile quando il valore viene salvato in un database o comunicato a sistemi esterni.

---

## Enum e switch

Gli `enum` si combinano molto bene con `switch`.

```csharp id="emexg9"
public enum StatoOrdine
{
    Bozza,
    Confermato,
    Spedito,
    Annullato
}
```

Uso:

```csharp id="xsbz3m"
string descrizione = stato switch
{
    StatoOrdine.Bozza => "Ordine in bozza",
    StatoOrdine.Confermato => "Ordine confermato",
    StatoOrdine.Spedito => "Ordine spedito",
    StatoOrdine.Annullato => "Ordine annullato",
    _ => "Stato sconosciuto"
};
```

---

## Attenzione ai valori non validi

Un `enum` è basato su un numero intero. Questo significa che possiamo creare valori non definiti.

Esempio:

```csharp id="naileb"
StatoCliente stato = (StatoCliente)999;

Console.WriteLine(stato);
```

Il codice compila.

Per verificare se un valore è definito:

```csharp id="mk4hq4"
if (!Enum.IsDefined(typeof(StatoCliente), stato))
{
    Console.WriteLine("Stato non valido.");
}
```

Nei confini del sistema, per esempio leggendo da database o API esterne, è bene validare i valori enum.

---

## Enum con Flags

A volte un `enum` rappresenta una combinazione di opzioni.

Esempio:

```csharp id="n9kw7g"
[Flags]
public enum Permessi
{
    Nessuno = 0,
    Lettura = 1,
    Scrittura = 2,
    Cancellazione = 4
}
```

Uso:

```csharp id="i4k5ko"
Permessi permessi = Permessi.Lettura | Permessi.Scrittura;

bool puòScrivere = permessi.HasFlag(Permessi.Scrittura);

Console.WriteLine(puòScrivere);
```

Output:

```text id="g7dd8b"
True
```

`[Flags]` va usato solo quando i valori possono essere combinati.

---

# Record

I `record` sono tipi pensati per rappresentare dati.

Sono utili quando vogliamo:

* immutabilità;
* uguaglianza per valore;
* sintassi compatta;
* oggetti DTO;
* risultati di operazioni;
* messaggi;
* value object.

Esistono due forme principali:

```text id="0bfwrt"
record class
record struct
```

Quando scriviamo semplicemente:

```csharp id="sq90kw"
public record ClienteDto(string Nome, string Cognome);
```

stiamo creando una `record class`.

---

# `record class`

Una `record class` è un reference type, come una normale classe, ma con semantica orientata ai dati.

Esempio:

```csharp id="s2e6cb"
public record ClienteDto(string Nome, string Cognome);
```

Uso:

```csharp id="qtlza6"
ClienteDto cliente = new ClienteDto("Emilie", "Rollandin");

Console.WriteLine(cliente.Nome);
Console.WriteLine(cliente.Cognome);
```

Output:

```text id="0j17yt"
Emilie
Rollandin
```

Questa sintassi è molto compatta.

È simile a scrivere una classe con:

* proprietà;
* costruttore;
* uguaglianza per valore;
* `ToString`;
* supporto a `with`.

---

## Uguaglianza per valore

La caratteristica più importante dei record è l’uguaglianza per valore.

Esempio con classe normale:

```csharp id="cw7rfk"
public class ClienteClass
{
    public string Nome { get; set; } = "";
    public string Cognome { get; set; } = "";
}
```

Uso:

```csharp id="u3fwwz"
ClienteClass c1 = new ClienteClass { Nome = "Emilie", Cognome = "Rollandin" };
ClienteClass c2 = new ClienteClass { Nome = "Emilie", Cognome = "Rollandin" };

Console.WriteLine(c1 == c2);
```

Output:

```text id="shbhpm"
False
```

Sono due oggetti diversi.

Ora con record:

```csharp id="6qlgkm"
public record ClienteRecord(string Nome, string Cognome);
```

Uso:

```csharp id="o5qocx"
ClienteRecord c1 = new ClienteRecord("Emilie", "Rollandin");
ClienteRecord c2 = new ClienteRecord("Emilie", "Rollandin");

Console.WriteLine(c1 == c2);
```

Output:

```text id="4qeqae"
True
```

I due record hanno gli stessi valori, quindi sono considerati uguali.

---

## ToString nei record

I record generano anche un `ToString` utile.

```csharp id="msh7ma"
ClienteRecord cliente = new ClienteRecord("Emilie", "Rollandin");

Console.WriteLine(cliente);
```

Output:

```text id="2gukex"
ClienteRecord { Nome = Emilie, Cognome = Rollandin }
```

Questo è molto comodo durante debug e logging semplice.

---

# Immutabilità

Un oggetto è **immutabile** quando, dopo la creazione, non può essere modificato.

Esempio:

```csharp id="69uo4k"
public record ClienteDto(string Nome, string Cognome);
```

Uso:

```csharp id="sltc6n"
ClienteDto cliente = new ClienteDto("Emilie", "Rollandin");

// cliente.Nome = "Mario"; // errore
```

Le proprietà generate da un record posizionale sono inizializzabili, ma non modificabili liberamente dopo la creazione.

L’immutabilità è molto utile perché:

* rende gli oggetti più prevedibili;
* riduce bug da modifiche accidentali;
* semplifica il ragionamento;
* aiuta nei sistemi concorrenti;
* rende più sicuri DTO e messaggi;
* favorisce uno stile più dichiarativo.

---

## Tipo mutabile

```csharp id="tx2emd"
public class Cliente
{
    public string Nome { get; set; } = "";
    public string Cognome { get; set; } = "";
}
```

Uso:

```csharp id="4mu0ni"
Cliente cliente = new Cliente
{
    Nome = "Emilie",
    Cognome = "Rollandin"
};

cliente.Nome = "Mario";
```

Il valore cambia.

---

## Tipo immutabile

```csharp id="knnkwd"
public record ClienteDto(string Nome, string Cognome);
```

Uso:

```csharp id="hdqmbq"
ClienteDto cliente = new ClienteDto("Emilie", "Rollandin");

// cliente.Nome = "Mario"; // non consentito
```

Per cambiare un valore, creiamo una copia modificata con `with`.

---

# `with`

L’espressione `with` permette di creare una copia di un record cambiando alcuni valori.

Esempio:

```csharp id="mrqys1"
public record ClienteDto(string Nome, string Cognome, string Citta);
```

Uso:

```csharp id="1tsp9p"
ClienteDto cliente = new ClienteDto("Emilie", "Rollandin", "Aosta");

ClienteDto clienteAggiornato = cliente with
{
    Citta = "Torino"
};

Console.WriteLine(cliente);
Console.WriteLine(clienteAggiornato);
```

Output:

```text id="n4f0kl"
ClienteDto { Nome = Emilie, Cognome = Rollandin, Citta = Aosta }
ClienteDto { Nome = Emilie, Cognome = Rollandin, Citta = Torino }
```

L’oggetto originale non viene modificato.

Viene creato un nuovo oggetto con alcuni valori cambiati.

---

## Perché with è utile

Senza `with`, dovremmo riscrivere tutti i valori:

```csharp id="3ungth"
ClienteDto clienteAggiornato = new ClienteDto(
    cliente.Nome,
    cliente.Cognome,
    "Torino");
```

Con `with`:

```csharp id="unq28l"
ClienteDto clienteAggiornato = cliente with
{
    Citta = "Torino"
};
```

Più leggibile e meno fragile.

---

# `init`

La keyword `init` permette di impostare una proprietà durante l’inizializzazione, ma non modificarla dopo.

Esempio:

```csharp id="gqzwd4"
public class ClienteDto
{
    public string Nome { get; init; } = "";
    public string Cognome { get; init; } = "";
}
```

Uso:

```csharp id="8lxr4o"
ClienteDto cliente = new ClienteDto
{
    Nome = "Emilie",
    Cognome = "Rollandin"
};
```

Dopo l’inizializzazione:

```csharp id="tvtyd5"
cliente.Nome = "Mario"; // errore
```

---

## init in una classe

`init` permette di creare classi più immutabili senza obbligare sempre a scrivere un costruttore.

```csharp id="qa3q80"
public class DocumentoDto
{
    public required int Id { get; init; }
    public required string Numero { get; init; }
    public required DateOnly Data { get; init; }
}
```

Uso:

```csharp id="1ket53"
DocumentoDto documento = new DocumentoDto
{
    Id = 1,
    Numero = "DOC-001",
    Data = new DateOnly(2026, 4, 24)
};
```

Non possiamo cambiare dopo:

```csharp id="xqtiq8"
documento.Numero = "DOC-002"; // errore
```

---

## `set` vs `init`

```text id="96ttev"
set
  la proprietà può essere modificata anche dopo la creazione.

init
  la proprietà può essere impostata solo durante l’inizializzazione.
```

Esempio:

```csharp id="ofe4nm"
public string Nome { get; set; } = "";
```

Modificabile sempre.

```csharp id="5u8sn5"
public string Nome { get; init; } = "";
```

Impostabile solo alla creazione.

---

# `record struct`

Un `record struct` combina alcune caratteristiche dei record con il comportamento da value type.

Esempio:

```csharp id="3v848y"
public readonly record struct Punto(int X, int Y);
```

Uso:

```csharp id="itjtt3"
Punto p1 = new Punto(10, 20);
Punto p2 = new Punto(10, 20);

Console.WriteLine(p1 == p2);
```

Output:

```text id="dw1u2x"
True
```

Essendo una struct, `Punto` è un value type.

Essendo un record, ha uguaglianza per valore e una sintassi compatta.

---

## Record struct mutabile e readonly

Possiamo scrivere:

```csharp id="402y6u"
public record struct Punto(int X, int Y);
```

Oppure:

```csharp id="a4n8z5"
public readonly record struct Punto(int X, int Y);
```

La seconda forma è spesso preferibile per piccoli valori immutabili.

---

## Esempio: valore monetario

```csharp id="ecy5fe"
public readonly record struct Importo(decimal Valore, string Valuta);
```

Uso:

```csharp id="3uv5wz"
Importo prezzo1 = new Importo(10m, "EUR");
Importo prezzo2 = new Importo(10m, "EUR");

Console.WriteLine(prezzo1 == prezzo2);
```

Output:

```text id="11s2vk"
True
```

Questo può essere più espressivo di usare solo `decimal`.

Naturalmente, per un tipo monetario reale servirebbero più regole, per esempio validare valuta e arrotondamenti.

---

# Uguaglianza per valore

L’uguaglianza è uno dei motivi principali per usare record e record struct.

Con una classe normale, due oggetti sono uguali solo se sono lo stesso oggetto, salvo override manuale.

Con un record, due oggetti sono uguali se i loro valori sono uguali.

---

## Classe normale

```csharp id="7qccwa"
public class ProdottoClass
{
    public string Codice { get; set; } = "";
}
```

Uso:

```csharp id="fv2wgy"
var p1 = new ProdottoClass { Codice = "ABC" };
var p2 = new ProdottoClass { Codice = "ABC" };

Console.WriteLine(p1 == p2);
```

Output:

```text id="fgbufq"
False
```

---

## Record

```csharp id="iuwssg"
public record ProdottoRecord(string Codice);
```

Uso:

```csharp id="qvcxh8"
var p1 = new ProdottoRecord("ABC");
var p2 = new ProdottoRecord("ABC");

Console.WriteLine(p1 == p2);
```

Output:

```text id="qlikn9"
True
```

Questo è ideale per oggetti che rappresentano dati, non identità autonoma.

---

# Tuple vs record

Le tuple e i record possono sembrare simili perché entrambi permettono di raggruppare più valori.

Tuple:

```csharp id="6n1c5d"
(string Nome, string Cognome) cliente = ("Emilie", "Rollandin");
```

Record:

```csharp id="wm4f0u"
public record ClienteDto(string Nome, string Cognome);

ClienteDto cliente = new ClienteDto("Emilie", "Rollandin");
```

La differenza è nel significato.

---

## Tuple

Le tuple sono utili per risultati piccoli e locali.

Esempio:

```csharp id="qtxg26"
(decimal Imponibile, decimal Iva, decimal Totale) CalcolaTotale(decimal importo)
{
    decimal iva = importo * 22m / 100m;
    return (importo, iva, importo + iva);
}
```

Uso:

```csharp id="ia12ri"
var risultato = CalcolaTotale(100m);

Console.WriteLine(risultato.Totale);
```

Qui la tupla va bene perché il risultato è locale e semplice.

---

## Record

Un record è più adatto quando il dato ha un nome e un significato nel progetto.

```csharp id="vg2cbc"
public record TotaleDocumento(decimal Imponibile, decimal Iva, decimal Totale);
```

Uso:

```csharp id="i0q1bf"
TotaleDocumento CalcolaTotale(decimal importo)
{
    decimal iva = importo * 22m / 100m;
    return new TotaleDocumento(importo, iva, importo + iva);
}
```

Ora `TotaleDocumento` è un concetto esplicito del codice.

---

## Regola pratica

```text id="igw7f9"
Tuple
  per risultati temporanei, locali e semplici.

Record
  per dati che hanno significato nel dominio o nell’API.
```

Esempio:

```text id="0vdyl9"
Restituire due valori da un metodo privato
  tuple.

Rappresentare la risposta di una API
  record.

Rappresentare un DTO
  record.

Rappresentare un concetto del dominio
  spesso record o classe, a seconda delle regole.
```

---

# Quando usare una classe

Usa una classe quando il tipo rappresenta un’entità con identità, comportamento o stato che cambia nel tempo.

Esempi:

* cliente;
* ordine;
* documento;
* carrello;
* conto corrente;
* prenotazione;
* utente;
* sessione;
* servizio applicativo.

Esempio:

```csharp id="uc068q"
public class Ordine
{
    private readonly List<RigaDocumento> righe = new();

    public int Id { get; }
    public bool Confermato { get; private set; }
    public IReadOnlyList<RigaDocumento> Righe => righe;

    public Ordine(int id)
    {
        Id = id;
    }

    public void AggiungiRiga(RigaDocumento riga)
    {
        if (Confermato)
        {
            throw new InvalidOperationException("Ordine già confermato.");
        }

        righe.Add(riga);
    }

    public void Conferma()
    {
        if (righe.Count == 0)
        {
            throw new InvalidOperationException("Ordine senza righe.");
        }

        Confermato = true;
    }
}
```

Qui una classe è corretta perché `Ordine` ha:

* identità;
* stato;
* comportamento;
* regole;
* ciclo di vita.

---

# Quando usare una struct

Usa una `struct` quando rappresenti un valore piccolo, semplice e preferibilmente immutabile.

Esempi:

* coordinate;
* intervalli;
* misure;
* percentuali;
* codici piccoli;
* valori numerici con significato.

Esempio:

```csharp id="lzc31q"
public readonly struct Percentuale
{
    public decimal Valore { get; }

    public Percentuale(decimal valore)
    {
        if (valore < 0 || valore > 100)
        {
            throw new ArgumentOutOfRangeException(nameof(valore));
        }

        Valore = valore;
    }

    public decimal ApplicaA(decimal importo)
    {
        return importo * Valore / 100m;
    }
}
```

Uso:

```csharp id="9qunnb"
Percentuale sconto = new Percentuale(20m);

decimal valoreSconto = sconto.ApplicaA(100m);

Console.WriteLine(valoreSconto);
```

Output:

```text id="79vwyq"
20
```

# Quando usare un record

Usa un record quando vuoi rappresentare dati con uguaglianza per valore, spesso immutabili.

Esempi:

* DTO;
* risposte API;
* comandi;
* query;
* messaggi;
* configurazioni;
* value object semplici;
* risultati di operazioni.

Esempio DTO:

```csharp id="zx902w"
public record ClienteDto(int Id, string Nome, string Cognome);
```

Esempio risultato:

```csharp id="p8orn6"
public record EsitoOperazione(bool Successo, string? Messaggio);
```

Esempio value object:

```csharp id="g2mi5p"
public record CodiceCliente(string Valore);
```

Con validazione:

```csharp id="gv18is"
public record CodiceCliente
{
    public string Valore { get; }

    public CodiceCliente(string valore)
    {
        if (string.IsNullOrWhiteSpace(valore))
        {
            throw new ArgumentException("Il codice cliente è obbligatorio.", nameof(valore));
        }

        Valore = valore;
    }
}
```

# Classe, struct o record?

La scelta può essere guidata da alcune domande.

## Il tipo ha identità?

Esempio:

```text id="v716dg"
Cliente con Id 123
Ordine numero 456
Documento protocollo 789
```

Se sì, spesso è una classe.

---

## Il tipo rappresenta un piccolo valore?

Esempio:

```text id="rv4yo6"
Percentuale
Punto
Coordinate
Intervallo
```

Se sì, può essere una struct o record struct.

---

## Il tipo è soprattutto un contenitore dati immutabile?

Esempio:

```text id="a76lbh"
ClienteDto
RispostaApi
ComandoCreaOrdine
ConfigurazioneEmail
```

Se sì, spesso è un record.

---

## Il tipo deve cambiare stato nel tempo?

Esempio:

```text id="acahzk"
Ordine che passa da Bozza a Confermato
Carrello a cui aggiungo righe
Conto da cui prelevo denaro
```

Se sì, spesso è una classe.

---

## Voglio uguaglianza per valore?

Esempio:

```csharp id="we9zm2"
new CodiceCliente("ABC") == new CodiceCliente("ABC")
```

Se vuoi che questo sia vero, un record è spesso una buona scelta.

# Esempio completo del capitolo

Costruiamo un piccolo modello per rappresentare un documento.

Useremo:

* `enum` per lo stato;
* `record` per un DTO;
* `record struct` per una percentuale;
* classe per il documento con comportamento.

---

## Enum StatoDocumento

```csharp id="aqyjvf"
public enum StatoDocumento
{
    Bozza,
    Confermato,
    Annullato
}
```

---

## Record struct Percentuale

```csharp id="ac482u"
public readonly record struct Percentuale
{
    public decimal Valore { get; }

    public Percentuale(decimal valore)
    {
        if (valore < 0 || valore > 100)
        {
            throw new ArgumentOutOfRangeException(nameof(valore), "La percentuale deve essere tra 0 e 100.");
        }

        Valore = valore;
    }

    public decimal ApplicaA(decimal importo)
    {
        return importo * Valore / 100m;
    }

    public override string ToString()
    {
        return $"{Valore}%";
    }
}
```

---

## Record RigaDocumentoDto

```csharp id="o2plov"
public record RigaDocumentoDto(
    string Descrizione,
    int Quantita,
    decimal PrezzoUnitario,
    Percentuale Sconto);
```

Questo tipo è adatto a trasferire dati, per esempio verso una API o una UI.

---

## Classe Documento

```csharp id="6rprw6"
public class Documento
{
    private readonly List<RigaDocumentoDto> righe = new();

    public int Id { get; }
    public StatoDocumento Stato { get; private set; } = StatoDocumento.Bozza;
    public IReadOnlyList<RigaDocumentoDto> Righe => righe;

    public Documento(int id)
    {
        if (id <= 0)
        {
            throw new ArgumentOutOfRangeException(nameof(id), "L'ID deve essere positivo.");
        }

        Id = id;
    }

    public void AggiungiRiga(RigaDocumentoDto riga)
    {
        if (Stato != StatoDocumento.Bozza)
        {
            throw new InvalidOperationException("Puoi modificare solo documenti in bozza.");
        }

        if (string.IsNullOrWhiteSpace(riga.Descrizione))
        {
            throw new ArgumentException("La descrizione è obbligatoria.");
        }

        if (riga.Quantita <= 0)
        {
            throw new ArgumentOutOfRangeException(nameof(riga.Quantita), "La quantità deve essere positiva.");
        }

        if (riga.PrezzoUnitario < 0)
        {
            throw new ArgumentOutOfRangeException(nameof(riga.PrezzoUnitario), "Il prezzo non può essere negativo.");
        }

        righe.Add(riga);
    }

    public decimal CalcolaTotale()
    {
        decimal totale = 0m;

        foreach (RigaDocumentoDto riga in righe)
        {
            decimal lordo = riga.Quantita * riga.PrezzoUnitario;
            decimal sconto = riga.Sconto.ApplicaA(lordo);

            totale += lordo - sconto;
        }

        return totale;
    }

    public void Conferma()
    {
        if (righe.Count == 0)
        {
            throw new InvalidOperationException("Non puoi confermare un documento senza righe.");
        }

        Stato = StatoDocumento.Confermato;
    }
}
```

---

## Uso

```csharp id="s4x42l"
Documento documento = new Documento(1);

documento.AggiungiRiga(new RigaDocumentoDto(
    Descrizione: "Libro C#",
    Quantita: 2,
    PrezzoUnitario: 30m,
    Sconto: new Percentuale(10m)));

documento.AggiungiRiga(new RigaDocumentoDto(
    Descrizione: "Corso .NET",
    Quantita: 1,
    PrezzoUnitario: 100m,
    Sconto: new Percentuale(0m)));

Console.WriteLine($"Totale: {documento.CalcolaTotale()}");

documento.Conferma();

Console.WriteLine($"Stato: {documento.Stato}");
```

Output:

```text id="b1ghkx"
Totale: 154
Stato: Confermato
```

In questo esempio:

```text id="ze6y4u"
StatoDocumento
  è un enum, perché rappresenta un insieme chiuso di stati.

Percentuale
  è un record struct, perché rappresenta un piccolo valore.

RigaDocumentoDto
  è un record, perché rappresenta dati trasferibili.

Documento
  è una classe, perché ha identità, stato e comportamento.
```

# Errori comuni

## Usare stringhe al posto degli enum

Sconsigliato:

```csharp id="yey74w"
string stato = "CONFERMATO";
```

Meglio:

```csharp id="r63p5p"
StatoDocumento stato = StatoDocumento.Confermato;
```

Gli enum riducono errori di battitura e rendono il codice più chiaro.

---

## Usare struct per entità grandi

Sconsigliato:

```csharp id="awuxw2"
public struct Cliente
{
    public int Id { get; set; }
    public string Nome { get; set; }
}
```

Meglio:

```csharp id="p6s0we"
public class Cliente
{
    public int Id { get; }
    public string Nome { get; }

    public Cliente(int id, string nome)
    {
        Id = id;
        Nome = nome;
    }
}
```

Un cliente ha identità. Non è un semplice valore.

---

## Usare classi mutabili per DTO semplici

Possibile, ma spesso meno ideale:

```csharp id="8qsypw"
public class ClienteDto
{
    public int Id { get; set; }
    public string Nome { get; set; } = "";
}
```

Spesso meglio:

```csharp id="hfg7yi"
public record ClienteDto(int Id, string Nome);
```

Più compatto, più espressivo, più adatto ai dati.

---

## Pensare che record significhi sempre immutabile

Un record può essere reso mutabile.

```csharp id="tevm0e"
public record ClienteDto
{
    public string Nome { get; set; } = "";
}
```

Questo è un record, ma la proprietà è modificabile.

Se vuoi immutabilità, usa proprietà solo `init`, proprietà get-only o record posizionali.

---

## Usare tuple per concetti importanti

Sconsigliato:

```csharp id="twk4of"
(string Nome, string Cognome, DateOnly DataNascita) cliente;
```

Se questo dato circola nel progetto, meglio:

```csharp id="yebku9"
public record ClienteDto(string Nome, string Cognome, DateOnly DataNascita);
```

Le tuple sono ottime localmente, ma i record danno un nome al concetto.

---

## Dimenticare che i record class sono reference type

```csharp id="qhkw6p"
public record ClienteDto(string Nome);
```

Questo è un reference type.

Può essere `null`:

```csharp id="z6yzsx"
ClienteDto? cliente = null;
```

Se vuoi un value type, devi usare `record struct`.

# Riepilogo

In questo capitolo abbiamo completato il quadro sui tipi fondamentali per modellare dati in C#.

Abbiamo visto la distinzione tra:

```text id="bia1kh"
reference type
  variabili che contengono riferimenti a oggetti.

value type
  variabili che contengono direttamente valori.
```

Abbiamo studiato:

```text id="s0quff"
struct
  tipo valore definito dal programmatore.

enum
  insieme chiuso di valori nominati.

record class
  reference type orientato ai dati, con uguaglianza per valore.

record struct
  value type orientato ai dati, con uguaglianza per valore.

init
  proprietà impostabile solo durante l’inizializzazione.

with
  creazione di copie modificate di record.

immutabilità
  oggetti che non cambiano dopo la creazione.

uguaglianza per valore
  due oggetti sono uguali se hanno gli stessi dati.
```

Abbiamo poi distinto gli usi principali:

```text id="r3bii0"
Classe
  per entità con identità, stato e comportamento.

Struct
  per piccoli valori, preferibilmente immutabili.

Record
  per dati, DTO, messaggi, risultati e value object.

Enum
  per insiemi chiusi di opzioni o stati.

Tuple
  per risultati locali e temporanei.
```

La regola più importante è questa:

```text id="4vydz0"
Scegli il tipo in base al significato del dato,
non solo in base alla sintassi più breve.
```

Un buon modello C# non nasce solo da classi e proprietà. Nasce dalla scelta corretta tra entità, valori, stati, DTO e oggetti immutabili.

Nel prossimo capitolo approfondiremo ereditarietà, interfacce e polimorfismo: strumenti che permettono di creare gerarchie, contratti e comportamenti sostituibili.
