# 03.04 — File, directory e percorsi

Lavorare con file e cartelle è uno degli scenari più concreti nello sviluppo .NET.

Prima o poi quasi ogni applicazione deve:

* leggere un file di configurazione;
* generare un report;
* salvare un log;
* importare dati da CSV;
* esportare documenti;
* controllare se una cartella esiste;
* creare file temporanei;
* gestire permessi;
* leggere o scrivere testo con l’encoding corretto.

In .NET, la maggior parte delle API per il file system si trova nel namespace:

```csharp
System.IO
```

I tipi principali che useremo in questo capitolo sono:

```text
File
Directory
Path
FileInfo
DirectoryInfo
StreamReader
StreamWriter
Encoding
```

Il punto centrale è questo:

```text
File e directory sembrano semplici, ma nella pratica falliscono spesso:
permessi, path sbagliati, file già aperti, encoding errato, cartelle mancanti,
differenze tra sistemi operativi.
```

---

# `File`

La classe `File` fornisce metodi statici per creare, copiare, eliminare, spostare, aprire, leggere e scrivere file. Microsoft la descrive proprio come una classe con metodi statici per lavorare su singoli file e per creare oggetti `FileStream`. ([Microsoft Learn][1])

Esempio semplice:

```csharp
using System.IO;

File.WriteAllText("messaggio.txt", "Ciao dal file!");

string contenuto = File.ReadAllText("messaggio.txt");

Console.WriteLine(contenuto);
```

Output:

```text
Ciao dal file!
```

---

## Metodi comuni di `File`

```csharp
File.Exists(path);
File.ReadAllText(path);
File.ReadAllLines(path);
File.WriteAllText(path, testo);
File.WriteAllLines(path, righe);
File.AppendAllText(path, testo);
File.Copy(sorgente, destinazione);
File.Move(sorgente, destinazione);
File.Delete(path);
File.OpenRead(path);
File.OpenWrite(path);
```

Esempio:

```csharp
string path = "dati.txt";

if (File.Exists(path))
{
    string testo = File.ReadAllText(path);
    Console.WriteLine(testo);
}
else
{
    Console.WriteLine("File non trovato.");
}
```

---

## `File.Exists`

`File.Exists` controlla se un file esiste.

```csharp
if (File.Exists("dati.txt"))
{
    Console.WriteLine("Il file esiste.");
}
```

Attenzione: `File.Exists` può restituire `false` anche se il file esiste ma non abbiamo permessi sufficienti o il path non è valido.

Quindi non bisogna usarlo come unica protezione assoluta.

Esempio fragile:

```csharp
if (File.Exists(path))
{
    string testo = File.ReadAllText(path);
}
```

Tra il controllo e la lettura il file potrebbe essere cancellato, spostato o bloccato.

Meglio gestire comunque le eccezioni.

---

# `Directory`

La classe `Directory` fornisce metodi statici per lavorare con cartelle.

Microsoft documenta metodi come `Exists`, `CreateDirectory`, `EnumerateFiles`, `EnumerateDirectories` e altri per leggere e manipolare directory. ([Microsoft Learn][2])

Esempio:

```csharp
using System.IO;

string cartella = "Export";

if (!Directory.Exists(cartella))
{
    Directory.CreateDirectory(cartella);
}

Console.WriteLine("Cartella pronta.");
```

---

## Metodi comuni di `Directory`

```csharp
Directory.Exists(path);
Directory.CreateDirectory(path);
Directory.Delete(path);
Directory.GetFiles(path);
Directory.GetDirectories(path);
Directory.EnumerateFiles(path);
Directory.EnumerateDirectories(path);
Directory.GetCurrentDirectory();
Directory.SetCurrentDirectory(path);
```

Esempio:

```csharp
string cartella = "Documenti";

if (Directory.Exists(cartella))
{
    string[] file = Directory.GetFiles(cartella);

    foreach (string path in file)
    {
        Console.WriteLine(path);
    }
}
```

---

## `GetFiles` vs `EnumerateFiles`

`Directory.GetFiles` restituisce un array con tutti i file trovati.

```csharp
string[] file = Directory.GetFiles(cartella);
```

`Directory.EnumerateFiles` restituisce una sequenza enumerabile.

```csharp
IEnumerable<string> file = Directory.EnumerateFiles(cartella);
```

Per cartelle molto grandi, `EnumerateFiles` è spesso preferibile perché permette di iniziare a elaborare i risultati senza dover caricare subito tutto in memoria.

Esempio:

```csharp
foreach (string file in Directory.EnumerateFiles(cartella, "*.txt"))
{
    Console.WriteLine(file);
}
```

---

# `Path`

La classe `Path` serve a manipolare stringhe che rappresentano percorsi.

Microsoft definisce un path come una stringa che indica la posizione di un file o di una directory, non necessariamente esistente su disco. ([Microsoft Learn][3])

Esempio:

```csharp
string cartella = "Export";
string nomeFile = "report.txt";

string path = Path.Combine(cartella, nomeFile);

Console.WriteLine(path);
```

Output su Windows:

```text
Export\report.txt
```

Output su Linux/macOS:

```text
Export/report.txt
```

---

## Perché usare `Path.Combine`

Sconsigliato:

```csharp
string path = cartella + "\\" + nomeFile;
```

Questo funziona su Windows, ma non è portabile.

Meglio:

```csharp
string path = Path.Combine(cartella, nomeFile);
```

`Path.Combine` usa il separatore corretto per il sistema operativo.

---

## Metodi comuni di `Path`

```csharp
Path.Combine(...)
Path.GetFileName(path)
Path.GetFileNameWithoutExtension(path)
Path.GetExtension(path)
Path.GetDirectoryName(path)
Path.GetFullPath(path)
Path.GetTempPath()
Path.GetTempFileName()
Path.ChangeExtension(path, ".bak")
Path.HasExtension(path)
```

Esempio:

```csharp
string path = @"C:\Dati\Clienti\report.csv";

Console.WriteLine(Path.GetFileName(path));
Console.WriteLine(Path.GetFileNameWithoutExtension(path));
Console.WriteLine(Path.GetExtension(path));
Console.WriteLine(Path.GetDirectoryName(path));
```

Output:

```text
report.csv
report
.csv
C:\Dati\Clienti
```

---

# Percorsi Windows, Linux e macOS

I sistemi operativi non rappresentano i percorsi nello stesso modo.

Windows usa spesso:

```text
C:\Dati\Clienti\report.txt
```

Linux e macOS usano:

```text
/home/emilie/dati/report.txt
```

Differenze importanti:

```text
Windows
  separatore tipico: \
  drive letter: C:
  path spesso case-insensitive

Linux/macOS
  separatore: /
  nessuna drive letter
  path normalmente case-sensitive
```

---

## Separatore di directory

In .NET possiamo leggere il separatore corrente:

```csharp
char separatore = Path.DirectorySeparatorChar;

Console.WriteLine(separatore);
```

Ma nella pratica, per costruire path, è meglio usare:

```csharp
Path.Combine(...)
```

invece di concatenare separatori manualmente.

---

## Attenzione al backslash nelle stringhe C#

In C#, il carattere `\` ha significato speciale.

Questo può creare problemi:

```csharp
string path = "C:\Dati\Clienti";
```

Meglio usare stringhe verbatim:

```csharp
string path = @"C:\Dati\Clienti";
```

Oppure raddoppiare i backslash:

```csharp
string path = "C:\\Dati\\Clienti";
```

Oppure, ancora meglio, usare `Path.Combine`:

```csharp
string path = Path.Combine("C:\\", "Dati", "Clienti");
```

---

# Path assoluti e relativi

Un path **assoluto** contiene la posizione completa.

Esempio Windows:

```text
C:\Dati\Clienti\report.txt
```

Esempio Linux:

```text
/home/emilie/dati/report.txt
```

Un path **relativo** è interpretato rispetto alla directory corrente del processo.

Esempio:

```text
dati/report.txt
```

---

## Directory corrente

Per vedere la directory corrente:

```csharp
string corrente = Directory.GetCurrentDirectory();

Console.WriteLine(corrente);
```

Attenzione: la directory corrente non è sempre la cartella dove si trova l’eseguibile.

Può cambiare in base a:

* Visual Studio;
* terminale;
* servizio Windows;
* IIS;
* Docker;
* test automatici;
* task schedulato.

---

## BaseDirectory

Per ottenere la cartella dell’applicazione:

```csharp
string baseDirectory = AppContext.BaseDirectory;

Console.WriteLine(baseDirectory);
```

Spesso è più stabile di `Directory.GetCurrentDirectory()` quando vogliamo trovare file distribuiti insieme all’applicazione.

Esempio:

```csharp
string path = Path.Combine(
    AppContext.BaseDirectory,
    "config",
    "settings.json");
```

---

## Regola pratica

```text
Path relativo
  comodo, ma dipende dalla directory corrente.

Path assoluto
  più esplicito, ma meno portabile se scritto fisso.

AppContext.BaseDirectory
  utile per file vicini all’applicazione.

Path.Combine
  sempre preferibile per costruire percorsi.
```

---

# File temporanei

Le applicazioni spesso devono creare file temporanei:

* export intermedi;
* elaborazioni;
* upload;
* conversioni;
* cache temporanee;
* file di lavoro.

.NET offre metodi utili:

```csharp
Path.GetTempPath();
Path.GetTempFileName();
```

Esempio:

```csharp
string cartellaTemp = Path.GetTempPath();

Console.WriteLine(cartellaTemp);
```

---

## Creare un file temporaneo

```csharp
string fileTemporaneo = Path.GetTempFileName();

File.WriteAllText(fileTemporaneo, "Dati temporanei");

Console.WriteLine(fileTemporaneo);
```

`GetTempFileName` crea fisicamente un file vuoto e restituisce il path.

---

## Nome temporaneo senza creare subito il file

Se vuoi solo un nome casuale:

```csharp
string nome = Path.GetRandomFileName();

string path = Path.Combine(Path.GetTempPath(), nome);

Console.WriteLine(path);
```

---

## Pulizia dei file temporanei

Creare file temporanei significa anche pensare alla pulizia.

Esempio:

```csharp
string tempFile = Path.GetTempFileName();

try
{
    File.WriteAllText(tempFile, "contenuto temporaneo");

    // Elaborazione...
}
finally
{
    if (File.Exists(tempFile))
    {
        File.Delete(tempFile);
    }
}
```

Il `finally` aiuta a rimuovere il file anche in caso di errore.

---

# Permessi

Il file system può negare operazioni per molti motivi:

* il file non esiste;
* la cartella non esiste;
* l’utente non ha permessi;
* il file è in uso;
* il path punta a una directory;
* il file è read-only;
* il processo gira con un’identità diversa;
* la rete non è raggiungibile;
* il path è troppo lungo o non valido;
* antivirus o policy bloccano l’accesso.

`UnauthorizedAccessException` viene generata quando il sistema operativo nega l’accesso per un errore I/O o per un errore di sicurezza. ([Microsoft Learn][4])

---

## Esempio

```csharp
try
{
    File.WriteAllText(@"C:\Windows\test.txt", "prova");
}
catch (UnauthorizedAccessException)
{
    Console.WriteLine("Permessi insufficienti per scrivere il file.");
}
```

---

## Permessi e identità del processo

È importante chiedersi:

```text
Chi sta eseguendo il codice?
```

In locale, può essere il tuo utente Windows.

In IIS, può essere:

```text
ApplicationPoolIdentity
utente di dominio
identità configurata sull'application pool
```

In un servizio Windows, può essere:

```text
LocalSystem
NetworkService
utente dedicato
```

In Docker, può essere un utente del container.

Il codice è lo stesso, ma i permessi cambiano.

# Encoding del testo

Un file di testo è una sequenza di byte.

Per trasformare quei byte in caratteri serve un **encoding**.

Esempi:

```text
UTF-8
UTF-16
ASCII
Windows-1252
ISO-8859-1
```

In .NET, i tipi come `StreamReader` e `StreamWriter` leggono e scrivono caratteri usando un encoding per convertire testo e byte. ([Microsoft Learn][5])

---

## Scrivere con UTF-8

```csharp
using System.Text;

string path = "testo.txt";

File.WriteAllText(path, "Caffè", Encoding.UTF8);
```

Leggere con UTF-8:

```csharp
string testo = File.ReadAllText(path, Encoding.UTF8);

Console.WriteLine(testo);
```

---

## Perché l’encoding conta

Se scrivi un file con un encoding e lo leggi con un altro, i caratteri accentati possono rompersi.

Esempio di problema:

```text
Caffè
```

può diventare:

```text
CaffÃ¨
```

Questo accade spesso quando si mescolano UTF-8, Windows-1252 o ISO-8859-1.

---

## Regola pratica

```text
Per nuovi file testuali
  usa UTF-8.

Per file legacy
  verifica l’encoding reale.

Per import/export verso sistemi esterni
  concorda esplicitamente l’encoding.
```

---

# Lettura file

## Leggere tutto il testo

```csharp
string testo = File.ReadAllText("dati.txt");

Console.WriteLine(testo);
```

Comodo per file piccoli o medi.

---

## Leggere tutte le righe

```csharp
string[] righe = File.ReadAllLines("dati.txt");

foreach (string riga in righe)
{
    Console.WriteLine(riga);
}
```

Anche questo carica tutto in memoria.

---

## Leggere riga per riga

Per file grandi, meglio leggere riga per riga.

```csharp
foreach (string riga in File.ReadLines("dati.txt"))
{
    Console.WriteLine(riga);
}
```

`ReadLines` restituisce una sequenza enumerabile e permette di elaborare progressivamente.

---

## StreamReader

Per maggiore controllo:

```csharp
using StreamReader reader = new StreamReader("dati.txt", Encoding.UTF8);

string? riga;

while ((riga = reader.ReadLine()) is not null)
{
    Console.WriteLine(riga);
}
```

`using` assicura che il file venga chiuso.

---

# Scrittura file

## Scrivere tutto il testo

```csharp
File.WriteAllText("output.txt", "Contenuto del file");
```

Se il file esiste, viene sovrascritto.

---

## Scrivere righe

```csharp
string[] righe =
[
    "Prima riga",
    "Seconda riga",
    "Terza riga"
];

File.WriteAllLines("output.txt", righe);
```

---

## Accodare testo

```csharp
File.AppendAllText("log.txt", "Nuova riga di log" + Environment.NewLine);
```

---

## StreamWriter

Per scrivere progressivamente:

```csharp
using StreamWriter writer = new StreamWriter("report.txt", append: false, Encoding.UTF8);

writer.WriteLine("Report clienti");
writer.WriteLine("--------------");
writer.WriteLine("Totale: 100");
```

I costruttori di `StreamWriter` permettono di specificare path, modalità append, encoding e buffer; se il file non esiste, viene creato. ([Microsoft Learn][6])

---

# Lettura e scrittura binaria

Non tutti i file sono testo.

Per file binari possiamo usare:

```csharp
File.ReadAllBytes(path);
File.WriteAllBytes(path, bytes);
```

Esempio:

```csharp
byte[] dati = File.ReadAllBytes("immagine.jpg");

Console.WriteLine($"Byte letti: {dati.Length}");
```

Scrivere:

```csharp
File.WriteAllBytes("copia.jpg", dati);
```

# Gestione errori su file system

Le operazioni su file system possono fallire spesso.

Eccezioni comuni:

```text
FileNotFoundException
DirectoryNotFoundException
UnauthorizedAccessException
IOException
PathTooLongException
NotSupportedException
ArgumentException
SecurityException
```

In .NET moderno `SecurityException` è meno comune rispetto al vecchio .NET Framework, ma può comparire in contesti particolari.

---

## Esempio robusto di lettura

```csharp
try
{
    string testo = File.ReadAllText("dati.txt", Encoding.UTF8);
    Console.WriteLine(testo);
}
catch (FileNotFoundException)
{
    Console.WriteLine("Il file non esiste.");
}
catch (DirectoryNotFoundException)
{
    Console.WriteLine("La cartella non esiste.");
}
catch (UnauthorizedAccessException)
{
    Console.WriteLine("Permessi insufficienti per leggere il file.");
}
catch (IOException ex)
{
    Console.WriteLine("Errore di I/O durante la lettura.");
    Console.Error.WriteLine(ex);
}
```

---

## Non nascondere gli errori

Sconsigliato:

```csharp
try
{
    File.WriteAllText(path, testo);
}
catch
{
}
```

Questo rende i problemi difficili da diagnosticare.

Meglio:

```csharp
try
{
    File.WriteAllText(path, testo);
}
catch (Exception ex)
{
    Console.Error.WriteLine(ex);
    Console.WriteLine("Impossibile scrivere il file.");
}
```

Ancora meglio, in applicazioni reali, usare logging strutturato.

# Esempio completo: esportare un report

Costruiamo un piccolo esportatore di report.

Requisiti:

```text
Creare una cartella se non esiste.
Costruire il path in modo portabile.
Scrivere UTF-8.
Gestire errori comuni.
Restituire il path del file creato.
```

```csharp
using System.Text;

public class ReportExporter
{
    public string ExportDirectory { get; }

    public ReportExporter(string exportDirectory)
    {
        ArgumentException.ThrowIfNullOrWhiteSpace(exportDirectory);

        ExportDirectory = exportDirectory;
    }

    public string EsportaReportClienti(IEnumerable<string> clienti)
    {
        ArgumentNullException.ThrowIfNull(clienti);

        Directory.CreateDirectory(ExportDirectory);

        string nomeFile = $"clienti_{DateTime.Now:yyyyMMdd_HHmmss}.txt";
        string path = Path.Combine(ExportDirectory, nomeFile);

        using StreamWriter writer = new StreamWriter(
            path,
            append: false,
            Encoding.UTF8);

        writer.WriteLine("Report clienti");
        writer.WriteLine("==============");
        writer.WriteLine();

        foreach (string cliente in clienti)
        {
            writer.WriteLine(cliente);
        }

        return path;
    }
}
```

Uso:

```csharp
ReportExporter exporter = new ReportExporter(
    Path.Combine(AppContext.BaseDirectory, "export"));

List<string> clienti =
[
    "Emilie Rollandin",
    "Mario Rossi",
    "Anna Verdi"
];

try
{
    string path = exporter.EsportaReportClienti(clienti);

    Console.WriteLine($"Report creato: {path}");
}
catch (UnauthorizedAccessException)
{
    Console.WriteLine("Permessi insufficienti per creare il report.");
}
catch (IOException ex)
{
    Console.WriteLine("Errore durante la scrittura del report.");
    Console.Error.WriteLine(ex);
}
```

Questo esempio usa:

```text
Directory.CreateDirectory
  crea la cartella se manca.

Path.Combine
  costruisce il path in modo portabile.

AppContext.BaseDirectory
  usa una base stabile vicina all’applicazione.

StreamWriter
  scrive progressivamente.

Encoding.UTF8
  esplicita l’encoding.
```

# Esempio: importare un file riga per riga

Scenario:

```text
Leggere un file clienti.
Ogni riga contiene un nome.
Ignorare righe vuote.
Gestire file mancante e permessi.
```

```csharp
using System.Text;

public class ClienteImporter
{
    public List<string> ImportaNomiClienti(string path)
    {
        ArgumentException.ThrowIfNullOrWhiteSpace(path);

        List<string> clienti = new List<string>();

        foreach (string riga in File.ReadLines(path, Encoding.UTF8))
        {
            if (string.IsNullOrWhiteSpace(riga))
            {
                continue;
            }

            clienti.Add(riga.Trim());
        }

        return clienti;
    }
}
```

Uso:

```csharp
ClienteImporter importer = new ClienteImporter();

try
{
    List<string> clienti = importer.ImportaNomiClienti("clienti.txt");

    foreach (string cliente in clienti)
    {
        Console.WriteLine(cliente);
    }
}
catch (FileNotFoundException)
{
    Console.WriteLine("File clienti non trovato.");
}
catch (UnauthorizedAccessException)
{
    Console.WriteLine("Permessi insufficienti per leggere il file.");
}
catch (IOException ex)
{
    Console.WriteLine("Errore durante la lettura del file.");
    Console.Error.WriteLine(ex);
}
```

# Esempio: usare file temporanei

```csharp
string tempFile = Path.Combine(
    Path.GetTempPath(),
    Path.GetRandomFileName());

try
{
    File.WriteAllText(tempFile, "Contenuto temporaneo", Encoding.UTF8);

}
finally
{
    if (File.Exists(tempFile))

```

Qui:

```text
Path.GetTempPath
  trova la cartella temporanea.

Path.GetRandomFileName
  genera un nome casuale.

finally
  prova a pulire il file temporaneo.
```

# FileInfo e DirectoryInfo

Finora abbiamo usato classi statiche:

```csharp
File
Directory
Path
```

Esistono anche classi orientate agli oggetti:

```csharp
FileInfo
DirectoryInfo
```

Esempio:

```csharp
FileInfo file = new FileInfo("dati.txt");

if (file.Exists)
{
    Console.WriteLine(file.FullName);
    Console.WriteLine(file.Length);
    Console.WriteLine(file.LastWriteTime);
}
```

Esempio directory:

```csharp
DirectoryInfo directory = new DirectoryInfo("Export");

if (!directory.Exists)
{
    directory.Create();
}

foreach (FileInfo file in directory.EnumerateFiles("*.txt"))
{
    Console.WriteLine(file.Name);
}
```

---

## File o FileInfo?

```text
File / Directory
  comodi per operazioni singole e statiche.

FileInfo / DirectoryInfo
  comodi quando lavori più volte sullo stesso file o directory
  e vuoi proprietà e metodi sull’oggetto.
```

# Errori comuni

## Concatenare path manualmente

Sconsigliato:

```csharp
string path = cartella + "\\" + nomeFile;
```

Meglio:

```csharp
string path = Path.Combine(cartella, nomeFile);
```

---

## Assumere che la directory corrente sia quella dell’app

Sconsigliato:

```csharp
string path = "config.json";
```

Meglio, se il file è accanto all’app:

```csharp
string path = Path.Combine(AppContext.BaseDirectory, "config.json");
```

---

## Non specificare l’encoding nei file importanti

Sconsigliato:

```csharp
File.WriteAllText(path, testo);
```

Meglio, se il file deve essere interoperabile:

```csharp
File.WriteAllText(path, testo, Encoding.UTF8);
```

---

## Caricare file enormi in memoria

Sconsigliato:

```csharp
string[] righe = File.ReadAllLines(path);
```

Per file grandi:

```csharp
foreach (string riga in File.ReadLines(path))
{
    // elabora riga
}
```

---

## Dimenticare `using` con stream e writer

Sconsigliato:

```csharp
StreamWriter writer = new StreamWriter(path);
writer.WriteLine("testo");
```

Meglio:

```csharp
using StreamWriter writer = new StreamWriter(path);
writer.WriteLine("testo");
```

---

## Controllare `File.Exists` e pensare di essere al sicuro

Fragile:

```csharp
if (File.Exists(path))
{
    File.Delete(path);
}
```

Il file potrebbe cambiare tra controllo e cancellazione.

Meglio gestire anche le eccezioni:

```csharp
try
{
    if (File.Exists(path))
    {
        File.Delete(path);
    }
}
catch (IOException ex)
{
    Console.Error.WriteLine(ex);
}
catch (UnauthorizedAccessException ex)
{
    Console.Error.WriteLine(ex);
}
```

---

## Scrivere in cartelle senza permessi

Sconsigliato:

```csharp
File.WriteAllText(@"C:\Program Files\app\log.txt", testo);
```

Meglio usare cartelle appropriate:

```csharp
string appData = Environment.GetFolderPath(
    Environment.SpecialFolder.ApplicationData);

string cartella = Path.Combine(appData, "MiaApp");
Directory.CreateDirectory(cartella);

string path = Path.Combine(cartella, "log.txt");
```

---

# Linee guida pratiche

```text
Usa Path.Combine
  per costruire percorsi portabili.

Usa Directory.CreateDirectory
  prima di scrivere in una cartella che potrebbe non esistere.

Usa Encoding.UTF8
  per nuovi file testuali.

Usa File.ReadLines
  per leggere file grandi riga per riga.

Usa File.ReadAllText
  per file piccoli e semplici.

Usa StreamReader/StreamWriter
  quando vuoi più controllo.

Usa using
  per chiudere correttamente stream e writer.

Gestisci IOException e UnauthorizedAccessException
  nelle operazioni file system.

Non fidarti della directory corrente
  quando il programma gira in IIS, servizio Windows, Docker o scheduler.
```

In questo capitolo abbiamo studiato le API fondamentali per lavorare con file, directory e percorsi.

Abbiamo visto:

```text
File
  metodi statici per creare, leggere, scrivere, copiare, spostare ed eliminare file.

Directory
  metodi statici per creare, leggere ed enumerare cartelle.

Path
  metodi per manipolare stringhe di percorso in modo portabile.

FileInfo e DirectoryInfo
  alternative orientate agli oggetti.
```

Abbiamo distinto:

```text
Path assoluto
  posizione completa.

Path relativo
  interpretato rispetto alla directory corrente.

Directory corrente
  può cambiare in base al contesto di esecuzione.

AppContext.BaseDirectory
  spesso utile per file vicini all’applicazione.
```

Abbiamo introdotto:

```text

permessi
encoding
lettura e scrittura testo
lettura e scrittura binaria
gestione errori
```

