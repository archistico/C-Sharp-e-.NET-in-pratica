# 03.02 — Numeri, stringhe, date e formattazione

Dopo aver visto runtime, CLR, assembly e compilazione, entriamo nelle librerie fondamentali che usiamo ogni giorno.

Quasi ogni applicazione reale manipola:

* numeri;
* importi;
* percentuali;
* stringhe;
* testi formattati;
* date;
* orari;
* fusi orari;
* culture diverse.

Questi argomenti sembrano semplici, ma sono una fonte continua di bug.

Esempi tipici:

```text id="cmqw3y"
Un importo monetario calcolato con double produce arrotondamenti strani.
Una data salvata senza fuso orario viene interpretata male.
Una stringa ordinata alfabeticamente mette "file10" prima di "file2".
Un numero formattato in italiano non viene letto correttamente in ambiente inglese.
Una concatenazione massiva di stringhe crea molte allocazioni inutili.
```

Il punto centrale del capitolo è questo:

```text id="rrb01m"
Numeri, stringhe e date non sono dettagli banali.
Sono dati fondamentali e vanno trattati con precisione.
```

---

# Numeri interi

I numeri interi rappresentano valori senza parte decimale.

I tipi più usati sono:

```text id="5w9ymy"
byte
short
int
long
```

Nel codice quotidiano, il più comune è `int`.

```csharp id="4rq2ft"
int quantita = 10;
int eta = 42;
int numeroRighe = 150;
```

`int` è un intero a 32 bit.

Quando i valori possono diventare molto grandi, si usa `long`.

```csharp id="hp8grp"
long numeroOperazioni = 5_000_000_000;
```

---

## Scegliere il tipo intero

Regola pratica:

```text id="p9pucr"
int
  scelta normale per contatori, quantità, indici e piccoli identificativi.

long
  quando i valori possono superare i limiti di int.

byte, short
  usati più raramente, spesso in protocolli, file binari o ottimizzazioni.
```

Esempio:

```csharp id="pqmczo"
int numeroClienti = 100_000;
long numeroEventiLog = 10_000_000_000;
```

---

## Limiti dei tipi interi

Ogni tipo numerico ha limiti.

```csharp id="fh0z6d"
Console.WriteLine(int.MinValue);
Console.WriteLine(int.MaxValue);
```

Output:

```text id="fgobtc"
-2147483648
2147483647
```

Se un calcolo supera questi limiti, può verificarsi un overflow.

Per controllarlo:

```csharp id="szun5e"
checked
{
    int massimo = int.MaxValue;
    int risultato = massimo + 1;
}
```

Questo genera una `OverflowException`.

---

# Numeri decimali

I numeri con parte decimale possono essere rappresentati con:

```text id="3u9olz"
float
double
decimal
```

Questi tipi non sono equivalenti.

La scelta è molto importante.

---

# `decimal` per importi monetari

Per importi monetari e calcoli finanziari, in C# si usa normalmente `decimal`.

Esempio:

```csharp id="ft0p2k"
decimal prezzo = 19.90m;
decimal iva = prezzo * 22m / 100m;
decimal totale = prezzo + iva;

Console.WriteLine(totale);
```

Il suffisso `m` indica che il literal è di tipo `decimal`.

Senza `m`, un numero come `19.90` viene interpretato come `double`.

---

## Perché decimal per i soldi

`decimal` è pensato per rappresentare numeri decimali con maggiore precisione in base 10.

È più adatto a:

* importi;
* prezzi;
* IVA;
* sconti;
* aliquote;
* saldi;
* totali documento;
* calcoli contabili.

Esempio:

```csharp id="fd8q36"
decimal imponibile = 100m;
decimal aliquotaIva = 22m;

decimal iva = imponibile * aliquotaIva / 100m;
decimal totale = imponibile + iva;

Console.WriteLine($"IVA: {iva}");
Console.WriteLine($"Totale: {totale}");
```

Output:

```text id="hhhb8a"
IVA: 22
Totale: 122
```

---

## Attenzione agli arrotondamenti

Anche con `decimal`, gli arrotondamenti vanno gestiti esplicitamente.

```csharp id="6zsxne"
decimal valore = 10m / 3m;

Console.WriteLine(valore);
```

Output:

```text id="qxxbdi"
3,3333333333333333333333333333
```

Per arrotondare:

```csharp id="jzbgle"
decimal arrotondato = Math.Round(valore, 2);

Console.WriteLine(arrotondato);
```

Output:

```text id="5cgg41"
3,33
```

---

## Arrotondamento nei calcoli monetari

Esempio:

```csharp id="570h4c"
decimal prezzo = 19.99m;
decimal scontoPercentuale = 15m;

decimal sconto = prezzo * scontoPercentuale / 100m;
decimal totale = prezzo - sconto;

totale = Math.Round(totale, 2);

Console.WriteLine(totale);
```

Regola pratica:

```text id="cr2xmd"
Usa decimal per importi.
Arrotonda esplicitamente dove la regola di business lo richiede.
Non lasciare gli arrotondamenti al caso.
```

---

# `double` e precisione

`double` è un tipo in virgola mobile binaria a 64 bit.

È molto usato per:

* calcoli scientifici;
* misure;
* statistiche;
* grafica;
* coordinate;
* simulazioni;
* algoritmi numerici.

Esempio:

```csharp id="asb6q2"
double temperatura = 22.5;
double distanza = 1234.56;
```

---

## Il problema della rappresentazione binaria

Molti numeri decimali non possono essere rappresentati esattamente in binario.

Esempio classico:

```csharp id="fxs5m7"
double risultato = 0.1 + 0.2;

Console.WriteLine(risultato);
```

Output possibile:

```text id="jp8gh5"
0,30000000000000004
```

Questo non significa che `double` sia “sbagliato”. Significa che funziona con una rappresentazione approssimata, adatta a certi scenari ma non ideale per denaro.

---

## Confrontare double

Non conviene confrontare `double` con uguaglianza esatta dopo calcoli.

Sconsigliato:

```csharp id="kboli3"
double valore = 0.1 + 0.2;

if (valore == 0.3)
{
    Console.WriteLine("Uguale");
}
```

Meglio usare una tolleranza:

```csharp id="niy9j5"
double valore = 0.1 + 0.2;
double atteso = 0.3;
double tolleranza = 0.000001;

if (Math.Abs(valore - atteso) < tolleranza)
{
    Console.WriteLine("Abbastanza vicino");
}
```

---

## decimal o double?

```text id="8dqvea"
decimal
  importi monetari, contabilità, calcoli finanziari, base 10.

double
  misure, calcoli scientifici, coordinate, statistiche, performance numerica.
```

Esempio:

```csharp id="l408no"
decimal prezzo = 10.99m;
double latitudine = 45.7372;
```

---

# `BigInteger`

`BigInteger` rappresenta interi di dimensione arbitrariamente grande, limitati principalmente dalla memoria disponibile.

È nel namespace:

```csharp id="dfr0ht"
System.Numerics
```

Esempio:

```csharp id="8lj2f4"
using System.Numerics;

BigInteger numero = BigInteger.Parse("123456789012345678901234567890");

BigInteger risultato = numero * numero;

Console.WriteLine(risultato);
```

È utile quando `int` e `long` non bastano.

---

## Quando usare BigInteger

`BigInteger` è utile per:

* crittografia;
* matematica avanzata;
* calcoli combinatori;
* numeri enormi;
* algoritmi scientifici;
* problemi didattici o matematici.

Non serve per normali importi, quantità o identificativi.

Regola pratica:

```text id="7ma6rs"
Usa long per numeri grandi ordinari.
Usa BigInteger solo quando i limiti dei tipi primitivi non bastano davvero.
```

```text id="jmbmb7"
parte reale
```

```csharp id="2igb9w"
using System.Numerics;

Complex z1 = new Complex(3, 4);
Complex z2 = new Complex(1, 2);

Complex somma = z1 + z2;
```

I numeri complessi sono utili in contesti specifici:

* matematica;
* ingegneria;
* segnali;
* fisica;
* elaborazione numerica.

Nel gestionale classico si usano raramente, ma è utile sapere che esistono nella libreria base.

# Stringhe

In C#, `string` rappresenta una sequenza di caratteri Unicode.

Esempio:

```csharp id="eshl4i"
string nome = "Emilie";
string messaggio = "Ciao";
```

`string` è un alias di:

```csharp id="n9xk45"
System.String
```

Quindi questi due codici sono equivalenti:

```csharp id="2e3cy1"
string nome = "Emilie";
System.String nome2 = "Emilie";
```

---

## Le stringhe sono immutabili

Una stringa non viene modificata realmente. Ogni operazione che sembra modificarla crea una nuova stringa.

Esempio:

```csharp id="4ovfj2"
string testo = "Ciao";

testo = testo + " mondo";

Console.WriteLine(testo);
```

Il risultato è:

```text id="dinrdl"
Ciao mondo
```

Ma internamente la stringa originale `"Ciao"` non viene modificata. Viene creata una nuova stringa.

Questo è importante per le performance quando facciamo molte concatenazioni.

---

# Interpolazione

L’interpolazione permette di inserire valori dentro una stringa.

```csharp id="2lub8q"
string nome = "Emilie";
int eta = 42;

string messaggio = $"Ciao, {nome}. Hai {eta} anni.";

Console.WriteLine(messaggio);
```

Output:

```text id="4hwp3z"
Ciao, Emilie. Hai 42 anni.
```

È più leggibile della concatenazione:

```csharp id="4tzkq8"
string messaggio = "Ciao, " + nome + ". Hai " + eta + " anni.";
```

---

## Interpolazione con formattazione

Possiamo formattare valori dentro l’interpolazione.

```csharp id="vvjmx7"
decimal totale = 1234.5m;

Console.WriteLine($"Totale: {totale:C}");
```

`C` indica il formato valuta.

Il risultato dipende dalla cultura corrente.

In Italia può essere:

```text id="eimyke"
Totale: 1.234,50 €
```

In ambiente en-US può essere:

```text id="pt7uw1"
Totale: $1,234.50
```

---

# Formattazione

La formattazione trasforma un valore in testo.

Esempio:

```csharp id="9xzf9i"
decimal importo = 1234.5m;

string testo = importo.ToString("N2");

Console.WriteLine(testo);
```

`N2` significa numero con due decimali.

Microsoft documenta le stringhe di formato numerico standard come combinazione di uno specificatore di formato e, opzionalmente, uno specificatore di precisione, per esempio `C`, `N2`, `P1` o simili. ([Microsoft Learn][1])

---

## Formati numerici comuni

```text id="cdaj53"
C
  valuta.

N
  numero con separatori.

F
  fixed-point.

P
  percentuale.

D
  intero decimale.

X
  esadecimale.
```

Esempio:

```csharp id="6etfkj"
decimal valore = 1234.567m;

Console.WriteLine(valore.ToString("C"));
Console.WriteLine(valore.ToString("N2"));
Console.WriteLine(valore.ToString("F3"));
Console.WriteLine((0.25m).ToString("P"));
```

Output possibile in cultura italiana:

```text id="ybgrdo"
1.234,57 €
1.234,57
1234,567
25,00%
```

---

## Formattazione personalizzata

Possiamo usare anche formati personalizzati.

```csharp id="2in31j"
decimal valore = 1234.5m;

Console.WriteLine(valore.ToString("#,##0.00"));
```

Output in cultura italiana:

```text id="4jswk5"
1.234,50
```

Attenzione: separatori e simboli dipendono comunque dalla cultura, salvo uso esplicito di `CultureInfo`.

---

# Culture e localizzazione

La **culture** influenza il modo in cui numeri, date, valute e testi vengono formattati e interpretati.

Esempio:

```text id="d2b843"
Italia
  1.234,56

Stati Uniti
  1,234.56
```

Lo stesso numero può essere rappresentato in modi diversi.

---

## CultureInfo

Il tipo principale è:

```csharp id="ttjeem"
System.Globalization.CultureInfo
```

Esempio:

```csharp id="5a2e48"
using System.Globalization;

decimal importo = 1234.56m;

CultureInfo it = new CultureInfo("it-IT");
CultureInfo us = new CultureInfo("en-US");

Console.WriteLine(importo.ToString("C", it));
Console.WriteLine(importo.ToString("C", us));
```

Output:

```text id="c64y06"
1.234,56 €
$1,234.56
```

La documentazione Microsoft spiega che formati come `C` usano informazioni della cultura corrente, per esempio il simbolo valuta e il formato numerico definiti da `NumberFormatInfo`. ([Microsoft Learn][2])

---

## Parsing e culture

Anche la conversione da testo a numero dipende dalla cultura.

```csharp id="ejrdds"
using System.Globalization;

string testoItaliano = "1234,56";

decimal valore = decimal.Parse(
    testoItaliano,
    new CultureInfo("it-IT"));

Console.WriteLine(valore);
```

Se usiamo la cultura sbagliata, il parsing può fallire o interpretare il valore in modo inatteso.

---

## InvariantCulture

`CultureInfo.InvariantCulture` è una cultura stabile, indipendente dalle impostazioni dell’utente.

È utile per:

* file tecnici;
* log;
* dati da scambiare tra sistemi;
* serializzazioni semplici;
* formati interni.

Esempio:

```csharp id="ec8516"
using System.Globalization;

decimal importo = 1234.56m;

string testo = importo.ToString(CultureInfo.InvariantCulture);

Console.WriteLine(testo);
```

Output:

```text id="iz7qvc"
1234.56
```

Regola pratica:

```text id="vex03q"
Interfaccia utente
  usa la cultura dell’utente.

Dati tecnici o scambio macchina-macchina
  usa InvariantCulture o formati standard.
```

# `StringBuilder`

Poiché le stringhe sono immutabili, concatenare molte stringhe in un ciclo può creare molte allocazioni.

Esempio poco efficiente:

```csharp id="bdu3qe"
string risultato = "";

for (int i = 0; i < 1000; i++)
{
    risultato += i + ", ";
}
```

Ogni concatenazione può creare una nuova stringa.

`StringBuilder` rappresenta una stringa modificabile e può essere utile quando dobbiamo costruire testo con molte operazioni ripetute. Microsoft descrive `StringBuilder` come una classe per modificare stringhe senza creare ogni volta un nuovo oggetto stringa. ([Microsoft Learn][3])

---

## Esempio con StringBuilder

```csharp id="vo63sa"
using System.Text;

StringBuilder builder = new StringBuilder();

for (int i = 0; i < 1000; i++)
{
    builder.Append(i);
    builder.Append(", ");
}

string risultato = builder.ToString();

Console.WriteLine(risultato);
```

---

## Quando usare StringBuilder

Usalo quando:

* costruisci testo in un ciclo;
* fai molte concatenazioni;
* generi report testuali;
* generi SQL o HTML in modo controllato;
* accumuli righe progressivamente.

Esempio:

```csharp id="j12o7y"
using System.Text;

StringBuilder sb = new StringBuilder();

sb.AppendLine("Report clienti");
sb.AppendLine("--------------");

foreach (Cliente cliente in clienti)
{
    sb.AppendLine($"{cliente.Id}: {cliente.Nome}");
}

string report = sb.ToString();
```

---

## Quando non serve

Per poche concatenazioni, l’interpolazione è più leggibile.

```csharp id="3mncla"
string messaggio = $"{nome} {cognome} - {email}";
```

Non serve usare `StringBuilder` per ogni stringa.

Regola pratica:

```text id="k175b0"
Poche stringhe
  interpolazione.

Molte concatenazioni ripetute
  StringBuilder.
```

---

# Ordinamento e confronto stringhe

Confrontare stringhe non è banale.

Ci sono diversi tipi di confronto:

* sensibile a maiuscole/minuscole;
* non sensibile a maiuscole/minuscole;
* dipendente dalla cultura;
* ordinale;
* linguistico;
* numerico.

---

## Confronto semplice

```csharp id="xfwp9s"
string a = "emilie";
string b = "Emilie";

Console.WriteLine(a == b);
```

Output:

```text id="0pxnkd"
False
```

Il confronto distingue maiuscole e minuscole.

---

## Confronto ignorando maiuscole/minuscole

```csharp id="0mhy44"
bool uguali = string.Equals(
    a,
    b,
    StringComparison.OrdinalIgnoreCase);

Console.WriteLine(uguali);
```

Output:

```text id="7txb28"
True
```

---

## Ordinal vs culture-sensitive

Confronto ordinale:

```csharp id="7770os"
string.Equals(a, b, StringComparison.OrdinalIgnoreCase);
```

È basato sui valori Unicode ed è stabile.

Confronto dipendente dalla cultura:

```csharp id="d05bq5"
string.Equals(a, b, StringComparison.CurrentCultureIgnoreCase);
```

È influenzato dalla cultura corrente.

Regola pratica:

```text id="tpb3bg"
Chiavi tecniche, codici, identificatori
  usa Ordinal o OrdinalIgnoreCase.

Testi mostrati all’utente
  valuta confronti culture-sensitive.
```

Esempio tecnico:

```csharp id="x33p63"
if (codice.Equals("ADMIN", StringComparison.OrdinalIgnoreCase))
{
    Console.WriteLine("Codice amministratore");
}
```

---

# Ordinamento stringhe

Ordinamento lessicografico classico:

```csharp id="6x3l3o"
List<string> file =
[
    "file1",
    "file10",
    "file2"
];

file.Sort(StringComparer.Ordinal);

Console.WriteLine(string.Join(", ", file));
```

Output:

```text id="m9qyds"
file1, file10, file2
```

Dal punto di vista lessicografico, `"file10"` viene prima di `"file2"` perché il confronto procede carattere per carattere.

---

# Ordinamento numerico delle stringhe

In .NET 10 è stata introdotta l’opzione `CompareOptions.NumericOrdering`, che permette confronti in cui le sequenze di cifre vengono ordinate in base al loro valore numerico: per esempio `"2"` viene prima di `"10"` invece del contrario lessicografico. ([Microsoft Learn][4])

Esempio concettuale:

```csharp id="3z0mxk"
using System.Globalization;

List<string> file =
[
    "file1",
    "file10",
    "file2"
];

CompareInfo compareInfo = CultureInfo.InvariantCulture.CompareInfo;

file.Sort((a, b) => compareInfo.Compare(
    a,
    b,
    CompareOptions.NumericOrdering));

Console.WriteLine(string.Join(", ", file));
```

Output atteso:

```text id="1q7sdw"
file1, file2, file10
```

---

## Quando è utile

L’ordinamento numerico è utile per:

* nomi file;
* codici con numeri;
* versioni semplici;
* etichette tipo `Sala1`, `Sala2`, `Sala10`;
* ordinamenti mostrati all’utente.

Esempio:

```text id="s7pxw1"
Tavolo1
Tavolo2
Tavolo10
```

è spesso più naturale di:

```text id="bk0vmd"
Tavolo1
Tavolo10
Tavolo2
```

# Date e orari

Le date sono un altro punto delicato.

In .NET troviamo diversi tipi:

```text id="vzg7pi"
DateTime
DateOnly
TimeOnly
TimeSpan
DateTimeOffset
TimeZoneInfo
```

Abbiamo già introdotto alcuni di questi tipi nei capitoli precedenti. Qui li colleghiamo alla formattazione, alle culture e ai fusi orari.

---

# `DateTime`

`DateTime` rappresenta una data e un orario.

```csharp id="el1ml4"
DateTime adesso = DateTime.Now;

Console.WriteLine(adesso);
```

Può rappresentare:

* data locale;
* data UTC;
* data con `Kind` non specificato.

La proprietà importante è:

```csharp id="85m98c"
adesso.Kind
```

I valori possibili sono:

```text id="94yhrs"
Local
Utc
Unspecified
```

---

## DateTime.Now e DateTime.UtcNow

```csharp id="f1q7k8"
DateTime locale = DateTime.Now;
DateTime utc = DateTime.UtcNow;

Console.WriteLine(locale);
Console.WriteLine(utc);
```

`Now` usa l’ora locale della macchina.

`UtcNow` usa UTC.

Regola pratica:

```text id="3t8xzk"
DateTime.Now
  utile per visualizzare o lavorare localmente.

DateTime.UtcNow
  spesso migliore per log, eventi tecnici e salvataggi indipendenti dal fuso.
```

---

# `DateOnly` e `TimeOnly`

`DateOnly` rappresenta solo una data, senza orario.

```csharp id="c9r1mx"
DateOnly dataNascita = new DateOnly(1984, 5, 10);
```

È utile per:

* data di nascita;
* data documento;
* giorno contabile;
* data di scadenza;
* giorno di prenotazione.

`TimeOnly` rappresenta solo un orario.

```csharp id="p5tnv6"
TimeOnly oraApertura = new TimeOnly(9, 30);
```

È utile per:

* orario di apertura;
* orario di chiusura;
* fasce orarie;
* appuntamenti senza data specifica.

---

# `TimeSpan`

`TimeSpan` rappresenta una durata.

```csharp id="2t6mfu"
TimeSpan durata = TimeSpan.FromMinutes(90);

Console.WriteLine(durata);
```

Esempio:

```csharp id="m5gi3u"
DateTime inizio = DateTime.Now;
DateTime fine = inizio.AddHours(2);

TimeSpan differenza = fine - inizio;

Console.WriteLine(differenza.TotalMinutes);
```

Output:

```text id="p9fbcs"
120
```

# Formattare date

Le date possono essere formattate con stringhe standard o personalizzate.

```csharp id="gbacql"
DateTime data = new DateTime(2026, 4, 24, 15, 30, 0);

Console.WriteLine(data.ToString("d"));
Console.WriteLine(data.ToString("D"));
Console.WriteLine(data.ToString("yyyy-MM-dd"));
Console.WriteLine(data.ToString("dd/MM/yyyy HH:mm"));
```

Output possibile in italiano:

```text id="woeo9m"
24/04/2026
venerdì 24 aprile 2026
2026-04-24
24/04/2026 15:30
```

---

## Formato ISO

Per scambio dati, log e API è spesso preferibile un formato ISO.

```csharp id="n1li9q"
DateTime utc = DateTime.UtcNow;

string testo = utc.ToString("O");

Console.WriteLine(testo);
```

Il formato `"O"` è round-trip: conserva informazioni utili per ricostruire il valore.

---

# Culture e date

Anche le date dipendono dalla cultura.

```csharp id="k14dbp"
using System.Globalization;

DateTime data = new DateTime(2026, 4, 24);

Console.WriteLine(data.ToString("D", new CultureInfo("it-IT")));
Console.WriteLine(data.ToString("D", new CultureInfo("en-US")));
```

Output:

```text id="0iq361"
venerdì 24 aprile 2026
Friday, April 24, 2026
```

# Fusi orari

I fusi orari sono più complessi di quanto sembri.

Non basta dire:

```text id="zfmb9p"
Sono le 10:00.
```

Bisogna chiedersi:

```text id="y7movp"
Le 10:00 dove?
In quale fuso?
In quale data?
C’è ora legale?
Va salvato come istante assoluto o come orario locale?
```

---

## TimeZoneInfo

`TimeZoneInfo` rappresenta un fuso orario.

Esempio:

```csharp id="585f9a"
TimeZoneInfo locale = TimeZoneInfo.Local;

Console.WriteLine(locale.Id);
Console.WriteLine(locale.DisplayName);
```

Conversione:

```csharp id="tlxvqk"
DateTime utc = DateTime.UtcNow;

TimeZoneInfo fusoRoma = TimeZoneInfo.FindSystemTimeZoneById("Europe/Rome");

DateTime oraRoma = TimeZoneInfo.ConvertTimeFromUtc(utc, fusoRoma);

Console.WriteLine(oraRoma);
```

Attenzione: gli identificativi dei fusi possono differire tra sistemi operativi e versioni. Nelle applicazioni cross-platform bisogna verificare bene questo aspetto.

# `DateTimeOffset`

`DateTimeOffset` rappresenta una data e ora insieme a un offset rispetto a UTC.

Esempio:

```csharp id="2el1jt"
DateTimeOffset adesso = DateTimeOffset.Now;

Console.WriteLine(adesso);
Console.WriteLine(adesso.Offset);
```

La struttura `DateTimeOffset` include un valore `DateTime` e una proprietà `Offset` che rappresenta la differenza rispetto a UTC; a differenza di `DateTime`, non ha una proprietà `Kind`. ([Microsoft Learn][5])

---

## Perché DateTimeOffset è importante

`DateTimeOffset` è spesso preferibile quando dobbiamo registrare un momento reale nel tempo.

Esempi:

* log applicativi;
* eventi;
* movimenti;
* transazioni;
* audit;
* creazione/modifica record;
* messaggi tra sistemi.

Microsoft indica `DateTimeOffset` come adatto a identificare in modo univoco un singolo punto nel tempo, per esempio per loggare tempi di transazioni, eventi applicativi o creazione/modifica file. ([Microsoft Learn][6])

Esempio:

```csharp id="b6nvuy"
DateTimeOffset evento = DateTimeOffset.UtcNow;

Console.WriteLine(evento);
```

---

## DateTime vs DateTimeOffset

`DateTime` ha una proprietà `Kind`, che distingue solo tra UTC, Local e Unspecified. `DateTimeOffset` conserva invece l’offset rispetto a UTC, ma non conserva il vero fuso orario storico: un offset `+02:00` non dice da solo se il fuso fosse Europe/Rome, Africa/Johannesburg o altro. ([Microsoft Learn][7])

Regola pratica:

```text id="ot9b8z"
DateOnly
  solo data, senza orario.

TimeOnly
  solo orario, senza data.

DateTime
  data e ora, utile ma da usare con attenzione su Kind.

DateTimeOffset
  data e ora con offset, ottimo per eventi reali e log.

TimeSpan
  durata.
```

---

## Esempio pratico

```csharp id="ub5zp3"
public class Movimento
{
    public int Id { get; }
    public decimal Importo { get; }
    public DateTimeOffset DataRegistrazione { get; }

    public Movimento(int id, decimal importo)
    {
        Id = id;
        Importo = importo;
        DataRegistrazione = DateTimeOffset.UtcNow;
    }
}
```

Qui salviamo il momento della registrazione in modo non ambiguo.

# Esempio completo del capitolo

Costruiamo un piccolo riepilogo documento.

Vogliamo usare:

* `decimal` per importi;
* formattazione valuta;
* cultura italiana e americana;
* string interpolation;
* `StringBuilder`;
* `DateTimeOffset`;
* formato ISO per log.

```csharp id="5pp3mn"
using System.Globalization;
using System.Text;

public class RigaDocumento
{
    public string Descrizione { get; }
    public int Quantita { get; }
    public decimal PrezzoUnitario { get; }

    public decimal Subtotale => Quantita * PrezzoUnitario;

    public RigaDocumento(string descrizione, int quantita, decimal prezzoUnitario)
    {
        ArgumentException.ThrowIfNullOrWhiteSpace(descrizione);

        if (quantita <= 0)
        {
            throw new ArgumentOutOfRangeException(nameof(quantita));
        }

        if (prezzoUnitario < 0)
        {
            throw new ArgumentOutOfRangeException(nameof(prezzoUnitario));
        }

        Descrizione = descrizione;
        Quantita = quantita;
        PrezzoUnitario = prezzoUnitario;
    }
}

CultureInfo culturaItaliana = new CultureInfo("it-IT");
CultureInfo culturaAmericana = new CultureInfo("en-US");

List<RigaDocumento> righe =
[
    new RigaDocumento("Libro C#", 2, 29.90m),
    new RigaDocumento("Corso .NET", 1, 120m),
    new RigaDocumento("Supporto tecnico", 3, 45.50m)
];

decimal totale = righe.Sum(r => r.Subtotale);
DateTimeOffset dataDocumento = DateTimeOffset.UtcNow;

StringBuilder report = new StringBuilder();

report.AppendLine("Riepilogo documento");
report.AppendLine("-------------------");
report.AppendLine($"Data UTC: {dataDocumento:O}");
report.AppendLine();

foreach (RigaDocumento riga in righe)
{
    report.AppendLine(
        $"{riga.Descrizione} - " +
        $"{riga.Quantita} x " +
        $"{riga.PrezzoUnitario.ToString("C", culturaItaliana)} = " +
        $"{riga.Subtotale.ToString("C", culturaItaliana)}");
}

report.AppendLine();
report.AppendLine($"Totale IT: {totale.ToString("C", culturaItaliana)}");
report.AppendLine($"Totale US: {totale.ToString("C", culturaAmericana)}");

Console.WriteLine(report.ToString());
```

Output possibile:

```text id="1hg5zb"
Riepilogo documento
-------------------
Data UTC: 2026-04-25T14:30:00.0000000+00:00

Libro C# - 2 x 29,90 € = 59,80 €
Corso .NET - 1 x 120,00 € = 120,00 €
Supporto tecnico - 3 x 45,50 € = 136,50 €

Totale IT: 316,30 €
Totale US: $316.30
```

In questo esempio:

```text id="nh73nl"
decimal
  usato per importi.

CultureInfo
  usato per formattare in culture diverse.

StringBuilder
  usato per costruire un report multilinea.

DateTimeOffset
  usato per registrare un momento temporale non ambiguo.

Formato O
  usato per rappresentazione round-trip.
```

# Errori comuni

## Usare double per importi monetari

Sconsigliato:

```csharp id="tm0k9u"
double prezzo = 19.90;
```

Meglio:

```csharp id="hlm2mg"
decimal prezzo = 19.90m;
```

---

## Confrontare double con `==` dopo calcoli

Sconsigliato:

```csharp id="d90q76"
if (valore == 0.3)
{
}
```

Meglio usare una tolleranza.

---

## Ignorare la cultura nel parsing

Rischioso:

```csharp id="5a5675"
decimal valore = decimal.Parse("1234,56");
```

Meglio:

```csharp id="to6whm"
decimal valore = decimal.Parse(
    "1234,56",
    new CultureInfo("it-IT"));
```

---

## Usare concatenazione massiva di stringhe in ciclo

Sconsigliato:

```csharp id="6elnpe"
string testo = "";

foreach (var riga in righe)
{
    testo += riga + "\n";
}
```

Meglio:

```csharp id="v06qs7"
StringBuilder sb = new StringBuilder();

foreach (var riga in righe)
{
    sb.AppendLine(riga.ToString());
}
```

---

## Ordinare stringhe numerate con confronto lessicografico

Risultato poco naturale:

```text id="evr9e6"
file1
file10
file2
```

Con .NET 10, valutare `CompareOptions.NumericOrdering` quando serve ordinamento naturale. ([Microsoft Learn][4])

---

## Salvare date locali senza contesto

Rischioso:

```csharp id="xnf5g9"
DateTime dataEvento = DateTime.Now;
```

Se il dato rappresenta un evento reale da confrontare tra sistemi, spesso è meglio:

```csharp id="awfy6t"
DateTimeOffset dataEvento = DateTimeOffset.UtcNow;
```

---

## Confondere durata e data

Sbagliato concettualmente:

```csharp id="7useis"
DateTime durata = new DateTime(0);
```

Meglio:

```csharp id="el278o"
TimeSpan durata = TimeSpan.FromMinutes(90);
```
