# 14.04 — Roadmap di studio

Questa appendice propone diversi percorsi di studio attraverso il libro.

Non tutti i lettori hanno lo stesso obiettivo. Alcuni vogliono partire da zero, altri vogliono diventare backend developer, altri ancora sono interessati a desktop, architettura, performance o microservizi.

Il punto centrale è questo:

```text id="ueyw6f"
Il libro può essere letto in ordine,
ma può anche essere usato come mappa:
ogni percorso evidenzia quali parti studiare prima,
quali approfondire dopo e quali lasciare come riferimento.
```

---

# Come usare questa roadmap

La roadmap è divisa per obiettivo:

```text id="dmgw02"
percorso principiante
percorso sviluppatore backend
percorso sviluppatore web
percorso sviluppatore desktop
percorso architettura
percorso performance
percorso microservizi
```

Ogni percorso indica:

```text id="dz6o20"
capitoli prioritari
capitoli da studiare dopo
capitoli da consultare quando servono
obiettivo finale del percorso
```

La regola pratica è questa:

```text id="bqaeu7"
Se parti da zero, segui il percorso principiante.
Se hai già esperienza, scegli il percorso più vicino al tuo obiettivo.
```

# Cosa non fare

## Non iniziare dai microservizi

```text id="sgp7jg"
Prima impara monolite modulare, API, dati, test e deploy.
Poi valuta i microservizi.
```

---

## Non saltare i test

```text id="c8yd2e"
Senza test, refactoring e architettura restano fragili.
```

---

## Non studiare EF Core solo superficialmente

```text id="ic1qhu"
EF Core è comodo,
ma se non conosci query, tracking e performance
può generare problemi difficili da diagnosticare.
```

---

## Non confondere async con parallelismo

```text id="q88jiv"
async/await serve per non bloccare durante attese.
Il parallelismo serve per usare più CPU.
```

---

## Non scegliere la UI prima del problema

```text id="ioxo9a"
Blazor, WPF, MAUI e Avalonia risolvono problemi diversi.
Prima chiarisci target, distribuzione e utenti.
```

# Riepilogo

Questa roadmap propone diversi percorsi:

```text id="capdui"
principiante
backend
web
desktop
architettura
performance
microservizi
```

La regola principale è questa:

```text id="xkov30"
Non esiste un unico percorso valido per tutti.
Esiste il percorso giusto per il tuo obiettivo attuale.
```

Per chi vuole diventare sviluppatore .NET completo, la progressione più solida è:

```text id="m6jtoe"
C# moderno
.NET e librerie fondamentali
LINQ e dati
ASP.NET Core
EF Core
test
architettura modulare
Blazor/API
Docker e CI/CD
osservabilità
microservizi solo quando servono davvero
```

La roadmap non deve essere letta come una gabbia, ma come una mappa: aiuta a orientarsi, a scegliere cosa studiare subito e a capire quali argomenti approfondire quando il progetto o il lavoro lo richiedono.
